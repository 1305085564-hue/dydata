# 批改台与发布履约页面整改计划（按新美学规范调整版）

> 基于最新 `docs/美学规范.md` 调整。新规范仅保留 3 条硬标准，其余均为推荐/可破格。本清单据此软化措辞，将部分「必须整改」改为「优先整改/推荐调整」，并明确援引规范中的破格案例库。

---

## 核心设计裁决（调整版）

### 硬标准（不可动摇，必须整改）

1. **Primary 默认只有暖橙**：全站主 CTA 保持 `#D97757`。
2. **UI 组件默认不使用深色背景**：按钮/卡片/chip/徽章/标签/分页器/Tab 不使用 `bg-zinc-900/950/black`。
3. **图标系统统一 Lucide**：1.5px stroke。

### 推荐整改（保持克制，但允许破格）

- 优先使用「极淡墨水晕染（≤0.06 透明度）+ 状态点」替代高饱和浅彩色底。
- 输入框优先使用 Soft Fill（`bg-zinc-100/70 border-transparent`）。
- 优先移除 `animate-ping`、`animate-pulse` 等高频闪烁动效。
- Tooltip 统一使用 `bg-zinc-900 text-white`。

### 破格保留（已收入规范案例库）

- **今日待处理深色卡片**：案例库 #1，一页一魂主角特权。
- **AI 辅助分析暖橙导轨**：案例库 #2，AI 结论模块空间定位。
- **月度矩阵格子 Hover 缩放**：案例库 #3，小单元格物理反馈。

## 一、发布履约页面 (/admin/fulfillment)

### 1. 待审核申诉 Badge（数量指示器）

**偏离位置**：`fulfillment-workbench.tsx#L571-L574`

**当前代码**：`bg-amber-100 text-amber-800 animate-pulse`

**新规范立场**：
- `animate-pulse` **优先不推荐**用于状态指示。
- `bg-amber-100` 等高饱和浅底属于「推荐少用」的彩色塑料底，但琥珀色用于「待处理」语义是允许的。

**调整方案**：
- **必须**：移除 `animate-pulse`。
- **推荐**：将 Badge 改为极淡墨水晕染态或灰阶底色 + outline + 状态点，例如：
  - `bg-[#D99E55]/0.04 border border-[#D99E55]/15 text-[#D99E55]`
  - 或 `bg-zinc-100/50 border border-zinc-200 text-zinc-500`

### 2. 今日待处理焦点卡片 (StatsBar)

**偏离位置**：`stats-bar.tsx#L15-L29`

**当前代码**：`bg-zinc-950 p-5 text-white shadow-lg ring-1 ring-white/10 hover:shadow-xl` + `animate-ping`

**新规范立场**：
- 深色卡片违反硬标准 #2，但属于规范破格案例库 #1（一页一魂主角特权）。
- `animate-ping` 和强阴影属于「推荐少用」的视觉噪音。

**调整方案**：
- **破格保留**：深色卡片主体。
- **必须移除**：`animate-ping` 闪烁点。
- **推荐**：将 `shadow-lg / hover:shadow-xl` 降为极淡阴影，例如 `shadow-[0_2px_8px_rgba(0,0,0,0.08)]`。
- **内部约束**：数字和标签不使用彩色塑料底，保持 white/zinc 灰阶。

### 3. 数据大盘的多彩数字与标签 (StatsBar)

**偏离位置**：`stats-bar.tsx#L46`、`stats-bar.tsx#L83-L95`

**当前代码**：`bg-amber-50 text-amber-800`；数字使用 `text-[#6FAA7D]`、`text-[#8AA8C7]`、`text-[#C9604D]`

**新规范立场**：
- 彩色数字和饱和浅底**不推荐**用于精密数据大盘。
- 彩色应作为信号出现，且周围有充足留白。

**调整方案**：
- **推荐**：指标大字统一为 `text-zinc-800 font-semibold`，回归纯灰阶仪器感。
- **推荐**：顶部标签改用状态点表达语义（绿/青/红小圆点 + zinc 文字）。
- **可破格**：若该数据为页面级异常告警，允许使用矿物红高亮，但需保证是唯一强色焦点。

### 4. 团队/小组筛选器背景 (FilterBar)

### 5. 飞书自动催交控制台阴影 (FilterBar)

**偏离位置**：`filter-bar.tsx#L158`

**当前代码**：`shadow-sm hover:shadow-md transition-shadow`

**新规范立场**：
- 阴影为稀有资源，默认不推荐用于控制台/表单控件。

**调整方案**：
- **推荐**：移除 `shadow-sm` 和 `hover:shadow-md`，使用 `bg-white border-zinc-200` 呈现。
- **可破格**：若该控制台为页面高频锚点，可保留极淡阴影作为「实体感锚定」。

### 6. 异常队列的斑马纹与彩色次级按钮 (ExceptionQueue)

**偏离位置**：`exception-queue.tsx#L190`、`L252`、`L320-L347`

**当前代码**：
- 表格使用 `odd:bg-white even:bg-zinc-50/20` 斑马纹
- 操作按钮使用 `text-[#6FAA7D] border-[#6FAA7D]/30 hover:bg-[#6FAA7D]/5` 等彩色描边

**新规范立场**：
- 斑马纹现在属于「允许但默认不推荐」，极端密集场景可谨慎使用。
- 次级按钮推荐白底 outline，不强制禁止彩色描边，但推荐避免塑料感。

**调整方案**：
- **推荐**：去除斑马纹，还原纯白底 + `hover:bg-zinc-100`。
- **推荐**：异常操作按钮改为标准白底 outline：
  - `variant="outline" className="border-zinc-200 text-zinc-700 hover:bg-zinc-50"`
- **可破格**：若操作语义极强（如「驳回」为危险操作），允许使用矿物红描边，但需保证该红色在页面中有明确语义一致性。

### 7. 表单输入框默认状态 (ExceptionQueue & MemberDrawer)

**偏离位置**：`exception-queue.tsx#L372`、`member-drawer.tsx#L449`

**当前代码**：默认 `bg-white border-zinc-200`

**新规范立场**：
- Soft Fill 是输入框的推荐默认态，但不是硬标准。

**调整方案**：
- **推荐**：默认改为 Soft Fill：`bg-zinc-100/70 border-transparent text-zinc-800`。
- **Focus**：变为 `bg-white border-zinc-200 shadow-sm focus:ring-1 focus:ring-zinc-950/5`。
- **可破格**：高密度表单或搜索框可使用白底默认，但需保持聚焦态一致。

### 8. 月度矩阵的缩放抖动与 Tooltip 深色偏离 (MonthlyMatrix)

- **推荐**：确保 `scale` 不伴随 layout 属性变化，避免容器抖动。

## 二、批改台页面 (/admin/content)

### 1. 异常预警雷达闪烁与 Badge 塑料感

**偏离位置**：`content-page-client.tsx#L262-L265`、`L281-L287`

**当前代码**：`animate-ping` 红色闪烁点；限流 Badge 使用 `bg-red-50 text-red-600`

**新规范立场**：
- `animate-ping/pulse` 优先不推荐。
- `bg-red-50` 属于推荐少用的饱和浅底。

**调整方案**：
- **必须**：移除 `animate-ping`。
- **推荐**：异常 Badge 改为墨水晕染态：
  - `bg-[#C9604D]/0.04 border border-[#C9604D]/15 text-[#C9604D]`
- **可破格**：若该异常为页面级严重告警，允许使用更强烈的红色，但需是唯一 Critical 焦点。

- **可保留**：若当前 `active:translate-y-0` 已实现明显按压反馈且无 layout 抖动，可保留。

### 7. AI 辅助分析卡片内部的装饰性粗导轨与渐变底

**偏离位置**：`content-detail-dialog.tsx#L1312`

**当前代码**：`border-l-[3px] border-l-[#D97757] bg-gradient-to-b from-[#FFF8F3] to-white`

**新规范立场**：
- 该卡片属于规范破格案例库 #2：AI 结论模块可使用左侧暖橙导轨，但需克制。
- `from-[#FFF8F3]` 高饱和浅橙渐变不推荐。

**调整方案**：
- **破格保留**：左侧暖橙导轨，但收窄为规范 2px：`border-l-2 border-l-[#D97757]`。
- **必须移除**：高饱和渐变背景。
- **推荐**：内部改用 `bg-zinc-100/50` 次级盆地，或极淡墨水晕染 `bg-[#D97757]/0.04`。
- **推荐**：用 AI SparklesIcon + Eyebrow 描述替代纯装饰性导轨，降低「导轨过载」感。

### 8. 全屏 Loading 遮罩层深度偏离

**偏离位置**：`content-diagnosis-workbench.tsx#L1378`

**当前代码**：`bg-zinc-950/90`

**新规范立场**：
- 规范推荐模态遮罩 `bg-zinc-950/20`。
- 全屏 loading 遮罩可适当加深以保证内容隐藏，属于可破格场景。

**调整方案**：
- **推荐**：调整为 `bg-zinc-950/40` 配合 `backdrop-blur-sm`，既保证遮罩层级，又避免重度全黑压迫感。
- **可破格**：若需要更强的「阻断感」，可使用 `bg-zinc-950/60`，但不建议 `/90`。

1. **静态代码检查**：执行 `npm run lint` 和 `npm run build`，确保重构无 TS 错误或打包断裂。
2. **视觉验收**：在本地访问 `/admin/fulfillment` 和 `/admin/content`，验证：
   - 无 `animate-ping/pulse` 高频闪烁
   - Tooltip 统一深色材质
   - 输入框优先 Soft Fill
   - 主 CTA 保持暖橙
   - 深色 UI 组件仅出现在破格案例（今日待处理、AI 导轨等）
   - 表格/列表保持克制，无大面积塑料彩色底
