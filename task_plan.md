# 个人成长页后端数据契约实施计划

## 目标

在不改正式前端设计、不实现 AI V2、不改数据库结构的前提下，完成 `/growth` 真实数据底座、统一六维规则、可信度与规则诊断、最小实名对标、脚本降级、数据契约、一次性验证页和死代码清理。

## 阶段

- [x] 阶段 1：核对 GitHub 最新基线、项目规则、产品方案和现有成长页链路
- [x] 阶段 2：建立干净工作区并跑现有测试基线
- [x] 阶段 3：先写契约与规则层失败测试
- [x] 阶段 4：实现统一规则契约并接入加载器
- [x] 阶段 5：清理虚拟数据、孤儿组件与废弃接口
- [x] 阶段 6：产出数据契约和一次性验证页
- [x] 阶段 7：测试、构建、真实数据只读验证和范围复核

## 已确认边界

- 只使用真实数据，绝不补造报告、数字或同事。
- 六维唯一来源在后端逻辑层；旧正式前端由下一棒重写。
- 对标只输出姓名、最弱维度数值、文案片段。
- AI 只预留 `source`，本轮恒为 `rule`。
- 不改 migration、不 push。
