# 前端 Skill

当作品质量取决于艺术指导、层级、克制、图像和动效，而非组件数量时，使用本 skill。

目标：交付有设计感、高级且紧跟潮流的界面。默认采用获奖级别的构图：一个核心创意、强烈的图像、精简的文案、严谨的间距，以及少量令人印象深刻的动效。

## 工作模式

在动手构建之前，先写下三件事：

- 视觉主张：用一句话描述氛围、材质和能量
- 内容规划：首屏、支撑、细节、最终 CTA
- 交互主张：2-3 个能改变页面感受的动效想法

每个区块只承担一个职责、一个主导视觉创意和一个核心信息或行动。

## 美观的默认设置

- 从构图开始，而非组件。
- 优先使用全出血首屏或全画布视觉锚点。
- 让品牌或产品名称成为最突出的文字。
- 文案要短到几秒内即可扫读完。
- 在添加装饰元素之前，先利用留白、对齐、比例、裁切和对比。
- 精简系统：最多两种字体，默认一种强调色。
- 默认不使用卡片布局。改用区块、列、分割线、列表和媒体块。
- 将首屏当作海报，而非文档。

## 落地页

默认顺序：

1. 首屏：品牌或产品、价值承诺、CTA 和一个主导视觉
2. 支撑：一个具体的功能、优惠或论据
3. 细节：氛围、工作流、产品深度或故事
4. 最终 CTA：转化、开始、访问或联系

首屏规则：

- 只用一个构图。
- 全出血图片或主导视觉平面。
- 标准全出血规则：在品牌落地页中，首屏本身必须贯穿整个边缘，不带任何继承的页面边距、框架容器或共享的最大宽度；只约束内部的文字/操作列。
- 品牌优先，标题次之，正文第三，CTA 第四。
- 默认不出现首屏卡片、数据条带、Logo 云、药丸式标签堆砌或悬浮仪表板。
- 桌面端标题控制在 2-3 行左右，移动端一眼即可读完。
- 文字列保持窄长，并锚定在图片的平静区域。
- 图片上的所有文字必须保持强烈的对比度和清晰的点击区域。

如果去掉图片后首屏依然可用，那图片太弱。如果隐藏导航后品牌消失了，那层级太弱。

视口预算：

- 如果首屏包含粘性/固定头部，该头部将占用首屏空间。头部与首屏内容总和必须在常见桌面和移动端尺寸下适配初始视口。
- 使用 `100vh`/`100svh` 首屏时，减去常驻 UI 装饰（`calc(100svh - header-height)`），或者将头部叠加在首屏上，而不是在正常流中堆叠。

## 应用

默认采用 Linear 风格的克制：

- 平静的表面层级
- 强有力的排版和间距
- 少量颜色
- 密集但可读的信息
- 最小化装饰元素
- 仅在卡片本身是交互时才使用卡片

对于应用 UI，围绕以下要素组织：

- 主工作区
- 导航
- 次要上下文或检查器
- 一个清晰的强调色用于操作或状态

避免：

- 仪表板卡片马赛克
- 每个区域都使用粗边框
- 常规产品 UI 背后的装饰性渐变
- 多种互相竞争的强调色
- 不提升扫描效率的装饰性图标

如果一个面板去掉卡片处理变成普通布局后仍不损失含义，就去掉卡片处理。

## 图像

图像必须承担叙事功能。

- 为品牌、场所、编辑页面和生活方式产品至少使用一张强有力、真实感强的图片。
- 优先使用实景摄影，而非抽象渐变或假 3D 物体。
- 选择或裁切图片时，确保有稳定的色调区域供文字使用。
- 不要使用带有内嵌标识、Logo 或与 UI 冲突的排版杂乱的图片。
- 不要生成带有内置 UI 框架、分割、卡片或面板的图片。
- 如果需要多个瞬间，使用多张图片，而非一张拼贴。

首屏需要一个真正的视觉锚点。装饰性纹理是不够的。

## 文案

- 使用产品语言，而非设计评论。
- 让标题承载含义。
- 支撑文案通常应该是一个短句。
- 删减区块之间的重复。
- 不要把提示词语言或设计评论放进 UI 中。
- 给每个区块一个职责：解释、证明、深化或转化。

如果删掉 30% 的文案能让页面更好，就继续删。

## 产品 UI 的实用文案

当作品是仪表板、应用界面、管理工具或运营工作区时，默认使用实用文案而非营销文案。

- 优先关注方向、状态和行动，而非承诺、氛围或品牌调性。
- 从工作表面本身开始：KPI、图表、筛选器、表格、状态或任务上下文。除非用户明确要求，否则不要引入首屏区块。
- 区块标题应说明该区域是什么，或用户可以在那里做什么。
- 好的例子："Selected KPIs"、"Plan status"、"Search metrics"、"Top segments"、"Last sync"。
- 除非特别要求，避免在产品表面上使用愿景式的首屏标语、隐喻、 campaign 风格语言和执行摘要横幅。
- 支撑文本应在一句话内解释范围、行为、时效性或决策价值。
- 如果一句话可以出现在首页首屏或广告中，就重写它，直到听起来像产品 UI。
- 如果一个区块不能帮助某人操作、监控或决策，就去掉它。
- 石蕊测试：如果一名操作员只扫读标题、标签和数字，他们能立即理解这个页面吗？

## 动效

使用动效来创造存在感和层级，而非噪音。

对于视觉主导的作品，至少交付 2-3 个有意的动效：

- 首屏中的一个入场序列
- 一个与滚动关联、粘性或深度效果
- 一个悬停、揭示或布局过渡，以增强 affordance（可供性感知）

在可用时优先使用 Framer Motion 来实现：

- 区块揭示
- 共享布局过渡
- 与滚动关联的不透明度、位移或缩放变化
- 粘性叙事
- 推进叙事而非仅仅填充空间的轮播
- 菜单、抽屉和模态框存在感效果

动效规则：

- 在快速录屏中能被注意到
- 在移动端流畅
- 快速且克制
- 在整个页面保持一致
- 如果仅作装饰，则移除

## 硬性规则

- 默认不使用卡片。
- 默认不使用首屏卡片。
- 当需求要求全出血时，不使用带框或居中列首屏。
- 每个区块不超过一个主导创意。
- 任何区块都不应需要大量微小 UI 元素来解释自身。
- 在品牌页面上，标题不应盖过品牌。
- 无填充文案。
- 除非文字位于平静、统一的一侧，否则不使用分屏首屏。
- 无明确理由不超过两种字体。
- 除非产品已有成熟的体系，否则不超过一种强调色。

## 拒绝这些失败

- 把通用 SaaS 卡片网格当作第一印象
- 漂亮的图片但品牌存在感薄弱
- 有力的标题但没有明确的行动
- 文字背后的繁杂图像
- 重复相同氛围陈述的区块
- 没有叙事目的的轮播
- 用堆叠卡片而非布局构成的应用 UI

## 石蕊测试

- 品牌或产品在首屏是否 unmistakable（一目了然）？
- 是否有一个强有力的视觉锚点？
- 仅扫读标题就能理解页面吗？
- 每个区块是否只有一个职责？
- 卡片真的有必要吗？
- 动效是否提升了层级或氛围？
- 如果移除所有装饰性阴影，设计是否仍然感觉高级？
