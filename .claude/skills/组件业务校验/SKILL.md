# 组件业务校验 Skill

组件不是字段容器，组件是某个角色在某个阶段完成某个动作的工具。

## 核心三问

| 问题 | 回答 |
|---|---|
| 这个组件给谁用？（角色） | |
| 这个组件在哪个阶段出现？（阶段） | |
| 这个组件的唯一动作是什么？（主动作） | |

## 字段边界检查

| 检查项 | 不通过则删除或迁移字段 |
|---|---|
| 字段来源是否属于当前阶段？ | 审核结论不能出现在提交组件 |
| 字段是否只有一种来源类型？ | 混了 → 拆分字段或拆分组件 |
| 字段是否只展示给有权看的人？ | 没权限的角色看到敏感字段 → 删除或加条件 |

7 类来源：用户提交 / 系统生成 / AI 初判 / 管理员审核 / 结果反馈 / 复测 / 资产沉淀

## 状态流转检查

| 检查项 | 不通过则拆分 |
|---|---|
| 组件是否只有一个主动作？ | 多个主动作 → 拆成多个组件 |
| 组件状态是否覆盖所有可能？ | 缺失空状态/加载状态/错误状态 → 补全 |
| 状态转换是否有明确流向？ | 死路/无下一步 → 重设计 |

## 角色 × 阶段快速矩阵

| 角色 | 提交阶段 | 审核阶段 | 反馈阶段 |
|---|---|---|---|
| 普通用户 | 填写事实 | — | 查看结论 |
| 管理者 | — | 审核确认 | 下发反馈 |
| AI | 初判建议 | — | — |

**禁止：管理员结论字段出现在提交阶段组件里。**
**禁止：AI 初判结果未经确认直接展示为最终结论。**

## 输出

只输出：

1. 核心三问答案
2. 发现的问题及修改建议（如果有）
3. 字段边界是否通过
4. 状态流转是否通过
