# DYData 实际风格需求汇总

> 本文档基于 2026/05/08 阿禅与 AI 助手的对话整理，用于指导后续前端视觉改造。

---

## 一、核心定位：不是 SaaS，是创作者工作站

DYData 的界面不是给行政人员看的「企业管理后台」，而是**抖音内容生产者的专业中控工作台**——类似 Final Cut Pro 或 Logic Pro 那种「创作者打开后就知道该干什么」的确定性界面。

**拒绝的气质**：
- 蓝色调商务 SaaS 感
- 营销式大圆角、炫酷渐变
- 装饰性色块、数据卡片马赛克
- 毛玻璃层叠、过度阴影

**追求的气质**：
- 像一张高档设计工作台——干净、有物理质感、工具各归其位
- 界面本身不说话、不营销，安静托住生产流程
- 物理质感和精确的状态反馈让创作者感到「一切尽在掌控」

---

## 二、与 Claude 风格的关系：取其骨架，补其灵魂

Claude 的风格是我们的**底色参考**，但必须做关键修正：

| Claude 的特点 | 我们的态度 | 原因 |
|--------------|-----------|------|
| 极其干净的留白 | **保留** | 高通透度是工作站的基础 |
| 高对比度黑白灰 | **保留** | 确保信息扫描效率 |
| 信息密度适中 | **保留** | 不拥挤、不空旷 |
| 圆角极其克制（近乎直角）| **修正** | 外层容器用 32-40px 大圆角，营造「工作面板」感；内层组件 12-16px 保持精确感 |
| 没有多余装饰 | **保留** | 拒绝渐变、纹理、装饰性阴影 |
| 颜色极度克制到缺乏层次 | **修正** | Claude 太注重美学而忽视视觉交互；我们需要用**融合的颜色做轻度突出**，让状态可感知、焦点可锁定 |
| 交互反馈弱 | **修正** | 必须加入物理位移反馈、状态灯、单向聚焦 |

**一句话**：Claude 给了我们「不吵」的基底，但我们要在这个基底上**加回跟人交互的层次感**。

## 三、色彩系统：仅表达状态，绝不装饰

### 基础中性色（全局骨架）

| 用途 | 色值 | Tailwind |
|------|------|----------|
| 全局背景 | `#F9F9FB` | `bg-[#F9F9FB]` |
| 容器背景 | `#FFFFFF` | `bg-white` |
| 次级背景/导轨 | `#F4F4F5` | `bg-zinc-100` |
| 主文字 | `#09090B` | `text-zinc-950` |
| 次要文字 | `#71717A` | `text-zinc-500` |
| 边框 | `#E4E4E7` | `border-zinc-200` |

**铁律**：中性色承担 95% 的界面面积。任何脱离上述色值的彩色出现，必须是为了表达**状态**。

### 功能状态色（自然矿物色）

颜色特征：低饱和度、高识别度、自然矿物色感。绝不刺眼。

| 状态 | 主色 | 背景配套 | 使用场景 |
|------|------|----------|----------|
| 执行中 / 待处理 | 琥珀黄 `#EAB308` | `#FEFCE8` | 进行中节点、待确认截图、警告提示 |
| 已通过 / 成功 | 森林绿 `#067647` | `#ECFDF3` | 提交成功、已确认截图、通过审核 |
| 需修正 / 错误 | 晚霞红 `#B42318` | `#FEF3F2` | 识别失败、必填项缺失、驳回 |
| 已提交 / 待审 | 湖泊蓝 `#444CE7` | `#EEF4FF` | 已提交待审核（使用频率低于前三者）|

**禁止**：用上述颜色做按钮默认态、卡片装饰、渐变背景、文字高亮（除非表达状态）。

### Primary Action 色

**不用蓝色，不用品牌色**。

Primary action（提交、确认、选中）统一用 **zinc-950（`#09090B`）**——一种极致深灰黑。Hover 时变为 `zinc-800`，配合 `-translate-y-[1px]` + `shadow-lg` 的物理位移。

理由：深黑是「物理按键」的颜色，比蓝色更有确定感和权威感。

## 四、形态系统：外柔内刚

### 圆角逻辑

| 层级 | 圆角 | 理由 |
|------|------|------|
| 外层工作面板 | `32px` ~ `40px` (`rounded-[2rem]`) | 像一块块拼在工作台上的面板 |
| 内层表单卡片 | `16px` (`rounded-2xl`) | 保持区域感，但不过于圆润 |
| 按钮、输入框、标签 | `12px` (`rounded-xl`) | 操作件的精确感 |
| 状态点、头像 | `999px` (`rounded-full`) | 极小的元素用全圆角作为视觉锚点 |

### 边框

- 统一 **1px 细实线**，`border-zinc-200`
- 分隔线、进度轴用 **2px 实线**，强化「物理导轨」的工业感
- 禁止半透明黑边（`border-black/10` 之类全部改为 `border-zinc-200`）

### 阴影

- 弃用大面积重阴影
- 仅在**活跃容器**上使用极轻淡的弥散投影：`shadow-sm` 或 `shadow-md`
- Hover 时阴影深化：`hover:shadow-lg` + `hover:-translate-y-[1px]`
- 禁止装饰性渐变阴影、内发光

## 五、交互体验：物理反馈 + 单向聚焦

### 1. 确定感反馈（Hover / Active）

- **不是单纯变色**，而是微小位移 + 阴影深化
- 按钮 Hover：`hover:-translate-y-[1px] hover:shadow-lg`
- 卡片 Hover：`hover:-translate-y-[1px] hover:shadow-md`
- Active（按下）：`active:translate-y-0` 回到原位，模拟按键回弹

### 2. 呼吸式状态灯

当前任务或异常点使用微弱的 `animate-pulse` 或自定义 ping 动效，模拟物理指示灯：

- 执行中：琥珀黄圆点 + 呼吸扩散
- 异常/驳回：晚霞红圆点 + 呼吸扩散
- 成功：森林绿圆点（可静态，可微弱脉冲）

### 3. 单向聚焦

- 当前生产节点**高亮锁定**（`ring-2 ring-zinc-950/8` 或状态色 ring）
- 非活跃节点进行**灰度降权**（`opacity-50` 或更淡的边框）
- 输入框 focus 时，周边非相关区域不抢戏

### 4. 输入框 = 真实文档写作感

- 纯白背景 `bg-white`
- 1px 细边框 `border-zinc-200`
- 字间距略大 `tracking-wide`
- Focus：`focus:ring-1 focus:ring-zinc-950/10 focus:border-zinc-950/30`
- 绝对禁止蓝色 focus ring

## 六、排版气质：权威与辅助的层次

### 标题

- 采用 `font-black`（特粗，900 字重）
- 强调绝对权威，如「今日生产墙」「数据报表上传」
- 字母间距收紧 `tracking-tight`

### 辅助信息

- 采用 **italic（斜体）**或**全大写微缩字符**
- 如 `SCRIPT CONTENT`、`ID: #DY-88482`、`LIVE WORKFLOW`
- 暗示「这是辅助信息，别看太久」

### 元数据 / 标签

- 全大写 + 极窄字间距 `tracking-[0.15em]`
- 字号 10px ~ 11px
- 颜色 `text-zinc-400` 或 `text-zinc-500`

---

## 七、典型组件特征速查

| 组件 | 核心特征 |
|------|----------|
| **输入框 (Editor)** | 纯白、1px 细边框、tracking-wide、focus 时 zinc-950 微 ring |
| **状态灯 (Indicator)** | 实心圆点 + 微弱扩散光晕（pulse/ping），作为视觉锚点 |
| **矩阵网格 (Matrix)** | 高对比度格子排列，用符号（钩、圆、箭头）快速传递信息 |
| **进度轴 / 导轨** | 2px 实线，已完成段用森林绿，当前节点用深黑放大 |
| **浮动操作栏** | 实体白底（非毛玻璃）、zinc-200 边框、深黑主按钮 |
| **成功/错误提示** | 用配套背景色（极浅绿/极浅红）+ 对应状态色文字，禁止大面积色块 |

---

## 八、一句话总结

> 我们要的是一个**"看起来像专业创作硬件的界面"**——界面本身不说话、不营销，只是安静地托住生产流程，用物理质感和精确的状态反馈让创作者感到「一切尽在掌控」。在 Claude 的极简骨架上，加回跟人交互的层次感和确定感。
