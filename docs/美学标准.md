## A 哲学与判断框架

### A.1 四哲学

1. **Typography as Interface**：留白 / 字距 / 字重建立层级，× 线框 × 底色
2. **Subtractive Design**：能 × 用线 × 用，能 × 用阴影 × 用
3. **Monochromatic Kinetic**：日常交互以灰阶为主，状态信号用彩色。质感靠灰度变化 + 物理位移
4. **Controlled Vibrancy**：彩色是信号 × 装饰。信号载体（状态徽章、进度条、角标）不限面积，装饰性彩面 ≤ 3%

### A.2 信息量与密度

× 尽可能少，× 堆砌，中等信息量是常态
判断：元素是否承担职责。无职责删，有职责留
反例：装饰 logo + 品牌名 + 标题 + 副标题 四连堆（抢视觉）；裸标题无辅助（冷漠）。两端都拒

**密度偏好**：阿禅偏好**舒适大方**，× 高密度 × 多信息密度 × 视觉干扰

操作准则：
- 尺寸 / 间距选择上，默认偏**宽松档**，× 紧凑档（如卡片宽度、字号、行距、padding）
- 给尺寸选项时至少给 3 档（紧凑 / 舒适 / 气派），用"舒适档"作为推荐
- 单屏元素数量宁少勿多，每增加一个元素都要能回答"它承担什么职责"
- 信息密度 = 单位面积承载的视觉信号数。同等职责下，选密度更低的方案

### A.3 主次分明（硬要求）

任何页面 / 区块必须能一句话回答"第一眼看哪里"。无主视觉 = 设计失败
副视觉（eyebrow / 副标题 / 次级按钮）存在价值是衬托主视觉，× 并列。并列 = 竞争 = 失败

### A.4 交互区与背景必须分离

手法不限：卡片 / 底色分层 / 色带 / 阴影 / 边框 / 细节装饰
选择标准：和整体调性匹配（保守场景用卡片，追求设计感用 2px 色带 / 精确间距锚点）
× 交互区与背景融为一体

### A.5 动画判断三问

1. 打断用户操作？→ × 加
2. 循环？→ 慎用（循环是注意力窃贼）
3. 用户"感觉到"但 × "注意到"？→ × 到 × 加

三问全过才加

### A.6 设计感 > 保守

同等效果选更有设计感的。保守是兜底 × 首选。"保守且正确" ≠ "好"

### A.7 规范是底线 × 天花板

本文字号 / 四色 / 禁区是底线。底线之上允许"克制的惊喜"——一条 2px 色带、不寻常的间距、精确动画节奏
克制 = 一次一处，× 到处撒

### A.8 反面共性警报

好参考差异大（Vercel / Linear / Notion 各不同）→ × 抄某一家
反面案例共性稳定，见以下信号立即警报：
- 主次不分（多元素抢焦点）
- 元素抢戏（装饰抢内容）
- 无意义装饰（× 承担职责的图形 / 色块 / 线条）
- 过度极简导致冷漠（空气撑场，× 人情味）

警报 ≠ 禁用清单，看到警报后判断是不是当前场景真的问题

### A.9 阿禅语汇速查

| 阿禅说 | 真实诉求 |
|---|---|
| 画面"糊" | 主次不分 / 交互区与背景融合 |
| "大方" | 留白充足 + 主视觉明确 + 尺寸偏宽松，× 极简 × 紧凑 |
| "高级感" | 克制 + 惊喜细节，× 炫技 |
| "惊喜感设计" | 不常见但合理的手法，× 循规蹈矩 |
| "堆了" | 元素无职责 / 抢视觉 |
| "抢戏" | 副级元素视觉权重过大，需要**降级到点缀**级别（降饱和 / 降对比 / 缩尺寸） |
| "点缀" | 感觉得到但 × 抢主视觉，第一眼 × 落在它上面 |
| "交互第一" | 使用逻辑清晰优先于画面全 |
| "中等信息量" | × 极简 × 堆砌，每个元素有职责 |
| "舒适" | 默认选宽松档，× 紧凑 |
| "高密度" | 信息过载 / 视觉干扰过多，需降密度 |

## B 色彩系统

### B.1 底色三层（× shadow 做层级）

| 层级 | 色值 | 用途 |
|---|---|---|
| L0 | `#F9F9FB` | 页面背景（暖石灰 / 纸质感） |
| L1 | `#FAFAFB` zinc-50 | 卡片、主面板 |
| L2 | `#FFFFFF` white | 焦点区、Modal、Focus 输入、激活 Tab |

层级只由底色差 + 1px `zinc-200` border 构成

### B.2 文字灰阶

| 用途 | 色值 |
|---|---|
| 主文字 | `text-zinc-800`（× `zinc-950`） |
| 次要 | `text-zinc-500` |
| 元数据 / 空态 / 单位 | `text-zinc-400` |
| 分隔 / 边框 | `zinc-200`（优先用留白） |

### B.3 四语义色（统一光源，饱和 25-60 / 亮度 55-60）

| 语义 | 色值 | HSL | 场景 |
|---|---|---|---|
| 主 CTA | `#D97757` | H15 S60 L60 | 全站唯一主按钮 |
| 待处理 | `#D99E55` | H35 S60 L59 | 授权、生成中、提醒 |
| 成功 | `#6FAA7D` | H137 S25 L55 | 在线、通过、复制成功 |
| 异常 | `#C9604D` | H8 S52 L55 | 失败、驳回、违规 |

- CTA Hover 深色 → `#C96442`
- 优先用四语义色做锚点，其他场景可用 Tailwind 语义色族

**面积规则**
- 大面积深饱和底色（覆盖 > 15% 视口）≤ 3%，如卡片大底 / 横幅 / hero
- 小面积状态信号、标签、徽章不限
- 每屏最多 1 个破格元素（爆款数据 / 唯一 CTA），必须被留白包围

**语义判断**
- 动作 → 暖橙，状态 → 矿物色
- 在线 / 复制成功 → 绿（× 橙）
- 生成中 / 待授权 → 琥珀（× 橙）
- Primary 永远只有暖橙

**数据可视化色板**
```
#D97757 主橙   #E3B15C 琥珀   #7FA97E 绿
#8AA8C7 石青   #B08BB8 雾紫
```
- 折线：主橙 + 石青（最多 2 条主线）
- 柱状 / 分布：按顺序取色
- 网格线：`zinc-200` 虚线
- 坐标轴：`text-zinc-400 text-[10px] tracking-wider`

### B.4 克制的鲜艳（3% 规则松绑）

**核心判断句**
- **克制鲜艳** = 浅色族（50/100/200）+ 深色族（600/700）文字 + 小面积 + 不闪烁
- **刺眼深色** = 饱和度满格 400/500/600 大面积底色 + 高对比文字 + 动效

**允许的克制鲜艳**

| 场景 | 上色方式 | 举例 |
|---|---|---|
| 信息色块（图例 / 芯片 / 状态 chip） | `bg-{hue}-50 text-{hue}-700` + 浅边 | 日历图例 `bg-emerald-50 text-emerald-700` |
| 数据状态格（日历单日、表格单元、标签） | `bg-{hue}-50 border-{hue}-200 text-{hue}-600-700` | 漏交 `bg-rose-50 text-rose-600 ring-rose-100` |
| 进度条已完成段 | 主色 `#D97757` 或 `#6FAA7D` 实色 | 完播进度条 |
| 微图标底托（kbd / 数字角标） | `bg-{hue}-50/70` + 柔边 | 红点角标 |
| 角色徽章 | 彩色浅族 chip | 见 E.4 |
| 关键数值 hover 微上色 | `hover:text-[#D97757]` | 让数据活起来 |
| 信息性图标上色 | 类别 / 状态 / 分组图标用四色或浅族 600 | 类别标识、状态图标 |
| 悬浮层柔光阴影 | `shadow-[0_8px_24px_-12px_rgba(15,23,42,0.12)]` | 高级 dropdown / tooltip |

**仍然禁止的刺眼深色**
- × 大面积深饱和底色（头图 / 卡片背景 / 主画布）
- × 浅彩色底面积超过单视口 15%（大横幅、全宽 banner）
- × 深饱和底色 + 动效（闪烁 / 渐变移动 / 彩虹轮播）
- × 单视口同时 ≥ 4 个色相

**允许的色族扩展**

四语义色之外，信息色块场景允许：

| 语义 | Tailwind 色族 | 用途 |
|---|---|---|
| 成功 / 已完成 | `emerald-50/100/200` + `emerald-600/700` | 已交、完成、通过 |
| 提醒 / 进行中 | `amber-50/100/200` + `amber-600/700` | 请假、待处理、待确认 |
| 危险 / 漏交 | `rose-50/100/200` + `rose-600/700` | 漏交、未交、违规 |
| 信息 / 链接 | `sky-50/100/200` + `sky-600/700` | 链接、提示、次要操作 |

硬规：这四族优先用于 50/100/200 底 + 600/700 字。信号载体（状态徽章、进度条）可用 400/500 实底 + 白字，但须小面积

**锚点 vs 点缀**
- 主 CTA / 状态锚点 / 几何体语言（2px 导轨、状态点） → 用四色
- 信息色块 / 日历格 / 图例芯片 / 角色徽章 → 用 Tailwind 浅族
- 两者可共存：状态点用 `#C9604D` 实色 + 日历格底 `rose-50`，同色相不同饱和度层级

**落地范例**
```tsx
// 克制鲜艳（正）
<span className="inline-flex items-center gap-2 rounded-full bg-emerald-50 px-3 py-1 text-emerald-700">
  <span className="size-2 rounded-full bg-emerald-500" />
  已交
</span>

// 刺眼深色（禁）
<div className="bg-emerald-500 text-white p-8">已交</div>
```

**信号载体 vs 信息色块**
- 信号载体：需要用户第一眼识别状态（成功/失败/警告），可用四色实底 + 白字，小面积，不限于 3%
- 信息色块：展示分类/来源/属性，用浅族底 + 深族字
- 判断：去掉颜色会误操作 → 信号载体；去掉颜色只是信息减少 → 信息色块

## C 形态与节奏

### C.1 圆角

| 层级 | 圆角 | Tailwind |
|---|---|---|
| 对话区 / 主面板 | 16px | `rounded-2xl` × 更大 |
| 输入框外壳 | 20px | `rounded-[20px]` |
| 用户消息气泡 | 14px + 右上 4px | `rounded-[14px] rounded-tr-[4px]` |
| 内层卡片 / 数据块 | 12px | `rounded-xl` |
| 按钮 / 标签 | 8-10px | `rounded-lg` / `rounded-[10px]` |
| 状态点 / 头像 | 满圆 | `rounded-full` 仅正圆 |

悬浮层（Dropdown / Modal）：`border border-zinc-200` + `shadow-sm`，× 重模糊阴影

### C.2 边框温度

| 温度 | 色值 | 用途 |
|---|---|---|
| 极冷 | `zinc-200` | 分隔线、卡片边界、表格横线 |
| 微温 | `zinc-300` | 可交互元素默认边框 |
| 暖橙 | `#D97757` | 激活 / 选中持久态 1px 轮廓 |

Focus Ring：`focus:ring-1 focus:ring-zinc-950/5`（× 暖橙做临时聚焦）

### C.3 字号 7 档

| 档位 | 字号 / 行高 | 字重 | 字距 | 用途 |
|---|---|---|---|---|
| Display | `text-[32px] leading-[1.25]` | `font-semibold` | `tracking-tight` | 核心数据大数、仪表盘顶格指标 |
| Section | `text-[24px] leading-[1.33]` | `font-semibold` | `tracking-tight` | 模块标题、页面二级标题 |
| Title | `text-[18px] leading-[1.44]` | `font-medium` | 默认 | 弹窗标题、小模块标题 |
| Body-L | `text-[14px] leading-[1.7]` | 默认 | 默认 | 中文密集列表、长段正文 |
| Body | `text-[13px] leading-[1.7]` | 默认 | `tracking-[0.005em]` | 默认正文（英文 / 数字密集） |
| Label | `text-[13px] leading-[1.23]` | `font-medium` | 默认 | 表头、导航项、表单标签 |
| Caption | `text-[12px] leading-[1.33]` | 默认 | 默认 | 脚注、辅助说明 |
| Eyebrow | `text-[10px] leading-[1.6]` | `font-medium` | `uppercase tracking-[0.25em]` | 小眉题、分组标签、状态 chip |

硬规：
- × `text-[16px]` 作为正文默认
- × `font-bold` / `font-black`，最高 `font-semibold`
- × 同一视觉块内跨 3 档以上字号
- 数字 / ID / 图表数值全站强制 `tabular-nums font-mono`

Display 档边界：
- 每屏最多 1 处（仪表盘核心大数、"爆款数据"）
- 必须 ≥ 32px 留白包围
- × 弹窗内 × 列表项内

### C.4 间距 6 档

| 档位 | 像素 | Tailwind | 用途 |
|---|---|---|---|
| XS | 4px | `gap-1` / `p-1` / `space-y-1` | icon 与文字、chip 内部 |
| S | 8px | `gap-2` / `p-2` / `space-y-2` | 相邻表单字段、小标签内部 |
| M | 16px | `gap-4` / `p-4` / `space-y-4` | 卡片内部、模块次级分组 |
| L | 24px | `gap-6` / `p-6` / `space-y-6` | 卡片 padding、主分组间 |
| XL | 32px | `gap-8` / `p-8` / `space-y-8` | 模块间、Section 之间 |
| 2XL | 40px | `gap-10` / `p-10` / `space-y-10` | 页面级主分区、顶级 hero |

硬规：
- × 非档位值：`p-3` / `p-5` / `p-7` / `gap-3` / `space-y-5`
- 存量项目可保留 `p-3`，新增代码严格按 6 档
- × "差不多"就用，必须匹配语义档位

特例白名单：
- 导航项 `px-3 py-2`
- 输入框 `h-10 px-3` / `h-11 px-4`（高度优先于 padding 档）
- shadcn 组件内部默认间距保留

## D 交互与动效

### D.1 输入框（Soft Fill）

- 默认：`bg-zinc-100/70 border-transparent text-zinc-800`
- Focus：`bg-white border-zinc-200 shadow-sm focus:ring-1 focus:ring-zinc-950/5`

### D.2 动效曲线（× `transition-all`）

```css
transition: transform 150ms cubic-bezier(0.4, 0, 0.2, 1),
            background-color 150ms cubic-bezier(0.4, 0, 0.2, 1),
            box-shadow 150ms cubic-bezier(0.4, 0, 0.2, 1);
```

- Hover 150ms / Active 回弹 80ms
- × 弹簧过冲 overshoot

### D.3 Hover 去色化

列表 / 次级按钮 / 导航 Tab 的 Hover 严禁彩色：
- `hover:bg-zinc-100` 或 `hover:bg-white`（× 半透明 `bg-zinc-100/50`）
- 配合 `-translate-y-[1px]` + `shadow-sm`
- `active:translate-y-0` 物理回弹

### D.4 Hover 延迟（仅列表 / 导航）

```css
transition: background-color 150ms ease 50ms;
```
按钮 / CTA / 输入框 × 加延迟

### D.5 主 CTA Hover 弥散阴影

```css
box-shadow: 0 2px 8px rgba(217, 119, 87, 0.03);
```
透明度 ≤ 3%，次级按钮完全 × 阴影

## E 组件与模板

### E.1 排版细节

**字重字距**

| 类型 | 规则 |
|---|---|
| 标题 / 核心数据 | `font-semibold tracking-tight`（× `font-bold` / `font-black`） |
| 元数据 / 分组标签 | `text-[10px] uppercase tracking-[0.25em] text-zinc-400 font-medium` |
| 数字 / ID / 图表 | `tabular-nums font-mono` 全站强制 |
| 正文 | `text-[13px] leading-[1.7] tracking-[0.005em] text-zinc-800` |

**数字与单位分层**
- 数值：`zinc-800 font-semibold text-[20px] tabular-nums`
- 单位（`%` `¥` `万` `件`）：`zinc-400 font-medium text-[14px]`

### E.2 空状态 / 加载

空状态：
- `text-[13px] text-zinc-400` 诗意文案
- × 彩色插画 × 彩色图标

加载（静默）：
- < 200ms：× loading，直出
- ≥ 200ms：骨架屏（`bg-zinc-100` 矩形，结构与最终一致），200ms 淡入替换
- × spinner

### E.3 列表 / 表格

- 斑马线：`bg-white` + `bg-zinc-50/50` 交替，或仅 Hover 当前行 `bg-zinc-100`，× 边框切分
- 卡片底部元数据：`mt-6 pt-6` 留白分隔，× border-top

### E.4 典型组件

- **数据卡片**：`bg-white border-zinc-200 rounded-xl`，数值 `font-semibold tabular-nums`，增长箭头用语义色
- **Tabs**：默认 `text-zinc-500` 无底色；激活 `bg-white shadow-sm text-zinc-800 rounded-lg`（× 黑填充，E.7）
- **违规话术列表**：左侧 2px 实色 `#C9604D` 导轨，hover 整行微浮
- **消息气泡**：用户右对齐 `rounded-2xl rounded-tr-md bg-zinc-950 text-white`；AI 左排无背景或 `bg-zinc-50 rounded-xl`
- **欢迎页**：hero + 2x2 / 横排 shortcut 卡片，hover 时左上点 `zinc-300` → `#6FAA7D`
- **侧边栏**：同 L0 色，1px 边框区隔，单行 + 左状态点 + 右时间
- **可折叠配置栏**：240px 宽长，齿轮切换，宽度 transition，保持定态
- **确认卡**：左 2px 琥珀 `#D99E55` 导轨 + 右上浮动点 + 暖橙主按钮
- **图标**：Lucide `stroke-[1.5]`，与文字 `gap-1.5` 对齐
- **分页器**：当前页 `bg-white border border-[#D97757]/40 text-[#D97757]`（暖橙描边 × 黑填充）；非当前 `bg-white border-zinc-200 text-zinc-600 hover:bg-zinc-50`
- **角色徽章**：创始人 `bg-[#D97757]/10 text-[#D97757] border border-[#D97757]/20` / 管理员 `bg-amber-50 text-amber-700 border-amber-200` / 组长 `bg-emerald-50 text-emerald-700 border-emerald-200` / 成员 `bg-zinc-50 text-zinc-600 border-zinc-200`
- **进度条**：业绩主线 `bg-[#D97757]` / 中性流程 `bg-zinc-400` / 对比场景 "我" `bg-zinc-300` + "标杆" `bg-[#D97757]`

### E.5 状态点四态

| 态 | 尺寸 | 填充 | 外圈 | 动效 | 场景 |
|---|---|---|---|---|---|
| Critical | 8px `h-2 w-2` | `bg-[#C9604D]` | `ring-1 ring-white` | `animate-pulse` 2s | 失败、驳回、严重告警 |
| Active | 8px `h-2 w-2` | `bg-[#D97757]` | `ring-1 ring-white` | 静态 | 当前步、进行中、待处理主动作 |
| Done | 8px `h-2 w-2` | `bg-[#6FAA7D]` | `ring-1 ring-white` | 静态 | 完成、通过、在线 |
| Idle | 6px `h-1.5 w-1.5` | `bg-zinc-200` | 无外圈 | 静态 | 未开始、无状态、占位 |

并列降权：同一视觉块内 Critical + Active 共存 → Active 降权到 6px 静态，保持 Critical 唯一"跳出点"

```tsx
<span className="h-2 w-2 rounded-full bg-[#C9604D] ring-1 ring-white animate-pulse" /> // Critical
<span className="h-2 w-2 rounded-full bg-[#D97757] ring-1 ring-white" /> // Active
<span className="h-2 w-2 rounded-full bg-[#6FAA7D] ring-1 ring-white" /> // Done
<span className="h-1.5 w-1.5 rounded-full bg-zinc-200" /> // Idle
```

硬规：
- × 状态点多色渐变
- × 状态点 ≥ 10px
- 1px `ring-white` 外圈是四态视觉锚点，不可省

### E.6 左 2px 彩色导轨

标准配方：
```
border border-zinc-200 border-l-[2px] border-l-[#语义色]
```

语义映射：
- Critical 告警 / 违规 / 异常：`border-l-[#C9604D]`
- Pending 待处理 / 确认卡 / 提醒：`border-l-[#D99E55]`
- Done 已完成 / 通过：`border-l-[#6FAA7D]`
- Primary 当前激活：`border-l-[#D97757]`

分组并列导轨（多个子分组同区域并列时）：
- 主要分组（核心 / Primary）：`bg-[#D97757]/70`
- 次级分组（互动 / 过程）：`bg-sky-400/70`
- 收尾分组（完播 / 转化 / 结果）：`bg-emerald-400/70`
- 长度规则：`top-[5%] bottom-[5%]` 上下各缩 5%（× 顶格，避免压迫）
- 饱和度规则：多组并列时统一 `/70` 降一档（× 抢主视觉，单独使用时可用 100% 实色）
- 色相选择：同屏 ≤ 3 色相 + Primary 暖橙必须是"主要分组"位

必须有 2px 导轨的场景：
- 异常告警条（如 `alert-center` 外层容器）
- 违规话术列表项
- 确认卡 / 豁免申请卡
- 分组区（核心数据 / 互动 / 完播）标题区

禁区：
- × 2px 导轨 + 彩色背景面同时用（补丁感）
- × 导轨渐变（必须实色）
- × 同一容器 > 1 条 2px 导轨

### E.7 激活态黑墨 × 塑料

深色实心激活态（`bg-zinc-800 / bg-zinc-900`）：
- × `shadow` / `shadow-sm` / `shadow-md`（阴影 = 塑料感）
- × `border border-white` 或任何亮色描边
- 允许 `border border-zinc-900` 同色加深锚边

黑色锚点唯一性：
- 合法黑锚点统一 `bg-zinc-900`（× 950。例外：用户消息气泡 / 代码块 `<pre>` / 模态遮罩）
- Logo 方块 / 头像圆保留 `bg-zinc-800`
- 每屏至多 1 个 `bg-zinc-900` 主按钮，必须是终局动作（保存 / 提交 / 通过 / 批准）
- 角色徽章 / 权限标签 / 状态 chip / 分页器 / 计数徽章 × 黑底 → 浅族 chip 或 outline
- Tab 激活态统一 `bg-white shadow-sm text-zinc-800 rounded-lg`（× 黑填充；layoutId 动画滑块例外）
- 弹窗内至多 1 个黑按钮，其他一律 outline

主 CTA 例外：`bg-[#D97757]` Hover 弥散阴影 ≤ 3%（D.5）继续有效

## F 禁区速查

**色彩**
- 蓝色 Focus Ring / 大块深饱和彩底 → 微灰底 + 状态点
- 直出 `emerald-600` / `amber-500` / `red-600` 做锚点 → 替换为四语义色
- 装饰性彩面总面积 > 3% → 压缩
- 状态导轨渐变 → 改实色
- `bg-zinc-800/900` 当次级身份标签默认底（角色徽章 / 权限标签 / 状态 chip 全黑）→ 改浅族 chip
- 同屏 ≥ 2 个 `bg-zinc-900` 主按钮 → 只留 1 个终局锚点，其余 outline

**字**
- `font-bold` / `font-black` → `font-semibold`
- 正文 16px → 英文 13px / 中文 14px
- 数字没加 `tabular-nums` → 补

**灰阶**
- 纯黑 `zinc-950` 文字 → `zinc-800`
- 纯黑大面积背景 → `zinc-900` 以上

**交互**
- 无 `active:translate-y-0` → 补
- `transition-all` → 明确列属性
- 列表 hover 半透明 → 实色
- 弹簧过冲 → 删
- 次级按钮 hover 彩色 → 去色

**减法**
- 卡片底部 border-top → 删，换 `mt-6 pt-6`
- spinner → 骨架屏 + 静默加载
- 空状态彩色插画 → `zinc-400` 文案
