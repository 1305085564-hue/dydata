## 1. 弹性区-场景分化（详细覆盖规则）

### 1.1 U1 提交（今日提交 / 日报填写 / 视频上传 / 案例提交）

**心智**：快速完成，不要让我思考。

**覆盖规则**：
- **容器**：允许使用轻量白底卡片（`bg-white rounded-xl border-stone-200 p-6`）作为工作区容器，划定填写边界。禁止表单字段直接飘在 `stone-100` 背景上无边界。
- **密度**：中高密度
- **表单范式**：属性面板式——高频字段裸露，低频字段折叠。同组字段间距 8px，组间间距 24px，组标题 Subtitle 16px semibold。
- **空状态**：引导型——Lucide 图标 + 「今日暂无提交」 + 暖橙 CTA「创建日报」
- **Hover**：暴露模式——字段提示、操作提示在 hover 时显现
- **特殊**：OCR 识别期间表单可并行填写，拒绝教科书式禁用交互

**UX 方向**：用户从进入到完成提交，操作路径必须短、校验 inline、成功反馈在上下文内

### 1.2 U2 查阅（个人数据 / 历史记录 / 案例库 / 视频库）

**心智**：找到我要的，快速扫描。

**覆盖规则**：
- **容器**：行容器（列表），主体通透无卡片，靠留白和细线分隔。禁止每行独立成卡片。
- **密度**：高密度
- **列表行高**：48px
- **同状态行间距**：2px（几乎无缝，暗示同组）
- **不同状态组间距**：16px（暗示分组切换）
- **Hover**：高亮模式——`bg-stone-200`，快速响应（transition 100ms）
- **空状态**：中性型——「未找到相关结果」 + 建议关键词 + 清除筛选按钮

**UX 方向**：搜索即时响应、行点击区域覆盖整行、返回保持状态

### 1.3 U4 分析（增长雷达 / 团队对比 / 趋势图表 / 个人看板）

**心智**：看懂趋势，做出判断。

**覆盖规则**：
- **容器**：卡片网格（图表组），每个图表 / 指标组允许独立轻量卡片。同组数据指标共用同一容器，内部留白 + 细线分隔，禁止每个指标独立卡片。
- **密度**：中密度
- **Hover**：高亮模式——图表下钻提示、数据点高亮
- **空状态**：数据型——「暂无数据」 + 时间范围提示 / 切换维度提示
- **异常优先**：异常数据模块用矿物红 / 琥珀高亮，正常模块用灰阶退让
- **视觉重心**：核心数据大数（Display 32px bold）或异常警告模块，每屏唯一

**UX 方向**：图表渐进渲染、异常一眼可辨、下钻保留上下文

### 1.4 M1 审核（视频审核 / 违规案例 / 日报校验）

**心智**：逐条审阅，快速决策。

**覆盖规则**：
- **容器**：行容器（审核列表），主体通透无卡片
- **密度**：中高密度（30 条 / 天，行高 48–56px 含缩略图）
- **操作暴露**：
  - 通过按钮：常驻，使用矿物绿 `#6FAA7D`（实心白字或 outline）。禁用暖橙（避免与页面主 CTA 冲突）。
  - 拒绝按钮：默认隐藏，hover 时暴露，使用矿物红 `#C9604D`。
- **Hover**：高亮 + 暴露——`bg-stone-200` + 行内次要操作显现
- **批量**：选中后顶部出现批量操作栏（适用于多选场景）
- **空状态**：队列型——「审核队列已清空」 + 次按钮「刷新列表」
- **禁止**：通过和拒绝按钮并列常驻；禁止每行使用暖橙主按钮

**UX 方向**：审核决策行内完成、通过即时反馈、拒绝必须输入理由、批量操作有确认

### 1.5 M2 配置（个人设置 / 团队设置 / AI 渠道 / 权限）

**心智**：精确设置，一次搞定。

**覆盖规则**：
- **容器**：属性面板（左导航 + 右表单）或单卡片表单
- **密度**：中密度
- **表单范式**：传统表单，字段分组。组内字段间距 8px，组间间距 24px，组标题 Subtitle 16px semibold。
- **Hover**：暴露模式——设置项提示、帮助文案 hover 显现
- **空状态**：设置型——首次配置引导，分步骤指示
- **输入框**：遵循主文件锁死区 1.2 输入框边界原则。配置页通常使用属性面板或单卡片表单，输入框在白色容器内时允许底色差方案或边框方案择优

**UX 方向**：变更即时保存或有明确反馈、未保存离开有提示、危险操作有二次确认

### 1.6 M3 库管（选题库 / 案例库 / 视频资产库 / 账号库）

**心智**：检索、整理、归类。

**覆盖规则**：
- **容器**：行容器（列表视图）/ 轻量卡片（网格视图）。禁止卡片套卡片。
- **密度**：中高密度
- **Hover**：高亮 + 暴露——`bg-stone-200` + 行级管理操作显现
- **空状态**：检索型——搜索框 + 创建按钮 + 热门标签建议
- **搜索**：顶部常驻搜索栏，支持实时筛选

**UX 方向**：搜索即时响应、创建 2 步内完成、删除可撤销

### 1.7 M4 总览（Cockpit / 数据看板 / 团队绩效）

**心智**：掌控全局，发现异常。

**覆盖规则**：
- **容器**：仪表盘网格，模块级轻量卡片
- **密度**：中密度
- **异常优先**：异常数据模块用矿物红 / 琥珀高亮，正常模块用灰阶退让
- **视觉重心**：核心数据大数（Display 32px bold）或异常警告模块，每屏唯一
- **空状态**：总览型——全零状态 + 引导创建 + 时间范围提示
- **禁止**：多个 Display 数字同时抢占视觉重心

**UX 方向**：异常排在视觉优先级最高、模块提供下钻入口、时间范围全局联动

## 2. 参数速查表

### 2.1 色值

| 语义 | 色值 | 用途 |
|------|------|------|
| 页面背景 | `stone-100` `#F5F5F4` | L0，暖灰 |
| 卡片 / 输入框默认 | `white` `#FFFFFF` | L1，在暖灰上自然浮现 |
| 卡片内输入框 | `stone-100` `#F5F5F4` | L1.5，与页面背景同色，挖洞/穿透效果 |
| 卡片内次级区域 | `stone-100/50` | L1.5，高密度表单组 |
| 焦点容器 | `white` | L2，Modal、激活态输入框 |
| 标题 | `stone-950` | 近黑，唯一深锚点 |
| 正文 | `stone-800` | 深灰，保证可读 |
| 信息型辅助 | `stone-600` | 描述、指标名、副标题 |
| 引导型辅助 | `stone-500` | 表单字段标签、操作提示、分组标题 |
| 元数据 / 空态 | `stone-400` | 时间戳、占位符、单位 |
| 主 CTA | `#D97757` | 暖橙，行动召唤 |
| 待处理 / 进行中 | `#D99E55` | 琥珀，提醒 |
| 成功 / 通过 | `#6FAA7D` | 矿物绿，正常 |
| 异常 / 错误 | `#C9604D` | 矿物红，失败 |
| 选中 / 激活 | `#8AA8C7` | 石青，当前位置 |

### 2.2 间距

| 档位 | 像素 | 命名 | 用途 |
|------|------|------|------|
| XS | 4px | 1 | 图标与文字间隙 |
| S | 8px | 2 | 亲密元素（标题与副标题） |
| M | 16px | 4 | 同模块内不同组 |
| L | 24px | 6 | 默认模块间距 |
| XL | 32px | 8 | 需要呼吸感时 |
| 2XL | 40px | 10 | 页面级模块间距 |

### 2.3 圆角

| 元素 | 圆角 | 说明 |
|------|------|------|
| 外层容器 | `rounded-2xl` (16px) | 页面级主面板 |
| 内层卡片 / 大按钮 | `rounded-xl` (12px) | 数据块、模块卡片、高度≥40px按钮 |
| 标准按钮 | `rounded-lg` (8px) | 高度 28~36px |
| 小按钮 | `rounded-md` (6px) | 高度 24px |
| 标签 / 徽章 / 头像 / 分段选项 | `rounded-full` | 胶囊 / 正圆 |

### 2.4 阴影与边框

> Focus 态完整参数（含 ring）详见主文件锁死区 1.2。

| 场景 | 阴影 | 边框 |
|------|------|------|
| 悬浮层（Dropdown/Tooltip/Popover） | `shadow-lg` + `backdrop-blur` | `border-stone-200/50` |
| Modal / 命令面板 | `shadow-xl` + 遮罩 | `border-stone-200` |
| 独立卡片 hover | `shadow-sm` | — |
| 白卡默认态 | `shadow-[0_1px_2px_rgba(0,0,0,0.04)]` | `border-stone-200` |
| 表格行、列表行、表单控件 | 无 | `border-stone-200`（仅需要时） |
| 输入框默认 | 无 | `border-stone-200` |
| 输入框 Focus | `shadow-sm` | `border-stone-400` |

### 2.5 背景加深后的配套升级（stone-100 背景专用）

以下规则因页面背景从 `stone-50` 加深为 `stone-100` 而同步调整，确保对比度不丢失：

| 元素 | 规则 | 原因 |
|------|------|------|
| **页面级分隔线** | L0 上统一使用 `border-stone-300` | 原 `border-stone-200` 在 `stone-100` 上几乎隐形 |
| **页面级次要按钮** | L0 上禁止裸用透明/浅色，默认 `bg-white` 或带边框 | 防止与 `stone-100` 背景融为一体 |
| **页面级次要按钮 Hover** | `hover:bg-stone-200/40` | 避免亮度倒挂 |
| **分段控制器底色** | `bg-stone-200/60` | 与背景协调，未选中项可见 |
| **分段控制器选中态** | `bg-white` | 选中项浮起 |
| **骨架屏（Skeleton）** | `bg-stone-200/60` | 防止在 `stone-100` 背景中隐形 |
| **禁用状态** | 底色升级到 `stone-200` | 原 `stone-300` 在 `stone-100` 上辨识度不足 |
| **Placeholder** | 升级到 `stone-500` | 原 `stone-400` 在 `stone-100` 输入框上对比度不足 |
| **空状态图标（L0）** | 最低 `stone-500` 或加圆形衬底 | 原 `stone-300/400` 在 `stone-100` 上发虚 |

### 2.6 状态标签（墨水退晕）

- 状态徽章 / Chip **默认**保持视觉静默：极淡彩色面状底色 + 同色微浓文字
- 透明度按辨识需要调节：`0.04 ~ 0.12`
- **例外**：当状态本身是页面核心信息时（如审核页「待审核」数量徽章），允许适度提高饱和度或加细边框
- 禁止：饱和标签伪装成可点击按钮，抢走主体重心

### 2.7 渐进式操作显影

- 仅用于行级次要操作（编辑、删除等图标按钮）
- 默认 `opacity-40`，Hover `opacity-100`
- 主 CTA 和关键操作绝不使用此模式
- 必须保证 `:focus-visible` 可访问性
- 行内至少保留一个常显视觉提示（如 `···`）告知此处有操作

## 3. 反模式清单（自检参考）

以下情况如果出现在页面中，说明设计需要重新审视：

| 反模式 | 表现 | 为什么需要重新审视 |
|--------|------|---------------------|
| 保护色 | 标题用 `stone-500` 或更浅，与正文同色同重 | 用户无法定位层级，扫描时间延长 |
| 层级扁平 | 同一页面所有文字 `font-normal text-stone-500/600` | 扫一眼分不清主次，操作入口被淹没 |
| 操作隐形 | 主按钮用 `variant="ghost"` 或透明度 < 0.5 | 用户找不到下一步，操作路径断裂 |
| 全灰综合征 | 页面除暖橙 CTA 外全部是 `stone-300~500` | 缺乏视觉锚点，信息变噪音，扫描效率归零 |
| 悬浮层无纵深 | 弹窗 / 下拉无阴影，与底层平齐 | 用户分不清当前操作层，容易误触底层 |
| 假克制 | 为"安静"把功能性颜色弱化 | 去掉路标等于去掉辨识能力，操作错误率上升 |
| 密度失控 | 紧密相关元素间距过大，不相关元素紧贴 | 违反格式塔亲密性，用户误判元素关系 |
| 视线撕裂 | 一个页面同时出现通透画布 + 高密度卡片 + 属性面板 | 容器策略不统一，心智模型混乱，用户不知道当前该做什么 |

## 4. 动效参考

### 4.1 分层策略

| 场景 | 动效 | 参数 |
|------|------|------|
| 日常交互（Hover/Focus） | Ease | `150ms cubic-bezier(0.4,0,0.2,1)` |
| 卡片流转（拖动、排序） | Spring | `stiffness: 300, damping: 24` |
| 页面入场 | Stagger 淡入 | `opacity 0→1, translateY 8→0, 150ms` |
| 按压反馈 | 微缩回弹 | `active:scale-[0.98]` + `150ms ease` |

允许动画属性：`background-color` / `color` / `border-color` / `box-shadow` / `transform` / `opacity`。不动 layout 属性。`prefers-reduced-motion` 下关闭非必要过渡。

### 4.2 卡片流转三件套

- 共享布局：`<motion.div layout layoutId="{域}-{id}">`
- 入场 spring：`stiffness: 300, damping: 24`；退场 `opacity: 0, scale: 0.96, duration: 0.15`
- 撑开式操作条：`grid-template-rows: 0fr → 1fr` 过渡
