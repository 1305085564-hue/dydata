## 一、业务本质（重新定义）

这个页面的核心用户是**一线员工**，不是管理员。

### 用户进来干三件事（按权重）

| 权重 | 需求 | 用户动作 | 结果 |
|---|---|---|---|
| 40% | 上传话术 | 主动提交自己用的话术+截图 | 话术进入库，等审核 |
| 40% | 违规规避 | 看哪些话术/画面容易被罚，避坑 | 知道什么不能说/不能放 |
| 20% | 高转化话术 | 找转化率高的话术，复制来用 | 拿到能用的话术文案 |

### 话术的两个来源

| 来源 | 入口 | 说明 |
|---|---|---|
| 主动上传 | 话术案例库的「上传话术」入口 | 员工主动提交话术+截图，标记违规或转化 |
| 数据上传顺带采集 | 今日工作台 → 数据上传 → 导粉数>0 触发 | 员工填播放量+导粉数，系统要求填导粉话术，高转化的自动收录 |

### 转化率定义

**转化率 = 导粉数 ÷ 播放量**

来自数据上传页面，员工填写播放量和导粉数时系统自动计算。

### 管理员角色（轻量兜底）

| 权重 | 动作 | 说明 |
|---|---|---|
| 65% | 无需操作 | 大部分话术自动入库，管理员不需要逐条审 |
| 20% | 审批是否入库 | 筛掉低质量/无意义的提交 |
| 15% | 给点评意见 | 小概率对违规内容补充反馈，用户在数据上传页看到 |

**原则：管理越轻越好，核心价值在用户侧。**

## 二、页面职责（最终定义）

### 只有 2 个视角，不是 3 个

| 视角 | 谁用 | 核心动作 |
|---|---|---|
| 用户视角 | 全体员工 | 上传话术 / 看排行榜 / 点进去看详情+截图 / 复制话术 |
| 管理视角 | 有权限的管理员 | 审批入库 / 给点评 / 筛低质量 |

### 用户视角页面结构

```
话术案例库主页 (/violations)
├── 顶部：上传话术入口（醒目按钮）
├── 中部：两个排行榜并列
│   ├── 左：高转化话术 TOP（转化率排序）
│   └── 右：高违规话术 TOP（违规率/次数排序）
├── 下部：全部话术列表（两列卡片，可筛选）
└── 交互：点击任何话术 → 弹窗/抽屉看详情
```

### 话术详情（弹窗/抽屉，不跳页）

用户点击排行榜或卡片里的话术后看到：

| 内容 | 说明 |
|---|---|
| 话术原文 | 完整文案，支持一键复制 |
| 截图画面 | 短视频截图，支持多图翻页放大 |
| 使用次数 | 多少人用过这条话术 |
| 多号通过率 | 这条话术在 N 个账号测试，X 个通过 Y 个违规（核心指标） |
| 多次使用的转化率 | 每次使用的播放量/导粉数明细 |
| 平均转化率 | 所有使用记录的加权平均 |
| 违规记录（如有） | 被平台处罚过几次，什么类型（限流/封号/警告） |
| 管理员点评（如有） | 管理员给的补充意见 |
| 置信度标签 | 高/中/低置信（基于播放量 5万/2.5万/1.5万 阈值） |

### 管理视角页面结构

```
管理工作台 (/violations?view=manage)
├── 左栏：待处理（新提交的话术，需要审批）
├── 右栏：已处理（近期审批记录）
└── 操作：通过/驳回/写点评
```

## 三、违规规避的逻辑（难点说明）

用户需求：知道哪些话术/画面容易被抖音处罚，提前避坑。

### 当前能做的（V1）

- 展示「违规率排行榜」— 哪些话术被罚次数最多
- 点进去看具体：这条话术被罚了几次、什么类型的处罚（限流/封号/警告）
- 截图对比：违规的画面长什么样

### 需要 AI 做的（V2）

- 自动归类：把多条违规话术聚类，提炼共性（比如「承诺收益类」整体违规率 60%）
- 输出避坑总结：「抖音平台对 XX 类表述敏感度高，建议改为 YY」
- 这个 V1 不做，但数据结构要预留

### 为什么难

员工不会主动标记「我用了哪条话术」，所以「一条话术被多少人用过」这个数据：
- 高转化话术：来自数据上传页的导粉话术字段，系统可以做文本匹配/归类
- 违规话术：来自主动上传，员工被罚了才会来提交

## 五、交互路径（窗口设计）

### 用户视角

```
主页 → 点「上传话术」按钮 → /violations/submit（独立页面，因为是多步表单）
主页 → 点排行榜里的话术 → 抽屉弹出完整详情（不跳页）
主页 → 点卡片列表里的话术 → 同上，抽屉弹出
抽屉里 → 点截图 → Lightbox 弹窗（多图翻页）
抽屉里 → 点「复制话术」→ 复制到剪贴板 + toast 提示
```

### 管理视角

```
主页 → 切到「管理」tab → 显示待处理/已处理双栏
待处理列表 → 点条目 → 抽屉弹出详情
抽屉里 → 点「审批」→ Dialog 弹窗（通过/驳回/写点评）
待处理列表 → 勾选多条 → 底部浮动栏批量操作
```

### 关键原则

- **最多 2 层**：页面 → 抽屉 → 弹窗，不再多
- **不跳页看详情**：所有详情都在抽屉里完成
- **返回不丢状态**：关闭抽屉回到原位，筛选/滚动位置不变
- `/violations/[id]` 保留但只作为分享链接（别人发给你一个话术链接能打开）

## 七、已确认事项

1. ✓ 转化率 = 导粉数 ÷ 播放量
2. ✓ 管理员角色轻量化（65% 无需操作）
3. ✓ 高转化话术收录门槛：播放量 ≥ 1.5 万（低置信起步），2.5 万中置信，5 万高置信
4. ✓ 违规排行榜：V1 按原始数据展示（被罚次数），V2 靠 AI 做开放性洞察总结违规共性
5. ✓ 话术归类：V1 各自独立展示，V2 AI 按骨架聚类（同模板 = 同类型，员工各自改 30%-50% 只是变体）
