# 违规话术系统 — 后续迭代计划

> 基于 2026-05-06 会议需求文档 vs 现有实现的 gap 分析，2026-05-26 制定

## 当前状态

系统核心闭环已通：提交→审核→公示→多号测试→通过率。完成度约 70%。

## 本次迭代（V1.1）— 补闭环

### 1. Dashboard 风险速览区（/violations 首页顶部）

- 位置：现有"找话术"页面顶部，折叠式速览卡片
- 内容：
  - 最危险话术 TOP 3（通过率最低，样本≥3）
  - 最安全话术 TOP 3（通过率最高，样本≥3）
  - 本周新增违规数 / 本周新增案例数
  - 最新违规公示（最近 3 条红色卡片）
- 后端：新增 `/api/violations/dashboard-summary` 接口
- 前端：简单卡片，后续重构时再打磨

### 2. 画面标签体系

- 目的：解决"同一画面在不同账号表现"的聚合问题
- 方案：管理员审核时给案例打"画面标签"（如"蓝底箭头引导图"）
- 后端：
  - 新表 `visual_tags`（id, name, description, created_by, created_at）
  - 关联表 `violation_case_visual_tags`（case_id, tag_id）
  - 聚合接口 `/api/violations/visual-tags/[tagId]/stats` → 返回该画面的通过率/使用账号列表
- 前端：审核面板加标签选择器，列表页支持按画面标签筛选

### 3. 批量导入

- 模板：Excel(.xlsx)，6 列：话术原文 / 违规或可用 / 账号名 / 处罚类型 / 平台 / 备注
- 后端：`/api/violations/import` POST，解析 xlsx，逐行校验+入库
- 前端：管理员可见的上传页面（简单 file input + 结果反馈）
- 依赖：`xlsx` 或 `exceljs` 库

## 决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| Dashboard 位置 | /violations 首页顶部折叠区 | 不增加入口，团队规模小 |
| 画面聚合方式 | 管理员手动打标签 | 月 300 条，AI 图片相似度过重 |
| 批量导入格式 | 自定义 Excel 模板 | 杜老师目前没有现成表格，按模板填最简单 |
| VisualAsset 独立表 | 暂不建，用标签聚合替代 | 数据量小，标签够用 |
