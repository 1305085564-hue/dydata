## 十四、字号阶梯（V1 第 9.1 节扩充）

V1 第 9.1 节字号音阶为「12 / 13 / 14 / 16 / 20 / 24」，日常填写够用，但大数据展示缺顶格档，分组标签缺小眉题档。V1.2 扩充为 **7 档完整阶梯**，并明确每档语义用途：

| 档位 | 字号 / 行高 | 字重 | 字距 | 用途 |
|---|---|---|---|---|
| Display | `text-[32px] leading-[1.25]` | `font-semibold` | `tracking-tight` | 核心数据大数、仪表盘顶格指标 |
| Section | `text-[24px] leading-[1.33]` | `font-semibold` | `tracking-tight` | 模块标题、页面二级标题 |
| Title | `text-[18px] leading-[1.44]` | `font-medium` | 默认 | 弹窗标题、小模块标题 |
| Body-L | `text-[14px] leading-[1.7]` | 默认 | 默认 | 中文密集列表、长段正文 |
| Body | `text-[13px] leading-[1.7]` | 默认 | `tracking-[0.005em]` | **默认正文（英文/数字密集）** |
| Label | `text-[13px] leading-[1.23]` | `font-medium` | 默认 | 表头、导航项、表单标签 |
| Caption | `text-[12px] leading-[1.33]` | 默认 | 默认 | 脚注、辅助说明 |
| Eyebrow | `text-[10px] leading-[1.6]` | `font-medium` | `uppercase tracking-[0.25em]` | 小眉题、分组标签、状态 chip |

### 硬规
- **禁** `text-[16px]` 作为正文默认（V1 原规，继续有效）
- **禁** `font-bold` / `font-black`，最高 `font-semibold`
- **禁** 在同一视觉块内跨 3 档以上字号切换
- 数字 / ID / 图表数值 **全站强制** `tabular-nums font-mono`

### Display 档使用边界
- 每屏 **最多 1 处** Display 档（仪表盘核心大数、"爆款数据"）
- 必须有 ≥ 32px 留白包围
- × 弹窗内 × 列表项内

## 十五、间距 6 档（V1 未定义，本次补齐）

V1 缺空间尺度标准，导致不同组件留白随意。V1.2 定义 **6 档间距令牌**：

| 档位 | 像素 | Tailwind | 用途 |
|---|---|---|---|
| XS | 4px | `gap-1` / `p-1` / `space-y-1` | icon 与文字、chip 内部 |
| S | 8px | `gap-2` / `p-2` / `space-y-2` | 相邻表单字段、小标签内部 |
| M | 16px | `gap-4` / `p-4` / `space-y-4` | 卡片内部、模块次级分组 |
| L | 24px | `gap-6` / `p-6` / `space-y-6` | 卡片 padding、主分组间 |
| XL | 32px | `gap-8` / `p-8` / `space-y-8` | 模块间、Section 之间 |
| 2XL | 40px | `gap-10` / `p-10` / `space-y-10` | 页面级主分区、顶级 hero |

### 硬规
- **禁** 非档位值：`p-3` / `p-5` / `p-7` / `gap-3` / `space-y-5` 等
- 存量项目可保留 `p-3`，新增代码严格按 6 档
- × 视觉"差不多"就用，必须匹配语义档位

### 特例白名单
- 导航项 `px-3 py-2`（V1 原规，保留）
- 输入框 `h-10 px-3` / `h-11 px-4`（高度优先于 padding 档）
- shadcn 组件内部默认间距保留

## 十六、状态点四态（V1 第 12 节「状态灯」升级）

V1 原：「实心圆点 + 同色 12% 扩散光晕，紧急加 animate-pulse」。
V1.2 升级为 **四态 + 白外圈锚定** 标准，提升仪表盘质感（V2 草案验证有效，与底色无关）：

| 态 | 尺寸 | 填充 | 外圈 | 动效 | 场景 |
|---|---|---|---|---|---|
| Critical | 8px `h-2 w-2` | `bg-[#C9604D]` | `ring-1 ring-white` | `animate-pulse` 2s | 失败、驳回、严重告警 |
| Active | 8px `h-2 w-2` | `bg-[#D97757]` | `ring-1 ring-white` | 静态 | 当前步、进行中、待处理主动作 |
| Done | 8px `h-2 w-2` | `bg-[#6FAA7D]` | `ring-1 ring-white` | 静态 | 完成、通过、在线 |
| Idle | 6px `h-1.5 w-1.5` | `bg-zinc-200` | 无外圈 | 静态 | 未开始、无状态、占位 |

### 并列降权规则
同一视觉块内若 Critical + Active 共存：**Active 降权到 6px 静态**，保持 Critical 唯一"跳出点"。

### 实现模板
```tsx
// Critical（跳动红点）
<span className="h-2 w-2 rounded-full bg-[#C9604D] ring-1 ring-white animate-pulse" />

// Active（静态暖橙）
<span className="h-2 w-2 rounded-full bg-[#D97757] ring-1 ring-white" />

// Done（静态绿）
<span className="h-2 w-2 rounded-full bg-[#6FAA7D] ring-1 ring-white" />

// Idle（小灰点，无外圈）
<span className="h-1.5 w-1.5 rounded-full bg-zinc-200" />
```

### 硬规
- **禁** 状态点带多色渐变
- **禁** 状态点 ≥ 10px（会失去"点"的克制）
- 1px `ring-white` 外圈是四态的**视觉共识锚点**，不可省

## 十七、左 2px 彩色导轨（V1 第 4.3 / 12 节明确化）

V1 合法彩色载体之一是"卡片左侧 2px 实色导轨边框"，V1.2 明确其**哲学含义**与**使用场景**：

### 哲学
左 2px 导轨 = **刻在纸面上的刻度条**。彩色用于"几何体语言"（2px 条、小圆点、数字本身），不用于"彩色底面"。

### 标准配方
```
border border-zinc-200 border-l-[2px] border-l-[#语义色]
```

### 语义映射
- Critical 告警 / 违规 / 异常行：`border-l-[#C9604D]`
- Pending 待处理 / 确认卡 / 提醒：`border-l-[#D99E55]`
- Done 已完成 / 通过状态行：`border-l-[#6FAA7D]`
- Primary 当前激活项：`border-l-[#D97757]`

### 使用场景（必须有 2px 导轨）
- 异常告警条（如 `alert-center` 外层容器）
- 违规话术列表项（V1 原规，继续）
- 确认卡 / 豁免申请卡（V1 原规，继续）
- 分组区（核心数据 / 互动 / 完播）的标题区——多彩导轨构成"报表刻度带"

### 禁区
- × 2px 导轨 + 彩色背景面同时用（会成补丁感）
- × 导轨渐变（V1 原规，必须实色）
- × 同一容器同时存在 > 1 条 2px 导轨

### 落地范例
`src/app/(app)/dashboard/components/alert-center.tsx` L36：
```tsx
<div className="overflow-hidden rounded-2xl border border-zinc-200 border-l-[2px] border-l-[#C9604D] bg-white shadow-sm">
```

## 十八、激活态黑墨不用塑料（V1 第 4.3 / 第 12 节补充）

> 黑色不是问题，塑料感才是问题。

V1 已允许"激活 Tab `bg-white shadow-sm`"、"用户消息气泡 `bg-zinc-950 text-white`"。V1.2 明确：**深色实心激活态的禁用项**，避免"塑料按钮"回归：

### 当激活态用 `bg-zinc-800 / bg-zinc-900` 深色实心时：
- **禁** 带 `shadow` / `shadow-sm` / `shadow-md`（阴影 = 塑料感来源）
- **禁** 带 `border border-white` 或任何亮色描边
- 允许 `border border-zinc-900` 同色加深锚边

### 哲学对应
- 纸本可以盖黑墨印章（`bg-zinc-800` 无 shadow 无亮边 = 黑印章）
- 纸本不应出现塑料按钮（`bg-zinc-800 shadow-sm` 就是塑料感）

### 主 CTA 维持 V1 原规
主 CTA `bg-[#D97757]` 的 Hover 弥散阴影 ≤ 3%（V1 第 8.4 节），继续有效，不受本条限制（因为主 CTA 不是"深色实心激活态"）。

## 十九、克制的鲜艳（V1 第 4 节"3% 彩色面积"松绑）

> 颜色不是问题，刺眼才是问题。
> 克制的鲜艳 ≠ 禁用鲜艳。过度灰化会把页面做成"工业控制面板"，失去设计感。

V1 原规是"彩色面积 ≤ 3%"+"仅几何体语言可上色"。V1.2 实际落地中发现这条过严——漏交日历、状态芯片、图例圆点这些**本身就是信息色块**的位置被迫退成灰底，美感严重损失。V1.2 明确**色彩丰富**与**色彩刺眼**的分界线，允许"克制的鲜艳"。

### 核心判断句
**克制的鲜艳** = **浅色族（50/100/200）+ 深色族（600/700）文字 + 小面积 + 不闪烁**
**刺眼的深色** = 饱和度满格的 400/500/600 大面积底色 + 高对比文字 + 动效

### 允许的"克制鲜艳"（V1 原规未禁，但过去误伤）
| 场景 | 允许的上色方式 | 举例 |
|---|---|---|
| 信息色块（图例/芯片/状态 chip） | `bg-{hue}-50 text-{hue}-700` + 浅边 | 日历图例 `bg-emerald-50 text-emerald-700` |
| 数据状态格（日历单日、表格单元、标签） | `bg-{hue}-50 border-{hue}-200 text-{hue}-600-700` | 漏交 `bg-rose-50 text-rose-600 ring-rose-100` |
| 进度条已完成段 | 主色 `#D97757` 或 `#6FAA7D` 实色 | 完播进度条 |
| 微图标底托（kbd / 数字角标） | `bg-{hue}-50/70` + 柔边 | 红点角标 |

### 仍然禁止的"刺眼深色"
- × 大面积 `bg-{hue}-400/500/600`（头图 hero / 卡片背景 / 主画布）
- × `bg-{hue}-50` 面积超过**单视口 15%**（如大横幅、全宽 banner）
- × 深饱和底色 + 动效（闪烁 / 渐变移动 / 彩虹色轮播）
- × 单视口内同时出现 ≥ 4 个色相（hue）

### 允许的色族扩展（V1.2 打开）
V1 只允许暖橙 `#D97757` + 琥珀 `#D99E55` + 砖红 `#C9604D` + 苔绿 `#6FAA7D` 四个语义色。V1.2 允许在**信息色块**场景使用 Tailwind 的 50/100/200/600/700 浅族：

| 语义 | Tailwind 色族 | 用途 |
|---|---|---|
| 成功 / 已完成 | `emerald-50/100/200` + `emerald-600/700` | 已交、完成、通过 |
| 提醒 / 进行中 | `amber-50/100/200` + `amber-600/700` | 请假、待处理、待确认 |
| 危险 / 漏交 | `rose-50/100/200` + `rose-600/700` | 漏交、未交、违规 |
| 信息 / 链接 | `sky-50/100/200` + `sky-600/700` | 链接、提示、次要操作 |

**硬规**：这四族**仅用于 50/100/200 底 + 600/700 字**组合，× 单独用 400/500 深色做背景。

### 4 个语义色 vs Tailwind 浅族 —— 何时用哪个？
- **主 CTA / 状态锚点 / 几何体语言（2px 导轨、状态点）** → 用 V1 四色（`#D97757` / `#D99E55` / `#C9604D` / `#6FAA7D`）
- **信息色块 / 日历格 / 图例芯片** → 用 Tailwind 浅族（rose-50 / emerald-50 / amber-50 / sky-50）
- 两者可共存：状态点用 `#C9604D` 实色，日历格底用 `rose-50`，**同色相但不同饱和度层级**，构成"锚点 + 点缀"双层关系

### 落地范例
```tsx
// 克制的鲜艳（正）
<span className="inline-flex items-center gap-2 rounded-full bg-emerald-50 px-3 py-1 text-emerald-700">
  <span className="size-2 rounded-full bg-emerald-500" />
  已交
</span>

// 刺眼的深色（禁）
<div className="bg-emerald-500 text-white p-8">已交</div>
```

### 哲学
- 法典不是"灭色"，是"驯色"
- 克制的鲜艳像**素描本上的水彩点染**——纸本（zinc 灰 + 白）为底，彩色点缀在信息锚点上
- 颜色丰富 ≠ 颜色跳跃。同一视口可以 3-4 个色相**同时出现**，只要都在 50/100/200 浅族内，视觉依然克制

### 修订：V1 第 4 节的"3% 彩色面积"
V1 原文保留，但**补充例外**：
- 信息色块（图例、芯片、状态格、数据单元）**不计入 3%**
- 3% 面积规则仅适用于"卡片大底 / 横幅 / hero 区 / 模块级背景"等**装饰性**彩面
- 信息色块属于"内容色"，不是"装饰色"，豁免
