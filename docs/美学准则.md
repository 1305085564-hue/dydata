# 美学准则

适用：阿禅名下所有项目（数据分析 / 话术库 / 团队中控 / AI 工作站）
权威：`~/.claude/memory/美学准则.md`（本文 = 哲学与方向）
执行：`~/.claude/memory/美学标准.md`（量化硬规，施工时必须对照）
附件：`~/.claude/memory/阿禅美学附件-V1.2.md`（grep 验收 / 版本史 / 项目锚点，参考用）

本文 = 风格哲学 + 判断框架。不写像素值，不写 Tailwind 类名。具体数值见《美学标准》。

---

## 一、四哲学

1. **Typography as Interface**
   留白、字距、字重建立层级。× 用线框 × 用底色堆层级。文字的呼吸感本身就是界面。

2. **Subtractive Design**
   能不用线就不用，能不用阴影就不用。每加一个元素都要回答"去掉它会不会更干净"。

3. **Monochromatic Kinetic**
   日常交互纯灰阶，× 彩色。质感靠灰度变化 + 物理位移（微浮、回弹），不是靠颜色热闹。

4. **Controlled Vibrancy**
   彩色是信号，× 装饰。信号必须克制——小面积、不闪烁、有明确语义。装饰性彩面总面积 ≤ 3%。

---

## 二、信息量与密度

**舒适大方** — 阿禅的默认偏好。

- 留白充足、主视觉明确、尺寸偏宽松
- × 极简（冷漠） × 紧凑（拥挤）
- 中等信息量是常态：每个元素必须承担职责，无职责就删
- 单屏元素数量宁少勿多

**密度判断**：信息密度 = 单位面积承载的视觉信号数。同等职责下，选密度更低的方案。

---

## 三、主次分明（硬要求）

任何页面 / 区块必须能一句话回答"第一眼看哪里"。无主视觉 = 设计失败。

副视觉（小眉题 / 副标题 / 次级按钮）的存在价值是衬托主视觉，× 并列。并列 = 竞争 = 失败。

---

## 四、交互区与背景必须分离

手法不限：卡片 / 底色分层 / 色带 / 阴影 / 边框 / 细节装饰。

选择标准和整体调性匹配：保守场景用卡片，追求设计感用 2px 色带或精确间距锚点。

× 交互区与背景融为一体（画面"糊"的主因之一）。

---

## 五、动画判断三问

1. 打断用户操作？→ 不加
2. 循环？→ 慎用（循环是注意力窃贼）
3. 用户"感觉到"但没"注意到"？→ 不加

三问全过才加。优先用 CSS 过渡，× 弹簧过冲。

---

## 六、设计感 > 保守

同等效果选更有设计感的。保守是兜底，× 首选。

"保守且正确" ≠ "好"。允许克制的惊喜——一条 2px 色带、不寻常的间距、精确动画节奏。克制 = 一次一处，× 到处撒。

---

## 七、规范是底线 × 天花板

字号、四色、禁区是底线，不能破。底线之上允许"克制的惊喜"。

---

## 八、反面共性警报

好参考差异大（Vercel / Linear / Notion 各不同）→ × 抄某一家。

反面案例共性稳定，见以下信号立即警报：
- **主次不分**（多元素抢焦点）
- **元素抢戏**（装饰抢内容）
- **无意义装饰**（不承担职责的图形 / 色块 / 线条）
- **过度极简导致冷漠**（空气撑场，× 人情味）

警报 ≠ 禁用，看到后判断当前场景是否真的有问题。

---

## 九、阿禅语汇速查

| 阿禅说 | 真实诉求 |
|---|---|
| 画面"糊" | 主次不分 / 交互区与背景融合 |
| "大方" | 留白充足 + 主视觉明确 + 尺寸偏宽松，× 极简 × 紧凑 |
| "高级感" | 克制 + 惊喜细节，× 炫技 |
| "惊喜感设计" | 不常见但合理的手法，× 循规蹈矩 |
| "堆了" | 元素无职责 / 抢视觉 |
| "抢戏" | 副级元素视觉权重过大，需要**降级到点缀**级别 |
| "点缀" | 感觉得到但不抢主视觉，第一眼不落在它上面 |
| "交互第一" | 使用逻辑清晰优先于画面全 |
| "中等信息量" | × 极简 × 堆砌，每个元素有职责 |
| "舒适" | 默认选宽松档，× 紧凑 |
| "高密度" | 信息过载 / 视觉干扰过多，需降密度 |

## 十、色彩方向

### 10.1 层级观

底色只做三层区分（页面 / 卡片 / 焦点区），不靠阴影做层级。层级由底色差 + 细边框构成。

### 10.2 灰阶克制

主文字不用纯黑，用深灰。元数据用更浅的灰。分隔优先用留白，× 线条。

### 10.3 四语义色

- **动作** → 暖橙（主 CTA、当前激活）
- **状态** → 矿物色（绿 = 完成/通过，琥珀 = 待处理/提醒，红 = 失败/违规）
- Primary 永远只有暖橙
- 在线 / 复制成功 → 绿（× 橙）
- 生成中 / 待授权 → 琥珀（× 橙）

### 10.4 克制的鲜艳

**核心判断句**：
- 克制鲜艳 = 浅色族底 + 深色族字 + 小面积 + 不闪烁
- 刺眼深色 = 饱和度满格大面积底色 + 高对比文字 + 动效

信息色块（图例、芯片、状态格）是内容色不是装饰色，不受 3% 限制。

---

## 十一、形态方向

### 11.1 圆角

对话区 / 主面板用最大圆角，输入框外壳用更大圆角，内层卡片适中，按钮 / 标签较小，状态点 / 头像正圆。

### 11.2 边框温度

分隔线极冷，可交互元素微温，激活态用暖橙描边。Focus Ring 用微灰，× 暖橙做临时聚焦。

### 11.3 字号档位

七档制，禁止非档位值。正文默认 13-14px，× 16px。最高字重 semibold，× bold/black。

Display 档（最大字号）每屏最多一处，必须是核心数据大数或仪表盘顶格指标。

### 11.4 间距档位

六档制，禁止非档位值。默认偏宽松档，× 紧凑。

---

## 十二、交互方向

### 12.1 输入框

Soft Fill：默认灰底透明边框，Focus 时白底 + 微灰描边 + 微灰 focus ring。× 蓝色 focus ring。

### 12.2 动效

Hover 150ms，Active 回弹 80ms。× 弹簧过冲。列表 / 导航 Hover 可延迟 50ms，按钮 / CTA / 输入框不加延迟。

### 12.3 Hover 去色化

列表 / 次级按钮 / 导航 Tab 的 Hover 严禁彩色。用灰底或白底，配合微浮 + 微阴影，Active 物理回弹。

### 12.4 主 CTA

暖橙底色，Hover 弥散阴影（透明度 ≤ 3%）。次级按钮完全 × 阴影。

---

## 十三、组件方向

### 13.1 排版

标题 / 核心数据用 semibold + 紧凑字距。元数据 / 分组标签用小字号 + 大写 + 宽字距 + medium 字重。数字强制等宽字体。

### 13.2 空状态

灰阶文案，× 彩色插画 × 彩色图标。诗意一点，不要冷冰冰的"暂无数据"。

### 13.3 加载

< 200ms 直出，≥ 200ms 骨架屏（灰矩形，结构与最终一致），200ms 淡入替换。× spinner。

### 13.4 列表

斑马线或仅 Hover 当前行。卡片底部元数据用留白分隔，× border-top。

### 13.5 状态点

四态：Critical（脉冲）/ Active（静态）/ Done（静态）/ Idle（更小）。× 渐变 × ≥ 10px。

### 13.6 导轨

异常 / 违规用红色导轨，待处理用琥珀，完成用绿色，当前激活用暖橙。必须实色，× 渐变。× 导轨 + 彩色背景同时用。

### 13.7 激活态

深色实心激活态（黑底）× 阴影（塑料感）。每屏至多一个黑底主按钮，必须是终局动作（保存 / 提交 / 通过）。角色徽章 / 权限标签 / 状态 chip × 黑底。
