# 选题库接口文档

> 最后更新：2026-07-16  
> 范围：批次三选题库后端。只包含后端接口、字段、错误和回滚说明；不包含前端页面。

## 一、数据表

- `topics`：母题，字段 `id/name/sort_order/created_at`。
- `topic_groups`：母题下分组，字段 `id/topic_id/name/sort_order/created_at`。
- `sub_topics`：子题，字段 `id/title/hook/topic_id/group_id/emotion_tag/source/audience/created_by/created_at`。
- `sub_topic_claims`：认领记录，字段 `id/sub_topic_id/user_id/status/claimed_at/returned_at`，`status` 只能是 `candidate`、`scripting`、`returned`。
- `videos.topic_id`：作品关联到子题；为空代表暂无对应选题。

## 二、通用规则

- 所有 `/api/topics/*` 接口都要求登录，未登录返回 `401 { "error": "未登录" }`。
- 子题读取对已登录用户开放；编辑和删除只允许创建人。
- 作品、文案和播放数据按当前用户可见范围过滤，避免普通成员看到不该看的视频数据。
- 候选认领上限为 5 条；超过返回 `409`。

## 三、接口

### `POST /api/topics/sub-topics`

创建子题。

请求字段：
- `title` 必填，子题名。
- `hook` 必填，一句话选题。
- `topic_id` 必填，母题 ID。
- `emotion_tag/source/audience` 可选。

行为：后端会按 `title + hook` 自动匹配 `topic_groups`；匹配不上则 `group_id = null`。

### `GET /api/topics/sub-topics/:id`

获取子题详情，包含子题基础信息和前 20 条作品/文案数据。

### `PATCH /api/topics/sub-topics/:id`

编辑子题。只允许创建人编辑。修改 `title/hook/topic_id` 时会重新自动归类分组。

### `DELETE /api/topics/sub-topics/:id`

删除子题。只允许创建人删除；已有 `videos.topic_id` 关联作品时返回 `409`，不能删除。

### `POST /api/topics/sub-topics/:id/claim`

认领到候选。若当前用户已有 5 条 `candidate`，返回：

```json
{ "error": "候选选题最多保留 5 条，请先放回一个选题" }
```

### `POST /api/topics/sub-topics/:id/start-scripting`

把当前用户对该子题的认领状态从候选改为 `scripting`。

### `POST /api/topics/sub-topics/:id/return`

把当前用户对该子题的认领状态改为 `returned`，并写入 `returned_at`。

### `GET /api/topics/pool`

选题池列表。

查询参数：
- `view`：`all`、`my_claims`、`my_created`，默认 `all`。
- `time_range`：`3d`、`1w`、`1m`、`3m`，默认 `1m`。
- `topic_id`：可选，按母题筛选。
- `page/page_size`：可选，默认 `1/50`，最大 `page_size=100`。

返回包含子题列表、分页、认领人数、作品汇总。作品汇总只统计播放量不低于 1000 的作品。

### `GET /api/topics/active`

今日工作台“最近活跃”数据源。

返回四组：
- `recentlyClaimed`：最近被认领。
- `recentlyWorked`：最近有新作品。
- `recentlyCreated`：最近录入。
- `worthRedoing`：当前可见范围内已有至少 1 条播放不低于 1000 的作品，按合格作品平均播放倒序；每项含子题基础信息和 `summary`。

### `GET /api/topics/sub-topics/:id/claims`

认领动态，用于提示同一子题是否有人正在制作。

- 身份列表只返回当前用户业务可见范围内的成员。
- `candidateCount` 与 `scriptingCount` 仍按全部未归还认领计数，避免范围外成员造成撞车风险被忽略。
- 返回项按 `scripting` 优先、再按认领时间倒序。

### `GET /api/topics/comparison?dimension=topic|account&days=30&topicId=<可选>`

选题效果横向比较。`dimension=topic` 按母题聚合；`dimension=account` 按账号和母题交叉聚合。只统计当前可见范围内、指定时间内有 24 小时播放快照的活跃作品。

每行包含作品数、达标数、达标率、平均播放、最高播放；作品数少于 3 时 `lowConfidence=true`。默认按达标率、平均播放、作品数倒序。

### `GET /api/topics/recommendations?days=14&accountId=<可选 UUID>`

基于当前可见范围的近期爆款样本生成 AI 选题建议。`days` 只接受 1 到 90 的整数；传入 `accountId` 时必须是 UUID。返回 `suggestions`、`evidenceSummary`、`sampleCount` 与 `marketDate`，建议项结构与 content-tools 的 `TopicSuggestionItem` 一致。

### `POST /api/topics/sub-topics/from-recommendation`

采纳一条 AI 建议并写入子题库。

请求字段：`title`、`angle` 必填；`category`、`emotion_tag`、`audience` 可选。`angle` 写入 `hook`，写入来源固定为 `ai_recommendation`。

`category` 必须精确匹配线上已有母题；当前数据库没有“常规母题”且子题不能没有母题，因此缺失或不匹配时返回 `400`，不进行错误归类。

### `GET /api/topics/sub-topics/:id/works`

子题作品和文案收纳。

查询参数：
- `sort`：`best` 或 `recent`，默认 `best`。
- `page/page_size`：默认 `1/50`。

返回：
- `items`：当前子题作品。
- `similarReferences`：同母题同分组下其他子题的相近作品，标记 `referenceType: "similar"`。
- `summary`：不低于 1000 播放作品的平均播放、最好文案、最近文案。

### `GET /api/topics/sub-topics/suggest?title=xxx&content=xxx`

兜底匹配推荐。根据标题和文案做关键词相似度排序，返回 top 3 子题候选；不调用 AI，不做智能评分。

### `POST /api/video-submit`

新增可选字段：
- `topic_id`：子题 ID，可空。

传入后写入 `videos.topic_id`；不传则旧提交链路保持不变。

## 四、错误码

- `400`：参数或请求体格式不正确。
- `401`：未登录。
- `403`：没有权限或不是创建人。
- `404`：子题或认领记录不存在。
- `409`：候选超过 5 条，或已有作品关联导致不能删除。
- `500`：数据库或服务端错误。
