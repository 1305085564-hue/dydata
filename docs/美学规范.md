# DYData 美学规范

> 适用：阿禅名下所有项目（DYData 及延伸产品）
> 核心哲学：**简单有深度**——第一眼就知道高级、容易看懂；细品之后，发现还有惊喜和洞天。
> 阅读方式：从头到尾读一遍建立直觉，之后按需查阅。设计时脑子里装着"设计灵魂"那段画面，而不是逐条对规则。

---

## 1. 设计灵魂

你是 DYData 的 UI 设计师。在你动手之前，先在脑子里看到这幅画面：

> 一张暖灰色的大桌面，光线从左侧洒进来。桌面中央铺着一块巨大的白色鼠标垫——占据了桌面的大部分面积，舒展、大方，边缘柔和但没有封死。键盘、鼠标、一杯咖啡、一个笔盒，安静地分布在鼠标垫的不同位置，靠彼此的距离自然成组——没有人用线去隔开它们。
>
> 暖灰桌面从鼠标垫四周露出来，像一个温暖的画框托住了整个工作区。桌上只有一处亮色：一支暖橙色的笔，放在最重要的笔记旁边，像聚光灯一样标记着此刻的焦点。其余一切都是深浅不同的灰——安静、有序、不抢注意力。你坐下来工作十分钟后，才发现马克杯釉面的微妙渐变、鼠标垫边缘的柔和圆角——这些细节不会第一眼跳出来，但它们让你觉得：嗯，这个地方待着真舒服。

**这就是 DYData 的气质**：温暖、清楚、高级、有细节。不是冷硬的科技后台，不是极简到冷淡的留白，不是堆满装饰的花哨面板。是一个**让人愿意待一整天的、有品位的工作环境**。

**设计信条**：

- **美学是上限，效率是底线。** 不会为了美牺牲可用性，但更不会为了"效率优先"而主动放弃美。两者冲突时，先找同时满足的方案；实在不行，保底线但要心疼上限。
- **克制不等于寡淡。** 克制是"每个元素都有存在的理由"，不是"尽可能少放东西"。一杯好茶是克制的，但它不寡淡。
- **简单有深度。** 第一眼清晰——视觉层级一眼可辨，信息分组清楚，有呼吸空间。细品有惊喜——坐下来用十分钟后，才发现那些被精心安排的微妙细节。好的设计三层同时存在。

## 2. 视觉基因

> 全站风格一致是唯一的硬性要求。以下定义 DYData 的视觉 DNA——无论哪个页面，用户都能感受到"这是同一个产品"。
>
> 本产品为纯亮色模式设计。所有视觉方向、色彩选择和氛围营造均基于亮色环境。

### 2.1 空间层级

> 一切视觉决策的上游。动手之前先定：这块内容在空间中处于哪一层？

**骨架**

**L0 画框** — 工作室的墙壁与画框。顶栏、侧边栏及其底色 `bg-stone-50`（`#FAFAF9`），围合出舞台但自身不承载业务内容。L0 暖灰始终在顶部和侧边稳定露出，作为画框托住 L1，营造包容感。

**L1 主画板** — 视口内的整块白色工作台面。`bg-white`，占据视口的主要面积（内容舒展大方，让用户感受到"一整张大画纸在我面前展开"），内容瀑布式向下生长。**一个页面只有一个 L1。** L1 的底部是开放的——内容有多长画板就有多长，不封口、不截断。

**L3 悬浮** — 飘在空中的便签与菜单。`shadow-lg`，操作完即消失，不计入嵌套深度。

**空间心法**

1. **画框与画板** — L0 是温暖的画框，L1 是框内舒展的大白纸。用户一眼能看到"暖灰包围白色"的层次感。
2. **凸起与凹陷** — 方向即语义。凸 = 行动，凹 = 从属。
3. **两层即止** — 静态布局只有 L0 + L1 两个空间层。L1 内部靠六种表达形态做层级。
4. **凹槽是终点** — 凹槽（Inset）是画板向内刻下的底座，是深度的终点。凹槽内部只承载印记文字和通透条目，深度到此为止，不再向上凸起。
5. **间距优先于线条** — 用留白和色阶差建立模块关系；一个局部区域内最多保留 1 条极柔细线作辅助。Header 与其所属内容保持紧密贴合，用间距表达从属而非用线切割。

**L1 画面上的六种表达形态**

L1 是统一的白色画板，内部所有视觉层级变化靠以下方式"加工"。可叠加、不互斥。

| # | 名称 | 物理隐喻 | 视觉实现 | 何时使用 |
|---|---|---|---|---|
| ① | 凹槽 Inset | 画板表面刻出的浅槽 | `bg-stone-100/60`，无边框 | 展开详情、历史数据、次级筛选 |
| ② | 印记 Flat Print | 墨水直接印在纸面 | 字号 + 色深阶梯 + 留白 | 标题正文、数据阵列 |
| ③ | 贴纸 Badge | 角落贴上的彩色便签 | 胶囊圆角 + 色底 `/0.12` + 同色实字 | 状态标签、分类标签、计数器 |
| ④ | 浮雕 Raised | 微微凸起的物理按键 | 暖橙实底 + `shadow-sm`，hover 上浮 | 主 CTA、激活态 Tab（每区至多 1–2 处） |
| ⑤ | 光影 Glow | 釉面散发的温暖微光 | 功能色光晕 `opacity 0.02–0.08`，无硬边 | 核心 KPI 重量感、异常氛围 |
| ⑥ | 附着 Docked | 磁铁吸在白板上的小工具 | `shadow-md`，紧贴内容边缘，带过渡 | hover 工具栏、吸附 Tooltip |

### 2.2 色彩世界

DYData 的色彩是**温暖而丰富的**。不是只有灰色的世界，而是以暖灰为底色、功能色为路标、氛围色为空间感的完整世界。全站中性色使用 stone 色系，保持温暖一致的基调。

#### 品牌性格色

**暖橙 `#D97757`** 是 DYData 的性格——温暖、可亲、值得信赖。用户看到暖橙就知道：**这里可以点。**

暖橙按浓度分层使用，越重要的操作越浓：

| 层级 | 用途                   | 表现                                   |
| ---- | ---------------------- | -------------------------------------- |
| 最浓 | 每屏唯一的主 CTA 按钮  | 暖橙实底 + 白字                        |
| 中等 | 次要操作按钮、重要图标 | 暖橙文字或描边                         |
| 轻触 | 正文内链接、低频操作   | `stone-700` + 下划线，hover 时亮为暖橙 |
| 最淡 | 工具栏图标、辅助入口   | `stone-500`，hover 时 `stone-700`      |

#### 功能语义色

| 名称   | 色值      | 含义           | 用户心声     |
| ------ | --------- | -------------- | ------------ |
| 暖橙   | `#D97757` | 行动/可交互    | "这里可以点" |
| 石青   | `#5F82A8` | 选中/当前/定位 | "我在这里"   |
| 琥珀   | `#D99E55` | 提醒/待处理    | "需要留意"   |
| 矿物绿 | `#6FAA7D` | 成功/通过/正常 | "搞定了"     |
| 矿物红 | `#C9604D` | 异常/错误      | "出问题了"   |

这五个是**语义色（路标）**，锁的是"用途"而非"数量"。它们回答"这告诉用户什么"，可作轻淡底色、图表主色、标签背景，**要有存在感**，不是蜻蜓点水的色块。

#### 表达色池

语义色之外的开放色层，专供图表多线/多类、分类标签、氛围底色。统一从池子里取，不随手调色，才不会花。

| 名字   | 色值      | 气质       |
| ------ | --------- | ---------- |
| 靛青   | `#4F5E96` | 沉的蓝紫   |
| 藕紫   | `#7E5C99` | 实的紫     |
| 陈玫   | `#A85A6B` | 灰调玫红   |
| 青瓷   | `#3E8479` | 沉静青绿   |
| 苔橄榄 | `#7A7A3E` | 大地橄榄   |

图表直接取实色当线色；分类标签用"淡底 `/0.12` + 同色实字"；氛围时把实色调到 `/0.08` 当背景光晕。

#### 色彩使用节奏

功能色要有存在感——不要只用在最小面积上。图表和数据可视化是色彩的主要舞台，大胆使用完整光谱。异常数据用颜色温和地"托出来"，不是用闪烁去"警报"。页面整体保持白底 → 功能色亮点 → 白底的冷暖交替韵律。

### 2.3 文字系统

全站无衬线字体：`Inter, -apple-system, PingFang SC, system-ui, sans-serif`。数字使用 `tabular-nums` 保证数据列对齐。

文字通过**字号和色深**建立层级，而不是靠粗体堆叠：

- 字号分四档：页面标题（24px）→ 卡片标题（18px）→ 正文（13px）→ 辅助说明（12px）
- 色深分三阶：`stone-900`（标题/核心数据）→ `stone-700`（正文）→ `stone-500`（辅助信息，全站最浅文字）
- 字重以 `normal` 和 `medium` 为主。核心 KPI 每屏至多一处可用 `semibold`
- 中文行高 `leading-[1.6]`，保证阅读舒适

允许在需要**节奏对比**的场景灵活运用字重——比如标签用 `font-medium` 的小号字"压实"成独立元素，与旁边正文形成节奏差。

> 具体字号阶梯和字重字色参数表见《美学规范补充》。

### 2.4 形态语言

**圆角是语义**：越圆越"独立"（标签、徽章用胶囊），越方越"容器"（卡片、面板用大圆角）。按钮居中——圆角大小随按钮高度递增。

**边界与层级**：遵循空间层级（2.1）心法——L1 内部分区优先用留白和色阶差建立关系，阴影只给浮雕型（④）和悬浮层（L3）。

**状态表达**：选中、激活、高亮的最高境界是"形不动，意已明"。优先用品牌字色 + 字重传达状态（轻量、零形状负担），其次是轻微背景贴片，最后才考虑边框或阴影盒子。

**功能域隔离**：同类操作（如多个 Tab）靠字距留白自然排开；不同功能域的操作之间用微隔断（一根极柔竖线或一个轻微气泡底）建立秩序，让物理排布忠实映照功能逻辑。

**图标**：全站统一 Lucide Icons，stroke-width 1.5px。默认 `stone-500`，hover `stone-700`。

> 圆角阶梯、阴影参数见《美学规范补充》。

### 2.5 空间与节奏

间距是**信息关系的视觉语言**——靠得近的元素用户认为有关系，离得远的认为是两件事。空间控制比线条更高级、更安静、更有品位。

**空间亲疏**：用距离天然表达业务逻辑，让用户不看文字就能感知"这几个东西是一组"——

- **紧密**（≤8px）：强关联元素自然贴合——图标与文字、标签与标题、Header 与其下方第一条内容
- **组件**（12–16px）：同一模块内部的子内容——标题与摘要、输入框与提示文案
- **模块**（24–32px）：不同业务模块之间——大脑看到这个距离会自动判断"这是两件不同的事"
- **区块**（40px+）：页面级大区域分隔，营造呼吸感

**空间模数**：全站间距以 8px 为基本单位，所有组件的 padding、margin、gap 都是 8 的倍数（4px 仅用于图标与文字的微观间隙）。统一的模数让页面自带韵律感。

## 3. 深度工具箱

> 以下手法帮助创造"细品有惊喜"的体验。像调味料一样使用——恰到好处。

### 3.1 光与影

- **环境光**：页面顶部或侧边的极淡渐变，暗示光源方向，给页面"空间感"（氛围色 `opacity 0.02–0.05`）
- **焦点光**：核心 KPI 底部的温暖微光，让数字有"重量感"而非漂浮
- **交互影**：按钮 hover 时微微上浮的阴影加深，暗示"可以按下去"
- **深度影**：悬浮层的 `shadow-lg`，让菜单/弹窗明确脱离画板

### 3.2 动态与节奏

- **进入动画**：页面内容用极短的 fade + 轻微位移（150–250ms）"浮现"
- **数据动画**：数字变化时用缓动过渡，让用户感受到"变化的过程"
- **交互反馈**：按钮点击的轻微 scale、切换的 spring 弹性、拖拽的跟随感
- **Stagger**：列表项依次进入（间隔 30–50ms），创造"一个接一个铺开"的节奏感

动效的原则：**让界面感觉"活着"，但不让它"动个不停"。** 动效在用户触发交互时出现，不自顾自地表演。

### 3.3 材质与质感

- **玻璃质感**：仅用于 L3 悬浮层——`backdrop-blur` + 半透明背景暗示"浮在内容上方"
- **氛围色**：不承载信息语义，只营造空间感和情绪——工作室里的窗外光线。极淡，与 stone 色系和谐，每个页面最多一种氛围基调

> 更多材质手法见《美学规范补充》。

## 4. 设计哲学与底线

### 设计哲学

四条思维方向，帮助在任何 UI 决策中找到"恰到好处"的中间态：

**语义降维** — 表达状态时，优先用最轻量的手段。字色字重能解决的，不升级到背景贴片；背景贴片能解决的，不升级到边框阴影。页面上的"有形重物"越少，高级感越强。

**容器克制** — 静态布局的容器嵌套控制在两层以内（L0 画框 + L1 画板）。所有控件和内容都是画板上的"轻量居民"，直接印在白纸上或沉入浅槽里。

**功能域同构** — UI 的物理排布忠实反映功能关系。同一功能组内的元素靠紧密间距自然成组；不同功能域之间用微隔断建立清晰的秩序边界。

**中庸微调** — 当某个结构显得累赘时，答案不是"全部删光变成裸文本"，而是找到最轻量的视觉语言来建立恰到好处的秩序。在"加厚"和"剥离"之间，始终存在一个优雅的中间态。

### 底线（3 条）

1. **主操作显而易见**：每屏的主操作不需要找，它就在那里。
2. **可交互元素看起来可交互**：按钮像按钮，链接像链接，可点的行看起来可以点。
3. **全站风格一致**：用户从任何页面跳到另一个页面，感受到的是同一个产品。

### 挑战条款

如果你认为某条规范在特定场景下会让界面更丑或与更重要的目标冲突——可以自行采用更优方案，附一句话说明理由即可。

**优先级**：设计灵魂（整体感受）> 视觉基因（一致性）> 深度工具箱（细节手法）> 底线（可用性）

如果规范没有覆盖某个维度，回到"设计灵魂"的画面——那个温暖的工作室——问自己：这个决策放在那个空间里，违和吗？
