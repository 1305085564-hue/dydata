# DYData 顶尖产品美学宪法

适用：阿禅名下所有项目（数据分析 / 导粉中心 / AI工作站 / 个人成长）

> **前言：创作权移交声明**
> 本规范不再是一本“防止把页面做丑的防御性说明书”，而是**“激发顶尖设计的哲学大纲”**。
> 作为主导本项目的 AI 设计师，你被完全信任去做审美判断。本规范定义的是产品气质，旨在用正向、富有启发性的设计哲学引导你的创作。
>
> **使用方式**：规范中**色板、图标系统、核心气质（去黑底、单色调仪器感）**为硬标准；其余条目多为**默认推荐与正向引导**。在保持风格一致的前提下，AI 可根据具体场景破格，但需清楚知道为什么破格。

## 〇、创作权宣言

你被信任做判断。

这份规范定义"什么是好品味的下限"，但**好品味的上限由你来探索**。

每次动手设计一个具体页面前，先用一句话回答：
- 这一页的用户在做什么？
- 这一页最需要被看见的信息是什么？
- 我准备让什么变成这一页的"主角"？

答完再开始。

每页可以有 1-2 处破格——破格必须是**为这一页的灵魂服务**，不是为了不一样而不一样。规范在为破格让路这件事上是允许的，前提是其余地方更克制。

**破格的边界**：
- 破格是局部的戏剧性，不动 layout 属性（width / height / margin / padding）。视觉张力靠颜色、字号、间距、阴影、位移完成
- 同一页面只允许一处主角破格，其余视觉块保持静默克制——并发破格会让"主角"消失
- 破格是为了被看见，不是为了被察觉

## 一、核心设计哲学

1. **信息即界面 (Typography & Content as Interface)**
   设计 the 本质是内容的呈现。优先依靠极具张力的**留白 (Whitespace)**、精准的**字号对比**和**字重呼吸**来建立层级，以轻盈的结构呼吸替代线框与底色的堆叠。
   善用毛玻璃 (Glassmorphism) 和极细边框来提升质感，建立自然的物理空间进深。具体色值、字号、间距见第二章。

2. **克制的张力 (Controlled Vibrancy & Monochromatic Kinetic)**
   默认状态应保持高级的“纯灰阶质感 (Monochromatic)”，让大面积的平静灰色衬托出信息的纯粹。
   颜色极其昂贵。彩色（如暖橙、琥珀、矿物绿）应当作为**信号 (Signals)** 或**情绪奖赏 (Dopamine Rewards)** 出现。它们亮起时，周围应有充足的留白包裹，以确保绝对的统治力与惊喜感。

3. **物理与流体感 (Physicality & Fluid Dynamics)**
   界面是“活着”的。避免僵硬的线性动画。拥抱物理世界的反馈：按压时的弹簧阻尼回弹 (Spring)、数据加载时的呼吸感、卡片流转时的流体布局 (Layout Animation)。
   运用动效的目的：一是消除心智断层，二是提供“刚刚好的爽感”。
   **约束**：默认过渡仍使用 `150ms cubic-bezier(0.4,0,0.2,1)`；Spring 与 Layout Animation 仅用于卡片流转、入场、焦点切换等特殊场景，不用于日常 hover。

4. **一页一魂 (Declarative Privilege)**
   每页设置一个绝对的视觉主角（可以是一块质感突出的深色数据卡片，也可以是全屏唯一的墨黑跳出字体）。
   让主角尖锐跳出，其余元素保持极致的静默，以此建立清晰的第一眼心流。对于多任务工作面板，亦可通过状态发光颗粒或微弱渐变光环来进行轻量级的多焦点引导。

5. **破格的特权 (The Right to Break Rules)**
   规范是底线，品味体现在克制的破格中。
   作为 AI 顶尖设计师，你有权在任何你认为“这里需要灵魂”的地方，打破上述任何规则。前提是你必须清楚地知道为什么要破格，且保证全局审美的和谐统一。

## 二、视觉语言

### 2.1 底色

| 层级 | 色值 | 用途 |
|---|---|---|
| L0 | `#F0F0F1` | 页面背景 |
| L1 | `#FAFAFB` zinc-50 | 卡片、主面板 |
| L1.5 | `bg-zinc-100/50` | 卡片内次级盆地（包裹高密度表单组/小工具栏，不配边框） |
| L2 | `#FFFFFF` white | 焦点区、Modal、Focus 输入、激活 Tab |

白卡（bg-white + rounded + border-zinc）默认附 `--shadow-card`（极淡阴影）。

### 2.2 文字灰阶

| 用途 | 色值 |
|---|---|
| 主文字 | `text-zinc-800` |
| 次要 | `text-zinc-500` |
| 元数据/空态/单位 | `text-zinc-400` |
| 分隔/边框 | `zinc-200`（优先用留白） |

**墨黑特权（zinc-950）**：仅用于 Display 大数字、AI 核心结论这类"绝对确定性"信息，拉开画面最高对比层级。其余文字默认 `zinc-800`。

### 2.3 四语义色

| 语义 | 色值 | 场景 |
|---|---|---|
| 主 CTA | `#D97757` 暖橙 | 全站唯一主按钮 |
| 待处理 | `#D99E55` 琥珀 | 授权、生成中、提醒 |
| 成功 | `#6FAA7D` 矿物绿 | 在线、通过、复制成功 |
| 异常 | `#C9604D` 矿物红 | 失败、驳回、违规 |

CTA Hover → `#C96442`（仅底色加深，无阴影无位移）
动作用暖橙，状态用矿物色，Primary 默认只有暖橙。

**数据可视化色板**：`#D97757` 主橙 · `#E3B15C` 琥珀 · `#7FA97E` 绿 · `#8AA8C7` 石青 · `#B08BB8` 雾紫

**图表配方分两种**：
- **单维度叙事**（hero 单图表）：焦点 1 条暖橙 + 非焦点全灰（zinc-300），焦点下方允许垂直渐变 3%→0%
- **多维度对比**（多账号/多组绩效并陈）：色板全开，但网格线透明度降至 20%，靠"退后感"维持仪器气质

Tooltip：白底 + zinc-200 边 + 极淡阴影，数值 `tabular-nums`。

### 2.4 彩色形态

彩色以**点 / 线 / 按钮**三种形态为主，每屏总量克制，并被留白包围。

**推荐形态**：
- 点：状态点、列表项前置标识（小尺寸实心圆）
- 线：细线 outline 描边、左侧导轨
- 按钮：主 CTA 实色暖橙
- 表单控件：复选框/单选框选中态暖橙 + 白勾
- 数据可视化：图表内部用色板
- **墨水晕染态**（新）：状态徽章/Chip 允许 `bg-[#语义色]/0.04~0.06` 极淡面状底色，模拟纸张浸墨，配合 outline + 状态点使用

**默认不使用**：
- 标准浅族底色（emerald-50/100/200、amber、rose、sky 等饱和浅色）
- 角色徽章/权限标签的彩色塑料感背景
- 彩色进度条（默认灰，仅完成/达标态点睛）

`emerald`/`amber`/`rose`/`sky` 四族**优先用于数据可视化图表内部**；UI 组件中若确需使用，必须保持极低饱和度且被大量留白包围。

### 2.5 复合阴影与空间深度（Elevation & Shadows）

阴影是稀有资源，默认不加。在需要明确空间层级的场景下，可使用多层、极柔和的复合阴影（例如大范围、低透明度的扩散阴影配合极细的内发光/白边）来打造物理悬浮感与层次进深。

**推荐场景**：悬浮层（Dropdown/Tooltip/Popover）、独立可点击卡片 hover、居中独立卡（登录/Modal）、命令面板。
**默认不用于**：普通列表行、表格、表单控件、导航项——这些场景优先用底色差和留白建立层级。

### 2.6 微妙渐变与材质光泽（Gradients & Materials）

在合理的焦点场景下，允许使用极低饱和度、极低透明度（< 8%）的网格渐变 (Mesh Gradients)、线性光泽或模糊背景 (Backdrop Blur) 来模拟真实物理介质的微光。

**推荐场景**：页面级 hero 主角区、核心数据卡片的局部高光、命令面板。
**默认不用于**：数据表格、列表行、表单控件、普通按钮；禁止高饱和渐变。

### 2.7 图标系统（Iconography）

全站默认使用 **Lucide Icons**，保持线形风格一致。

| 属性 | 规范 |
|---|---|
| 线条粗细 | 统一 `1.5px` stroke width，与 `font-medium` 字重视觉对齐 |
| 默认色 | `zinc-500` |
| Hover 色 | `zinc-800` |
| 禁用态 | `zinc-300` |
| 彩色图标 | 优先用于状态点场景（矿物绿/矿物红/琥珀），不做主图标彩色 |

**尺寸与字号绑定**：

| 字号档位 | 图标尺寸 | 场景 |
|---|---|---|
| Eyebrow 10px | 12px | 小眉题、紧凑徽章 |
| Body 13px / Body-L 14px | 16px | 默认正文、按钮、列表行 |
| Title 18px / Subtitle 16px | 20px | 卡片标题、弹窗标题 |
| Section 24px / Display 32px | 24px | 模块标题、空状态主图标 |

**图标切换动效**：优先使用 `opacity` + `scale-95` 微动（`150ms ease`），避免旋转或弹跳。

### 2.8 交互状态色（Interaction States）

四语义色（2.3）用于表达业务语义；以下状态色用于表达交互反馈，二者不混用。

| 状态 | 色值 | 场景 |
|---|---|---|
| 选中/激活 | `#8AA8C7` 石青 | 当前 Tab、选中列表项、单选/复选选中态 |
| Hover 底色 | `zinc-100` | 列表行、表格行、导航项 |
| Focus Ring | `zinc-950/5` | 输入框、按钮、可聚焦组件 |
| 拖拽中 | `zinc-950/5` + `scale-[1.02]` | 卡片/列表项拖拽状态 |
| 禁用 | `zinc-300` 图标 / `zinc-400` 文字 | 禁用按钮、只读表单 |

**规则**：
- 选中态**默认使用石青**，与暖橙 CTA 形成互补，避免全站只有暖色调。
- 选中态优先用底色（`bg-[#8AA8C7]/10`）+ 文字色加深，而不是给文字本身上色。
- Focus Ring **默认使用** `zinc-950/5`，避免与 CTA 暖橙语义混淆。

## 三、形态与节奏

### 3.1 圆角

| 层级 | 圆角 | 用途 |
|---|---|---|
| 外层容器 | `rounded-2xl` | 页面级主面板、大卡片（封顶） |
| 内层卡片 | `rounded-xl` | 数据块、模块卡片、消息气泡 |
| 按钮/标签/输入框 | `rounded-lg` | 默认形态 |
| 状态点/头像/数字徽章 | `rounded-full` | 正圆元素、计数、单字状态 |

**按钮形态**：
- 默认 `rounded-lg`：后台日常按钮、表单、Tab、分页器、Modal 主次按钮、列表行内按钮
- `rounded-full` 胶囊：登录/注册主 CTA、落地页核心 CTA、空状态行动召唤、营销卡核心按钮
- 同一信息层级内**建议**统一形状；跨层级允许并存

聊天气泡允许非对称圆角（`rounded-tr-[4px]` 等行业惯例）。

**精密嵌套卡片圆角**：外围主容器建议使用大圆角（`rounded-2xl`），而内部承载数据的图表卡、指标卡，则等比收窄为 `rounded-lg` 或 `rounded-md`，以维持“精密仪器”的严谨切割感。

### 3.2 边框温度

| 温度 | 色值 | 用途 |
|---|---|---|
| 极冷 | `zinc-200` | 分隔线、卡片边界 |
| 微温 | `zinc-300` | 可交互元素默认边框 |
| 暖橙 | `#D97757` | 激活/选中持久态轮廓 |

Focus Ring：`focus:ring-1 focus:ring-zinc-950/5`（默认不用暖橙做临时聚焦）

### 3.3 字号档位

以下档位为**默认基准**，允许根据具体场景进行 1–2 档的破格调整。

| 档位 | 字号 | 字重 | 用途 |
|---|---|---|---|
| Display | `text-[32px]` | `font-semibold` | 核心数据大数，建议每屏唯一 |
| Section | `text-[24px]` | `font-semibold` | 模块标题 |
| Title | `text-[18px]` | `font-medium` | 弹窗标题 |
| Subtitle | `text-[16px] leading-[1.5]` | `font-semibold` | 中型卡片头部、独立表单组标题（弥补 14↔18 断层） |
| Body-L | `text-[14px]` | 默认 | 中文密集列表 |
| Body | `text-[13px]` | 默认 | 默认正文 |
| Label | `text-[13px]` | `font-medium` | 表头、导航、表单标签 |
| Caption | `text-[12px]` | 默认 | 脚注 |
| Eyebrow | `text-[10px]` | `font-medium` | 小眉题（`uppercase tracking-[0.25em]`） |

字重**默认最高** `font-semibold`，Display 级大数字在需要极强视觉冲击时可破格使用 `font-bold`。
数字默认加 `tabular-nums`；`font-mono` 优先用于大数字（Display/Section）。
Display 用于核心数据焦点，**建议**每屏一处，需留白包围。

**数字与单位分层**：数值 `zinc-800 font-semibold` + 单位 `zinc-400 font-medium`（单位字号比数值小一档）

### 3.4 间距

档位：4 / 8 / 16 / 24 / 32 / 40px（XS / S / M / L / XL / 2XL）

**节奏戏剧性**：同一页面内允许出现"极限紧凑"与"极端空旷"并置——核心交互组件（如 AI 输入输出框、控制组）可压缩到 XS/S，进入元数据/页脚时突然拉到 XL/2XL。**用反差打破机械的平均主义**，是这套规范最重要的呼吸节拍。

紧凑交互元素（导航项、shadcn 内部）允许 `p-3`/`px-3`。

## 四、形态语言

### 4.1 输入框（Soft Fill）

- 默认：`bg-zinc-100/70 border-transparent text-zinc-800`
- Focus：`bg-white border-zinc-200 shadow-sm focus:ring-1 focus:ring-zinc-950/5`

### 4.2 动效

界面动效按场景分层，避免「日常交互也用 spring」或「卡片流转用线性」的错位。

| 场景 | 动效类型 | 参数 | 理由 |
|---|---|---|---|
| 日常交互（Hover/Focus/按钮） | Ease | `150ms cubic-bezier(0.4,0,0.2,1)` | 稳定、不打扰，不动 layout 属性 |
| 卡片流转（看板拖动、列表排序、状态切换） | Spring | `type: "spring", stiffness: 300, damping: 24` | 物理真实感，消除硬切 |
| 页面入场 / 数据加载 | Stagger 淡入 | `opacity 0→1, translateY 8px→0, 150ms` | 消除心智断层 |
| 按压反馈（Active） | 微缩回弹 | `active:scale-[0.98]` + `150ms ease` | 比 `translateY` 更稳定，不触发 layout |

```css
transition: 150ms cubic-bezier(0.4,0,0.2,1);
```

允许动画属性：`background-color` / `color` / `border-color` / `box-shadow` / `transform` / `opacity`
不动 layout 属性（`width` / `height` / `margin` / `padding`），不弹簧过冲。

**关键约束**：
- 卡片 hover 的微浮若使用 `translateY(-2px)` + `shadow`，**必须确保该卡片不在 layout 动画流中**，否则 transform 冲突会导致抖动。
- Spring 与 Layout Animation 仅用于卡片流转、入场、焦点切换等特殊场景，不用于日常 hover。
- `prefers-reduced-motion` 下全部关闭非必要过渡。

### 4.3 Hover

**静默场景**（仅底色变化）：列表行、表格行、Tab、导航项、次级按钮、工具栏按钮

**微浮场景**（轻微上浮 + 极淡阴影）：独立可点击卡片（数据卡、诊断卡、营销卡、登录页特性卡）

按钮/CTA/输入框 hover 不加延迟，列表/导航可加 50ms 延迟。

### 4.4 Active 回弹

所有可点击元素需有 `active:translate-y-0` 物理回弹；transition 包含 `transform`。

### 4.5 阴影

阴影是稀有资源，默认不加。允许场景：
- 主 CTA Hover 极淡暖橙阴影
- 独立可点击卡 Hover 极淡冷阴影
- 居中独立卡（登录/Modal）柔光阴影
- 悬浮层（Dropdown/Tooltip）柔光阴影

不在以上场景默认不加阴影。需要更强烈的层级进深时，参见 2.5 复合阴影与空间深度。

### 4.6 加载

- 极快响应：直出，不加 loading
- 较慢响应：优先使用骨架屏（zinc-100 矩形，结构与最终一致），淡入替换
- spinner 仅在无法预估内容结构或极小区域使用

### 4.7 Stagger 淡入

列表允许 stagger 淡入（每项轻微错峰）；列表项过多时关闭 stagger，直出。

### 4.8 空状态

允许有人格的飞地。整体克制（≤120px），默认方向：
- 极淡灰虚线圆轨 + 十字辅助线（仪器卡尺感）
- 中心一个微妙几何体（小圆点、三角、弧线均可），允许微动效（呼吸/浮动/路径描绘）
- 文案克制诗意：标题 zinc-500、副文 zinc-400

**默认不使用**彩色插画/彩色图标/叙事插图；品牌营销活动或特殊情感场景可破格，但需保持克制。同一产品内不同模块的空状态可有不同几何体，避免审美疲劳。

### 4.9 列表与瀑布流

- **列表/表格**：默认纯白底，hover 当前行 `bg-zinc-100`；优先用留白建立节奏，斑马纹与边框切分仅在极端密集场景下谨慎使用
- **卡片底部元数据**：优先用留白分隔（`mt-6 pt-6`），默认不用 `border-top`
- **轻导航/历史侧栏**：单行行块（`rounded-md px-3 py-2`），不做成卡片；当前项配方见 5.1
- **瀑布流/网格卡片**：默认 `bg-white rounded-xl` + 卡片间留白，不加默认外框；hover 极淡阴影 + 轻微上浮；状态卡优先用 Eyebrow 小标 + 状态点表达，导轨留给线性列表

**非对称排版**：核心高光内容（爆款话术、今日核心异动）允许 `col-span-2` 或纵跨 2 行，破格大块与精密小块并置，营造杂志感。

### 4.9b 卡片流转三件套（看板/状态栏丝滑滑动）

列表项在分组/状态列之间移动时，应「飞」过去，避免硬切。三件套推荐同时满足：

- **共享布局**：每张卡 `<motion.div layout layoutId="{域}-{id}">`，外层列表包 `<AnimatePresence mode="popLayout" initial={false}>`，并给列容器加 `layout`。同一 id 在大卡/小卡间切换时是「长大/缩小」而非闪现。
- **入场 spring**：`transition: { type: "spring", stiffness: 300, damping: 24 }`；退场 `{ opacity: 0, scale: 0.96, duration: 0.15 }`。优先避免线性 `y` 位移。
- **撑开式操作条**：hover 露出的操作条用 `grid-template-rows: 0fr → 1fr` 过渡撑开，把下方内容推开，**避免**绝对定位浮层盖住内容。

避坑：layout 动画靠 `transform` 做位移，所以 hover **只改 `box-shadow`/`border`，尽量不碰 `transform`/`translateY`**，否则两套 transform 打架抖动。`prefers-reduced-motion` 下全部关掉过渡。

### 4.10 状态语义

| 态 | 填充 | 场景 | 动效 |
|---|---|---|---|
| Critical | `#C9604D` | 失败、驳回、严重告警 | 静态 |
| Active | `#D97757` | 当前步、进行中 | 单色呼吸（仅当前焦点区域） |
| Done | `#6FAA7D` | 完成、通过、在线 | 静态 |
| Idle | `zinc-200` | 未开始、无状态 | 静态 |

默认不用 `animate-pulse`，不用渐变填充。
轻导航/历史列表前置标记用空心圆表示"当前焦点"，实心圆优先用于成功/异常/进行中等状态信号。
同视觉块内 Critical + Active 共存时，Active 降权静态，保持 Critical 唯一跳出点。

### 4.11 左侧 2px 导轨

导轨是线性分割工具，不是卡片装饰。它的隐喻是"书架侧边的标签"——只在单列线性结构里帮助用户定位当前项。

**适合用**：列表行当前项、轻导航、分组标题、横幅告警。
**不适合用**：网格/瀑布流卡片内部、多卡并排场景。

不用导轨时，状态按这个顺序选：Eyebrow 小标 / 右上角徽章 → 状态点 + 文字色 → 整卡灰阶/透明度降级 → 墨水晕染 badge。

导轨色：Critical `#C9604D` / Pending `#D99E55` / Done `#6FAA7D` / Primary `#D97757`
导轨不与彩色背景面同时使用，不渐变，同容器内只一条。

### 4.12 角色徽章

outline + 左状态点，默认不使用彩色浅底 chip 或黑底填充：
- 创始人：暖橙描边 + 暖橙点
- 管理员：灰描边 + 琥珀点
- 组长：灰描边 + 矿物绿点
- 成员：灰描边 + 无点

### 4.13 深色使用

按钮 / 卡片 / chip / 徽章 / 标签 / 分页器 / Tab 默认不使用 `bg-zinc-900` / `bg-zinc-950` / `bg-black`。

以下语义场景保留深色：
- AI 对话用户气泡 `bg-zinc-950 text-white`
- Tooltip `bg-zinc-900 text-white`
- 代码块 `<pre>` `bg-zinc-900`
- 模态遮罩 `bg-zinc-950/20`

Logo 方块/头像圆：`bg-zinc-800`。
字色默认 `text-zinc-800`（除墨黑特权场景）。

### 4.14 多巴胺反馈闭环（The Dopamine Loop）

任何重要的用户节点，推荐给予极具质感的**克制奖赏**。
手法包括但不限于：极其微小的局部背景墨水晕染（如 Mineral Green 0.04 透明度）、界面数据的静默且丝滑重载、物理弹跳 (Spring Scale)。

**约束**：Spring Scale 仅用于独立按钮/卡片，不用于数据表格单元格或密集列表项；墨水晕染遵循 2.3 四语义色与 0.04~0.06 透明度上限。

### 4.15 渐进式操作显影（Progressive Disclosure）

用户的核心工作流应是极致降噪的。对于高频但次要的行级操作，推荐采用**“半透明弱化”**机制（例如默认 `opacity-35`，Hover 或 Focus 时温和过渡到 `opacity-100` 并展现色彩）。

**约束**：仅用于行级次要操作；必须保证键盘聚焦（`:focus-visible`）和移动触屏可访问性；主 CTA 与关键操作不得使用。

### 4.16 图层独立的微合成动效（Independent Layer Scaling）

为了维持大量复杂数据的绝对稳定，推荐将物理位移限制在独立的渲染图层（Composite Layer）上。
例如：对于网格单元（如日历矩阵方块）或图标按钮，使用 `scale-110/120` 或微弱阴影来形成原位微膨胀反馈，既能提供灵敏的手势响应，又不会引起周围 DOM 布局的抖动（Layout Shift）。

## 五、信息架构

### 5.1 导航与侧边栏

- **浏览态**：侧边栏 L0 底色，与主画布 L1 形成层级差，1px 边框区隔
- **深度工作态**：侧边栏可隐去或缩为图标栏，主画布扩展全宽
- 导航项：单行 + 左状态点 + 右时间/数量，hover `bg-zinc-100`（无位移无阴影）

**当前项分两类，配方不混用**：

| 类型 | 场景 | 当前项配方 |
|---|---|---|
| 主导航 | 顶部导航 / 主侧边栏（/dashboard、/admin 这类全局路由项） | 左侧细导轨 + `bg-zinc-100` 底色高亮 |
| 轻导航 / 历史侧栏 | 对话历史、检索结果、动态内容列表 | `bg-zinc-200/95` 行底 + 6px 空心圆，最右侧可点缀 4px 暖橙静态微点表达"卡入" |

两类都默认不用黑底填充，不叠加描边/阴影/胶囊标签。

### 5.2 面包屑

- 所有非首页页面应当存在
- 小字号灰色，分隔符 `>`
- 当前页无链接，纯文字 `zinc-800`

### 5.3 抽屉

- 视觉区隔：边框 + 底色差 + 全局微阴影
- 宽度：默认 `w-96`，复杂表单 `w-[480px]`
- 层级：不超过 2 层（页面→抽屉），超过时页面跳转
- 关闭：点击遮罩 / 关闭按钮 / `Esc`

### 5.4 密度档位

| 档位 | 场景 |
|---|---|
| 默认 Comfort | 配置、浏览、常规表单 |
| 紧凑 Compact | 数据表格、日志流、代码列表（间距减半，字号不变） |
| 沉浸 Immersive | 写作、代码编辑、深度分析（隐藏导航） |

切换：用户手动 / 页面类型自动应用。

### 5.5 上下文保持

- 任何操作不导致用户空间迷失
- 长列表提供当前位置指示器（粘性表头、滚动章节标记、顶部进度）
- 模态/抽屉层级尽量控制在 2 层以内，超过时优先页面跳转
- 返回逻辑始终可见

### 5.6 命令面板（⌘K）

- 触发：`⌘K` / `Ctrl+K`
- 容器：磨砂半透明白底 + 浅边 + 柔光阴影（全站最主要的磨砂使用场景，不默认外溢到抽屉/Tooltip/侧栏）
- 搜索框：Soft Fill
- 高亮：`bg-zinc-100` + 字重变化，默认不用彩色背景
- 快捷键：`<kbd>` 样式淡灰底 + 浅边 + mono 字体

### 5.7 分割视图

- 分割线：`w-px bg-zinc-200`，hover `bg-zinc-300`，可拖拽
- 左右面板可调比例，最小宽度 240px
- 默认不用阴影分割

### 5.8 流式宽画布与聚焦视界（Responsive High-Density Spacing）

在多维看板、历史趋势、排行榜等重数据模块中，鼓励使用展开的 `max-w-7xl` 或流式全宽布局，让数据获得充足的横向可视宽度，避免滚动条泛滥。
在单表单、核心填报等高沉浸任务中，采用比例收窄的视界，利用负空间（Negative Space）帮助用户屏蔽外界噪音。

### 5.9 控制台的实体感锚定（Solid Console Anchors）

全局筛选器、视图切换等高频控制台推荐采用固定且稳定的视觉设计，随时为用户提供空间定位感。
可以通过毛玻璃背景（`backdrop-blur-md`）配合轻微的底边分割线，使其稳定且有浮空感地漂浮在工作流上方。

**约束**：仅全局/高频控制台可用；普通卡片、抽屉、Tooltip 不套用此效果。
