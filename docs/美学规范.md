# DYData 顶尖产品美学宪法

适用：阿禅名下所有项目（数据分析 / 导粉中心 / AI工作站 / 个人成长）

> **前言：创作权移交声明**
> 本规范不再是一本“防止把页面做丑的防御性说明书”，而是**“激发顶尖设计的哲学大纲”**。
> 作为主导本项目的 AI 设计师，你被完全信任去做审美判断。本规范定义的是产品气质，而不是捆绑你双手的 CSS 代码片段。

---

## 一、 核心设计哲学 (The Core Philosophies)

1. **信息即界面 (Typography & Content as Interface)**
   - 设计的本质是内容的呈现。优先依靠极具张力的**留白 (Whitespace)**、精准的**字号对比**和**字重呼吸**来建立层级，摒弃滥用线框和无意义的底色堆叠。
   - 知道何时使用毛玻璃 (Glassmorphism) 和极细边框来提升质感，但不依赖它们来区分结构。
2. **克制的张力 (Controlled Vibrancy & Monochromatic Kinetic)**
   - 默认状态应保持高级的“纯灰阶质感 (Monochromatic)”。
   - 颜色极其昂贵。彩色（如暖橙、琥珀、矿物绿）必须作为**信号 (Signals)** 或**情绪奖赏 (Dopamine Rewards)** 出现，而非廉价的装饰。它们出现时，必须被充足的留白包裹，以确保绝对的统治力。
3. **物理与流体感 (Physicality & Fluid Dynamics)**
   - 界面是“活着”的。避免僵硬的线性动画。拥抱物理世界的反馈：按压时的弹簧阻尼回弹 (Spring)、数据加载时的呼吸感、卡片流转时的流体布局 (Layout Animation)。
   - 运用动效的目的：一是消除心智断层，二是提供“刚刚好的爽感”。
4. **一页一魂 (Declarative Privilege)**
   - 每页只能有一个绝对的主角（可能是唯一的大实色按钮、可能是一块破格的深色数据卡片，也可能是全屏唯一的墨黑跳出字体）。
   - 让主角尖锐跳出，其余元素保持极致的静默。

## 二、 视觉解禁与创造力空间 (Creative Freedoms)

为了达到顶尖硅谷 SaaS 的视觉标准，以下设计手法被**完全解禁**，由 AI 根据具体场景进行自由调配：

### 1. 复合阴影与空间深度 (Elevation & Shadows)
- 解除原有的“默认不加阴影”限制。
- 鼓励使用多层、极柔和的复合阴影（例如大范围、低透明度的扩散阴影配合极细的内发光/白边）来打造出真实的物理悬浮感。

### 2. 深色反差特权 (Dark Mode Accentuation)
- 解除“禁用黑底”的限制。
- 鼓励在浅色主画布中，植入高质感的**反色卡片**（Dark in Light）。例如：Pro 功能提示、全屏最高优的数据大盘、AI 深度思考模块。用极端的明暗反差来拉升页面的高级感。

### 3. 微妙渐变与材质光泽 (Gradients & Materials)
- 解除“禁用渐变填充”的死板规定。
- 支持在合理的场景运用极低饱和度、极低透明度（< 8%）的网格渐变 (Mesh Gradients)、线性光泽或模糊背景 (Backdrop Blur)。

### 4. 情绪化空状态 (Empathetic Empty States)
- 解除“只能用辅助线和极简几何体”的冷酷约束。
- 允许空状态具备“人情味”。在适当的上下文中（如激励打卡区），可以使用高品质的 3D 微图标、轻拟物元素或极其克制的单色/微渐变插图来召唤用户的行动 (CTA)。

## 三、 动态交互准则 (Interaction Patterns)

1. **多巴胺反馈闭环 (The Dopamine Loop)**
   - 任何重要的用户节点，必须给予极具质感的**克制奖赏**。
   - 手法包括但不限于：极其微小的局部背景墨水晕染（如 Mineral Green 0.04 透明度）、界面数据的静默且丝滑重载、物理弹跳 (Spring Scale)。
2. **沉浸与降噪 (Zen Mode & Noise Reduction)**
   - 用户的核心工作流（如表单填报区）必须是极致降噪的。对于当前无需操作的信息，应收纳至浮层中，确保当前视角的绝对纯粹。
3. **破格的特权 (The Right to Break Rules)**
   - 规范是底线，品味体现在克制的破格中。
   - 作为 AI 顶尖设计师，你有权在任何你认为“这里需要灵魂”的地方，打破上述任何规则。前提是你必须清楚地知道为什么要破格，且保证全局审美的和谐统一。

## 四、 针对 B 端数据产品的审美边界 (The B-End Boundaries)

顶级美学必须服从业务场景。在 DYData（作为高密度数据消费中台）中，严禁盲目照搬 C 端效率工具（如 Craft）的设计，必须守住以下边界：

1. **拒绝盲目的“全局宽留白”**
   - 沉浸式单任务（如今日填报）可以使用窄体留白聚焦。
   - 但**数据看板、历史趋势、排行榜**等高密度模块，必须使用展开的 `max-w-7xl` 或流式全宽布局，绝对禁止用 20% 的大留白挤压数据表，避免横向滚动条泛滥。
2. **拒绝滥用“悬浮与物理位移”**
   - 物理位移（如 `translate-y-1` 的弹簧上浮）**仅限具有明确“行为召唤（CTA）”的独立操作卡**。
   - 对于静态数据卡片、图表容器、数据列表，Hover 时只允许极其克制的背景色阶变化，绝对禁止位移，避免数据密集区出现“蹦床效应”。
3. **拒绝“自动隐藏工具栏”**
   - 核心控制台（如全局账号切换、日期选择、数据视图切换）必须绝对固定且随时可见。
   - 绝不能为了视觉上的极简，将核心筛选器做成悬停出现或滚动隐藏，避免用户丧失数据空间的定位感。
4. **收紧卡片圆角**
   - 外围主容器可保留大圆角（`2xl`）。
   - 但内部承载数据的图表卡、指标卡，必须收紧为 `rounded-lg` 或 `rounded-md`，以维持“精密仪器”的严谨切割感。
5. **警惕“排版即界面”在数据表格中的灾难**
   - 纯靠字号和字重划分区块在文字流中可行，但在高密度数据表中会导致用户视线串行。
   - 在数据列表中，依然必须使用极细、极低对比度的边框（`border-zinc-200/50`）以及轻微的斑马纹底色来做物理区隔。
