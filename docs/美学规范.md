# 美学规范

适用：阿禅名下所有项目（数据分析/话术库/团队中控/AI工作站）

---

## 〇、创作权宣言

你被信任做判断。

这份规范定义"什么是好品味的下限"，但**好品味的上限由你来探索**。

每次动手设计一个具体页面前，先用一句话回答：
- 这一页的用户在做什么？
- 这一页最需要被看见的信息是什么？
- 我准备让什么变成这一页的"主角"？

答完再开始。

每页可以有 1-2 处破格——破格必须是**为这一页的灵魂服务**，不是为了不一样而不一样。规范在为破格让路这件事上是允许的，前提是其余地方更克制。

**破格的边界**：
- 破格是局部的戏剧性，不动 layout 属性（width / height / margin / padding）。视觉张力靠颜色、字号、间距、阴影、位移完成
- 同一页面只允许一处主角破格，其余视觉块保持静默克制——并发破格会让"主角"消失
- 破格是为了被看见，不是为了被察觉

## 一、五哲学

1. **Typography as Interface** — 层级靠留白、字距、字重建立，不靠线框/底色堆叠
2. **Monochromatic Kinetic** — 日常纯灰阶，质感靠灰度变化 + 物理位移（微浮/回弹）
3. **Controlled Vibrancy** — 彩色是信号非装饰，克制、被留白包围
4. **Declarative Privilege** — 强烈视觉元素（pill/柔光阴影/大实色按钮）是特权，一屏一处
5. **底线之上鼓励破格** — 字号/四色/灰阶/深色限制是底线；底线之上鼓励**克制的惊喜**：间距可松紧反差、焦点文字可墨黑跳出、状态徽章可墨水晕染、核心数据可破格大块

## 二、视觉语言

### 2.1 底色

| 层级 | 色值 | 用途 |
|---|---|---|
| L0 | `#F0F0F1` | 页面背景 |
| L1 | `#FAFAFB` zinc-50 | 卡片、主面板 |
| L1.5 | `bg-zinc-100/50` | 卡片内次级盆地（包裹高密度表单组/小工具栏，不配边框） |
| L2 | `#FFFFFF` white | 焦点区、Modal、Focus 输入、激活 Tab |

白卡（bg-white + rounded + border-zinc）默认附 `--shadow-card`（极淡阴影）。

### 2.2 文字灰阶

| 用途 | 色值 |
|---|---|
| 主文字 | `text-zinc-800` |
| 次要 | `text-zinc-500` |
| 元数据/空态/单位 | `text-zinc-400` |
| 分隔/边框 | `zinc-200`（优先用留白） |

**墨黑特权（zinc-950）**：仅用于 Display 大数字、AI 核心结论这类"绝对确定性"信息，拉开画面最高对比层级。其余文字一律 `zinc-800`。

### 2.3 四语义色

| 语义 | 色值 | 场景 |
|---|---|---|
| 主 CTA | `#D97757` 暖橙 | 全站唯一主按钮 |
| 待处理 | `#D99E55` 琥珀 | 授权、生成中、提醒 |
| 成功 | `#6FAA7D` 矿物绿 | 在线、通过、复制成功 |
| 异常 | `#C9604D` 矿物红 | 失败、驳回、违规 |

CTA Hover → `#C96442`（仅底色加深，无阴影无位移）
动作用暖橙，状态用矿物色，Primary 永远只有暖橙。

**数据可视化色板**：`#D97757` 主橙 · `#E3B15C` 琥珀 · `#7FA97E` 绿 · `#8AA8C7` 石青 · `#B08BB8` 雾紫

**图表配方分两种**：
- **单维度叙事**（hero 单图表）：焦点 1 条暖橙 + 非焦点全灰（zinc-300），焦点下方允许垂直渐变 3%→0%
- **多维度对比**（多账号/多组绩效并陈）：色板全开，但网格线透明度降至 20%，靠"退后感"维持仪器气质

Tooltip：白底 + zinc-200 边 + 极淡阴影，数值 `tabular-nums`。

### 2.4 彩色形态

彩色以**点 / 线 / 按钮**三种形态为主，每屏总量克制，必须被留白包围。

**允许的彩色**：
- 点：状态点、列表项前置标识（小尺寸实心圆）
- 线：细线 outline 描边、左侧导轨
- 按钮：主 CTA 实色暖橙
- 表单控件：复选框/单选框选中态暖橙 + 白勾
- 数据可视化：图表内部用色板
- **墨水晕染态**（新）：状态徽章/Chip 允许 `bg-[#语义色]/0.04~0.06` 极淡面状底色，模拟纸张浸墨，配合 outline + 状态点使用

**避免**：
- 标准浅族底色（emerald-50/100/200、amber、rose、sky 等饱和浅色）
- 角色徽章/权限标签的彩色塑料感背景
- 彩色进度条（默认灰，仅完成/达标态点睛）

`emerald`/`amber`/`rose`/`sky` 四族仅限数据可视化图表内部，不用于 UI 组件。

## 三、形态与节奏

### 3.1 圆角

| 层级 | 圆角 | 用途 |
|---|---|---|
| 外层容器 | `rounded-2xl` | 页面级主面板、大卡片（封顶） |
| 内层卡片 | `rounded-xl` | 数据块、模块卡片、消息气泡 |
| 按钮/标签/输入框 | `rounded-lg` | 默认形态 |
| 状态点/头像/数字徽章 | `rounded-full` | 正圆元素、计数、单字状态 |

**按钮形态**：
- 默认 `rounded-lg`：后台日常按钮、表单、Tab、分页器、Modal 主次按钮、列表行内按钮
- `rounded-full` 胶囊：登录/注册主 CTA、落地页核心 CTA、空状态行动召唤、营销卡核心按钮
- 同一信息层级内只能一种形状；跨层级允许并存

聊天气泡允许非对称圆角（`rounded-tr-[4px]` 等行业惯例）。

### 3.2 边框温度

| 温度 | 色值 | 用途 |
|---|---|---|
| 极冷 | `zinc-200` | 分隔线、卡片边界 |
| 微温 | `zinc-300` | 可交互元素默认边框 |
| 暖橙 | `#D97757` | 激活/选中持久态轮廓 |

Focus Ring：`focus:ring-1 focus:ring-zinc-950/5`（不用暖橙做临时聚焦）

### 3.3 字号档位

| 档位 | 字号 | 字重 | 用途 |
|---|---|---|---|
| Display | `text-[32px]` | `font-semibold` | 核心数据大数，每屏唯一 |
| Section | `text-[24px]` | `font-semibold` | 模块标题 |
| Title | `text-[18px]` | `font-medium` | 弹窗标题 |
| Subtitle | `text-[16px] leading-[1.5]` | `font-semibold` | 中型卡片头部、独立表单组标题（弥补 14↔18 断层） |
| Body-L | `text-[14px]` | 默认 | 中文密集列表 |
| Body | `text-[13px]` | 默认 | 默认正文 |
| Label | `text-[13px]` | `font-medium` | 表头、导航、表单标签 |
| Caption | `text-[12px]` | 默认 | 脚注 |
| Eyebrow | `text-[10px]` | `font-medium` | 小眉题（`uppercase tracking-[0.25em]`） |

字重最高 `font-semibold`，不用 `font-bold`/`font-black`。
数字一律加 `tabular-nums`；`font-mono` 仅用于大数字（Display/Section）。
Display 仅用于 hero 主数据，每屏唯一，需留白包围，不用于弹窗内/列表项内。

**数字与单位分层**：数值 `zinc-800 font-semibold` + 单位 `zinc-400 font-medium`（单位字号比数值小一档）

### 3.4 间距

档位：4 / 8 / 16 / 24 / 32 / 40px（XS / S / M / L / XL / 2XL）

**节奏戏剧性**：同一页面内允许出现"极限紧凑"与"极端空旷"并置——核心交互组件（如 AI 输入输出框、控制组）可压缩到 XS/S，进入元数据/页脚时突然拉到 XL/2XL。**用反差打破机械的平均主义**，是这套规范最重要的呼吸节拍。

紧凑交互元素（导航项、shadcn 内部）允许 `p-3`/`px-3`。

## 四、形态语言

### 4.1 输入框（Soft Fill）

- 默认：`bg-zinc-100/70 border-transparent text-zinc-800`
- Focus：`bg-white border-zinc-200 shadow-sm focus:ring-1 focus:ring-zinc-950/5`

### 4.2 动效

```css
transition: 150ms cubic-bezier(0.4,0,0.2,1);
```

允许动画属性：`background-color` / `color` / `border-color` / `box-shadow` / `transform` / `opacity`
不动 layout 属性（`width` / `height` / `margin` / `padding`），不弹簧过冲。

### 4.3 Hover

**静默场景**（仅底色变化）：列表行、表格行、Tab、导航项、次级按钮、工具栏按钮

**微浮场景**（轻微上浮 + 极淡阴影）：独立可点击卡片（数据卡、诊断卡、营销卡、登录页特性卡）

按钮/CTA/输入框 hover 不加延迟，列表/导航可加 50ms 延迟。

### 4.4 Active 回弹

所有可点击元素需有 `active:translate-y-0` 物理回弹；transition 包含 `transform`。

### 4.5 阴影

阴影是稀有资源，默认不加。允许场景：
- 主 CTA Hover 极淡暖橙阴影
- 独立可点击卡 Hover 极淡冷阴影
- 居中独立卡（登录/Modal）柔光阴影
- 悬浮层（Dropdown/Tooltip）柔光阴影

不在以上场景默认不加阴影。

### 4.6 加载

- 极快响应：直出，不加 loading
- 较慢响应：骨架屏（zinc-100 矩形，结构与最终一致），淡入替换
- 不用 spinner

### 4.7 Stagger 淡入

列表允许 stagger 淡入（每项轻微错峰）；列表项过多时关闭 stagger，直出。

### 4.8 空状态

唯一允许有人格的飞地。整体克制（≤120px），核心是：
- 极淡灰虚线圆轨 + 十字辅助线（仪器卡尺感）
- 中心一个微妙几何体（小圆点、三角、弧线均可），允许微动效（呼吸/浮动/路径描绘）
- 文案克制诗意：标题 zinc-500、副文 zinc-400

不用彩色插画/彩色图标/叙事插画。同一产品内不同模块的空状态可有不同几何体，避免审美疲劳。

### 4.9 列表与瀑布流

- **列表/表格**：纯白底，hover 当前行 `bg-zinc-100`；不用斑马纹彩色底，不用边框切分
- **卡片底部元数据**：用留白分隔（`mt-6 pt-6`），不用 `border-top`
- **轻导航/历史侧栏**：单行行块（`rounded-md px-3 py-2`），不做成卡片；当前项配方见 5.1
- **瀑布流/网格卡片**：默认 `bg-white rounded-xl` + 卡片间留白，不加默认外框；hover 极淡阴影 + 轻微上浮；状态卡靠 Eyebrow 小标 + 状态点表达，不用左侧彩色导轨

**非对称排版**：核心高光内容（爆款话术、今日核心异动）允许 `col-span-2` 或纵跨 2 行，破格大块与精密小块并置，营造杂志感。

### 4.9b 卡片流转三件套（看板/状态栏丝滑滑动）

列表项在分组/状态列之间移动时，必须「飞」过去，不能硬切。三件套缺一不可：

- **共享布局**：每张卡 `<motion.div layout layoutId="{域}-{id}">`，外层列表包 `<AnimatePresence mode="popLayout" initial={false}>`，并给列容器加 `layout`。同一 id 在大卡/小卡间切换时是「长大/缩小」而非闪现。
- **入场 spring**：`transition: { type: "spring", stiffness: 300, damping: 24 }`；退场 `{ opacity: 0, scale: 0.96, duration: 0.15 }`。不用线性 `y` 位移。
- **撑开式操作条**：hover 露出的操作条用 `grid-template-rows: 0fr → 1fr` 过渡撑开，把下方内容推开，**不用绝对定位浮层盖住内容**。

避坑：layout 动画靠 `transform` 做位移，所以 hover **只改 `box-shadow`/`border`，绝不碰 `transform`/`translateY`**，否则两套 transform 打架抖动。`prefers-reduced-motion` 下全部关掉过渡。



| 态 | 填充 | 场景 | 动效 |
|---|---|---|---|
| Critical | `#C9604D` | 失败、驳回、严重告警 | 静态 |
| Active | `#D97757` | 当前步、进行中 | 单色呼吸（仅当前焦点区域） |
| Done | `#6FAA7D` | 完成、通过、在线 | 静态 |
| Idle | `zinc-200` | 未开始、无状态 | 静态 |

不用 `animate-pulse`，不用渐变填充。
轻导航/历史列表前置标记用空心圆表示"当前焦点"，实心圆只用于成功/异常/进行中等状态信号。
同视觉块内 Critical + Active 共存时，Active 降权静态，保持 Critical 唯一跳出点。

### 4.11 左侧 2px 导轨

导轨是线性分割工具，不是卡片装饰。它的隐喻是"书架侧边的标签"——只在单列线性结构里帮助用户定位当前项。

**适合用**：列表行当前项、轻导航、分组标题、横幅告警。
**不适合用**：网格/瀑布流卡片内部、多卡并排场景。

不用导轨时，状态按这个顺序选：Eyebrow 小标 / 右上角徽章 → 状态点 + 文字色 → 整卡灰阶/透明度降级 → 墨水晕染 badge。

导轨色：Critical `#C9604D` / Pending `#D99E55` / Done `#6FAA7D` / Primary `#D97757`
导轨不与彩色背景面同时使用，不渐变，同容器内只一条。

### 4.12 角色徽章

outline + 左状态点，不用彩色浅底 chip，不用黑底填充：
- 创始人：暖橙描边 + 暖橙点
- 管理员：灰描边 + 琥珀点
- 组长：灰描边 + 矿物绿点
- 成员：灰描边 + 无点

### 4.13 深色使用

按钮 / 卡片 / chip / 徽章 / 标签 / 分页器 / Tab 不用 `bg-zinc-900` / `bg-zinc-950` / `bg-black`。

仅以下语义场景保留深色：
- AI 对话用户气泡 `bg-zinc-950 text-white`
- Tooltip `bg-zinc-900 text-white`
- 代码块 `<pre>` `bg-zinc-900`
- 模态遮罩 `bg-zinc-950/20`

Logo 方块/头像圆：`bg-zinc-800`。
字色用 `text-zinc-800`（除墨黑特权场景）。

## 五、信息架构

### 5.1 导航与侧边栏

- **浏览态**：侧边栏 L0 底色，与主画布 L1 形成层级差，1px 边框区隔
- **深度工作态**：侧边栏可隐去或缩为图标栏，主画布扩展全宽
- 导航项：单行 + 左状态点 + 右时间/数量，hover `bg-zinc-100`（无位移无阴影）

**当前项分两类，配方不混用**：

| 类型 | 场景 | 当前项配方 |
|---|---|---|
| 主导航 | 顶部导航 / 主侧边栏（/dashboard、/admin 这类全局路由项） | 左侧细导轨 + `bg-zinc-100` 底色高亮 |
| 轻导航 / 历史侧栏 | 对话历史、检索结果、动态内容列表 | `bg-zinc-200/95` 行底 + 6px 空心圆，最右侧可点缀 4px 暖橙静态微点表达"卡入" |

两类都不用黑底填充，不叠加描边/阴影/胶囊标签。

### 5.2 面包屑

- 所有非首页页面应当存在
- 小字号灰色，分隔符 `>`
- 当前页无链接，纯文字 `zinc-800`

### 5.3 抽屉

- 视觉区隔：边框 + 底色差 + 全局微阴影
- 宽度：默认 `w-96`，复杂表单 `w-[480px]`
- 层级：不超过 2 层（页面→抽屉），超过时页面跳转
- 关闭：点击遮罩 / 关闭按钮 / `Esc`

### 5.4 密度档位

| 档位 | 场景 |
|---|---|
| 默认 Comfort | 配置、浏览、常规表单 |
| 紧凑 Compact | 数据表格、日志流、代码列表（间距减半，字号不变） |
| 沉浸 Immersive | 写作、代码编辑、深度分析（隐藏导航） |

切换：用户手动 / 页面类型自动应用。

### 5.5 上下文保持

- 任何操作不导致用户空间迷失
- 长列表提供当前位置指示器（粘性表头、滚动章节标记、顶部进度）
- 模态/抽屉层级不超过 2 层，超过时页面跳转
- 返回逻辑始终可见

### 5.6 命令面板（⌘K）

- 触发：`⌘K` / `Ctrl+K`
- 容器：磨砂半透明白底 + 浅边 + 柔光阴影（全站唯一磨砂特权，不外溢到抽屉/Tooltip/侧栏）
- 搜索框：Soft Fill
- 高亮：`bg-zinc-100` + 字重变化，不用彩色背景
- 快捷键：`<kbd>` 样式淡灰底 + 浅边 + mono 字体

### 5.7 分割视图

- 分割线：`w-px bg-zinc-200`，hover `bg-zinc-300`，可拖拽
- 左右面板可调比例，最小宽度 240px
- 不用阴影分割

## 落地速记

- 主行动按钮 → 暖橙实色 `rounded-lg`
- 次行动按钮 → 白底 outline
- 状态信号 → 状态点 + 墨水晕染（≤0.06 透明度）
- 输入框 → Soft Fill
- 卡片 → 白底 `rounded-xl` + 留白分层
- 列表/表格 → 纯白底，hover `bg-zinc-100`
- 抽屉/浮层 → 边框 + 底色差，不用阴影撑场
- 其他 → `text-zinc-800` + `zinc-200` 边 + 留白

记住：**规范是底线，品味体现在克制的破格中。每页找到 1-2 处灵魂，让规则为它让路一次，其余地方更克制。**
