# DYData 美学规范：顺手且好看

适用：阿禅名下所有项目

## 设计目标

让界面同时做到"顺手"和"好看"，这两者本质不矛盾。好的交互逻辑会自然生长出好的视觉秩序，好的视觉层级会反过来降低认知负担。你的审美参照是Claude 和 Craft  ——它们的界面既赏心悦目又毫不费力，每一处视觉决策都同时服务于美感和效率。

**追求什么：**

- 清晰（不是极简）→ 用户第一眼就看懂，标题就是标题，按钮就是按钮
- 精准（不是克制）→ 颜色是路标，该用就大胆用，但说不出理由的颜色一处不许出现
- 有秩序的舒适（不是刻意的安静）→ 像光线充足的设计工作室，工具齐全、摆放有序、触手可及
- 质感来自打磨（不来自堆料）→ 对齐干净、间距有节奏、边框不浑浊、阴影不廉价

**拒绝什么：**
全灰综合征、保护色标题、操作隐形、假克制、冷硬后台风、过度表现/营销感

**判断标准：**
这个设计决策是否同时让界面更易读、更好看？如果冲突，易读优先；但冲突是少数，大多数时候好的设计两者兼得。

**验收方法：**
把页面截图缩小到 25%，仍能分清 3 层以上的视觉阶梯（标题/正文/辅助/背景），就及格。变成一片灰糊就不及格。

**优先级：** 交互可用性 > 信息辨识度 > 审美品位

## 原子级铁律（15 条）

这些是底层通用标准，任何新页面套用这些原则就能直接出效果。每条都是"If…Then"或"必须/禁止"的硬规则。

### 一、骨架与容器

**1. 主画布通透原则**

主体内容（表格、主表单、对话流）直接铺在页面背景上，禁止用卡片包裹。只有外来信息（系统弹出提示、从别处导入的数据块、可操作面板）才允许轻卡片（`bg-zinc-50` 或 `bg-white`，无阴影或极淡阴影）。

→ 为什么：页面不是扑克牌，是白纸。主体内容就是页面本身，包卡片会变成"一叠一叠的纸"，失去干净感。

**2. 嵌套深度上限**

禁止卡片嵌套超过一层。同组信息用留白（≥24px）和细线（`border-zinc-200`）分组，不用新卡片包旧卡片。

→ 为什么：每多一层嵌套，页面多一圈边框阴影，干净感指数级衰减。

**3. 底色温统一原则**

全站只用同一种色温的灰阶（Stone 石灰或 Zinc 锌灰），从 L0（大背景）到 L2（焦点容器）的亮度阶梯差严格控制在 2%-3%。

→ 为什么：冷暖灰交织是界面"脏"的底层元凶，失去纸本温润感。

### 二、信息层级

**4. 排版即界面**

相邻文字层级必须满足：字号差 ≥ 4px、字重差 ≥ 1 档、色深差 ≥ 2 档灰阶。禁用纯黑（`text-black`），靠灰阶亮度梯队托出重心。

→ 为什么：字号给再多，字重和灰阶不拉开，页面仍会糊成一片。缩小到 50% 仍能分清标题/正文/辅助，层级才到位。

**5. 容器内边距下限**

任何容器内边距不低于 `p-6`（24px），严禁文本、控件死贴容器边框。

→ 为什么：文字贴边像窒息，缺乏视觉缓冲，界面显得极平极糊。

**6. 留白代线原则**

模块切分优先靠 ≥24px 留白和底色差，严禁用深灰实线（`border-zinc-400/500/600`）。必须用线时，全站统一降级为 `border-zinc-200` 或 `border-zinc-100`。

→ 为什么：线条越深，视觉噪音越大。留白和底色差是隐形边界，更干净。

### 三、色彩与状态

**7. 单屏功能色上限**

单屏功能色（非灰阶）不超过 2 种。状态标签/徽章优先用灰阶（`zinc-100` 底 + `zinc-600` 字），仅高危/异常状态才允许彩色。数据可视化可以多色，但 UI 状态色必须克制。

→ 为什么：当到处都是彩色，就等于没有重点。颜色是路标，不是装饰。

**8. 单屏视觉重心唯一**

每屏必须有且仅有一个视觉重心（唯一主 CTA、核心数据大数、或 AI 结论卡）。主橙色（`#D97757`）仅限核心行动点，周围 90% 元素主动退让、弱化为静默灰阶。

→ 为什么：多个高亮同时抢戏，用户不知道看哪。

**9. 墨水退晕原则**

状态标签一律去边框、去高饱和。统一用极淡彩色面状底色 + 同色微浓文字（如 `bg-amber-50 text-amber-600 border-none`），透明度按辨识需要调节（0.04~0.12）。

→ 为什么：饱和标签会伪装成可点击按钮，抢走主体重心。状态色在界面中必须保持绝对静默。

**10. 同组数据共容器**

同组数据指标共用同一容器，内部用留白和细线分隔。禁止每个指标独立成卡片。

→ 为什么：指标墙碎片化像一墙便利贴，缺乏整体秩序。

### 四、交互识别

**11. 主 CTA 权重唯一**

主 CTA 必须唯一且视觉权重最高（暖橙实色 `#D97757`），次操作用边框或幽灵样式。禁止同级按钮并列。

→ 为什么：如果"保存草稿"和"确认下发"视觉对等，用户会愣住不知道先点哪个。

**12. Soft Fill 控件原则**

输入框/选择框默认浅灰填充无边框（`bg-zinc-100/80 border-transparent`），只有 Focus 激活时才反转为纯白底 + 细腻边框（`bg-white border-zinc-200 shadow-sm ring-1 ring-zinc-950/5`）。

→ 为什么：大白底 + 灰边的输入框死板，缺乏物理材质呼吸感。

**13. 三态分明原则**

Hover（悬浮）、Selected（持久选中）、Active（按下瞬间）必须视觉差异一眼可辨：
- Hover：`bg-zinc-100` 或 `translateY(-1px)`
- Selected：`bg-[#8AA8C7]/10 text-zinc-900 font-medium`
- Active：`scale-[0.98]` 物理回弹

表格行必须有 hover 显影，禁止仅靠彩色标签区分行状态。

→ 为什么：状态不分明，用户分不清"鼠标悬浮"和"已选中"，找不到当前位置。

**14. 行级操作可见性**

表格行/列表行必须常显至少一个主操作入口。次要操作可用渐进式显影（默认 `opacity-40`，至少保留一个"···"提示），hover 时 `opacity-100`。

→ 为什么：全部隐身的操作，用户鼠标不扫过去永远不知道有操作，移动端更是直接消失。

**15. Z 轴物理悬浮**

任何脱离文档流的浮层（Dropdown/Popover/Modal）必须用强阴影（`shadow-lg` 或 `shadow-xl`）+ `backdrop-blur` 与底层隔离。

→ 为什么：没有阴影的浮层像贴在网页上，用户分不清当前操作层。

## 硬标准（4 条，不可动摇）

1. **Primary 只有暖橙**：主 CTA 按钮使用 `#D97757` 暖橙，全站行动召唤统一入口色。
2. **UI 组件默认不使用深色背景**：按钮/卡片/chip/徽章/标签/分页器/Tab 不使用 `bg-zinc-900`/`bg-zinc-950`/`bg-black`。仅语义场景保留深色（AI 对话气泡、Tooltip、代码块、模态遮罩）。
3. **图标系统统一 Lucide**：全站 Lucide Icons，线形风格，1.5px stroke，图标尺寸严格与字号绑定（Body 13px 绑定 16px 图标，Title 18px 绑定 20px 图标）。
4. **可读性底线**：
   - 标题文字 ≥ `text-zinc-900`，字重 ≥ `font-semibold`
   - 正文 ≥ 13px + `text-zinc-800`
   - 次要文字 ≥ 12px + `text-zinc-500`
   - 正文与背景对比度 ≥ 4.5:1；大字体（≥18px 或 ≥14px bold）与背景对比度 ≥ 3:1
   - 可点击元素的最小触控区域 ≥ 32px
   - 违反以上任何一条即为 bug，不是风格选择

## 实施手册

以下是具体实施时的参数、配方和速查表。原子级铁律定义"做什么"，实施手册定义"怎么做"。

### 一、视觉语言

#### 1.1 底色层级

| 层级 | 色值 | 用途 |
|------|------|------|
| L0 | `#F0F0F1` | 页面背景 |
| L1 | `#FAFAFB` zinc-50 | 卡片、主面板 |
| L1.5 | `bg-zinc-100/50` | 卡片内次级区域（高密度表单组/工具栏） |
| L2 | `#FFFFFF` white | 焦点区、Modal、激活态输入框、当前 Tab |

白卡（bg-white + rounded + border-zinc）默认附 `--shadow-card`（极淡阴影）。

#### 1.2 文字层级（强制阶梯）

| 层级 | 色值 | 字重 | 用途 |
|------|------|------|------|
| 主角/标题 | `text-zinc-950` | `font-semibold` 或 `font-bold` | 页面标题、模块标题、核心数据 |
| 正文/主体 | `text-zinc-800` | 默认 | 正文内容、表格主列 |
| 辅助/次要 | `text-zinc-500` | 默认 | 描述、时间戳、副标题 |
| 元数据/空态 | `text-zinc-400` | 默认 | 占位符、分隔符、单位 |

**关键规则**：相邻层级之间的色深差必须 ≥ 2 档（如 950 vs 800，800 vs 500）。如果标题和正文用了同一个灰阶，就是反模式。

#### 1.3 色板（功能色大胆用，装饰色克制）

| 语义 | 色值 | 用法 |
|------|------|------|
| 主 CTA | `#D97757` 暖橙 | 全站主按钮，饱和度不打折 |
| 待处理/进行中 | `#D99E55` 琥珀 | 授权、生成中、提醒——直接用，不需要犹豫 |
| 成功/通过 | `#6FAA7D` 矿物绿 | 在线、通过、复制成功 |
| 异常/错误 | `#C9604D` 矿物红 | 失败、驳回、违规 |
| 选中/激活 | `#8AA8C7` 石青 | 当前 Tab、选中项、Focus 态 |

**颜色使用原则**：颜色是辨识工具。用不用、用多少，唯一判断标准是"这个颜色是否帮用户理解了什么"。

- If 元素承载状态/操作/层级信息 → Then 使用语义色，饱和度不打折，让用户一眼辨识
- If 元素是纯氛围/装饰 → Then 用灰阶或 ≤ 0.06 透明度的淡彩，不与功能色争夺注意力
- If 说不出"这个颜色告诉用户什么" → Then 不该用

**数据可视化色板**：`#D97757` 主橙 · `#E3B15C` 琥珀 · `#7FA97E` 绿 · `#8AA8C7` 石青 · `#B08BB8` 雾紫

**墨水晕染态**：状态徽章/Chip 使用 `bg-[#语义色]/0.04~0.12` 面状底色 + outline 或状态点。透明度按辨识需要调节——如果 0.06 看不清状态差异，就提到 0.10~0.12。

#### 1.4 字号档位

| 档位 | 字号 | 字重 | 用途 |
|------|------|------|------|
| Display | `text-[32px]` | `font-bold` | 核心数据大数，每屏至多一处 |
| Section | `text-[24px]` | `font-semibold` | 模块标题 |
| Title | `text-[18px]` | `font-semibold` | 弹窗标题、卡片标题 |
| Subtitle | `text-[16px] leading-[1.5]` | `font-semibold` | 中型卡片头部、表单组标题 |
| Body-L | `text-[14px]` | 默认 | 中文密集列表 |
| Body | `text-[13px]` | 默认 | 默认正文 |
| Label | `text-[13px]` | `font-medium` | 表头、导航、表单标签 |
| Caption | `text-[12px]` | 默认 | 脚注 |
| Eyebrow | `text-[10px]` | `font-medium` | 小眉题（`uppercase tracking-[0.25em]`） |

**行高**：中文正文 `leading-[1.6]`，密集列表 `leading-[1.5]`，Display 大数字 `leading-[1.1]`。
**数字**：默认 `tabular-nums`；Display/Section 级用 `font-mono`。

#### 1.5 间距

档位：4 / 8 / 16 / 24 / 32 / 40px（XS / S / M / L / XL / 2XL）

**格式塔亲密性原则**：
- 强关联元素（标题与副标题、输入框与提示文字）：间距 XS~S（4~8px）
- 同模块内不同组：间距 M~L（16~24px）
- 不同模块之间：间距 XL~2XL（32~40px）

If 两个元素属于同一逻辑组 → Then 间距 ≤ 8px
If 两个元素属于不同逻辑组 → Then 间距 ≥ 24px

紧凑交互元素（导航项、shadcn 内部）允许 `p-3`/`px-3`。

#### 1.6 图标系统

| 属性 | 规范 |
|------|------|
| 线条粗细 | 统一 `1.5px` stroke width |
| 默认色 | `zinc-500` |
| Hover 色 | `zinc-800` |
| 禁用态 | `zinc-300` |

**尺寸与字号绑定**：

| 字号档位 | 图标尺寸 | 场景 |
|----------|----------|------|
| Eyebrow 10px | 12px | 小眉题、紧凑徽章 |
| Body 13px / Body-L 14px | 16px | 默认正文、按钮、列表行 |
| Title 18px / Subtitle 16px | 20px | 卡片标题、弹窗标题 |
| Section 24px / Display 32px | 24px | 模块标题、空状态主图标 |

#### 1.7 圆角

| 层级 | 圆角 | 用途 |
|------|------|------|
| 外层容器 | `rounded-2xl` | 页面级主面板、大卡片 |
| 内层卡片 | `rounded-xl` | 数据块、模块卡片 |
| 按钮/标签/输入框 | `rounded-lg` | 默认形态 |
| 状态点/头像/数字徽章 | `rounded-full` | 正圆元素 |

嵌套规则：外围大圆角（`rounded-2xl`），内部元素等比收窄为 `rounded-lg` 或 `rounded-md`。

#### 1.8 阴影与 Z 轴纵深

**阴影是 Z 轴的定位工具——告诉用户"这个东西浮在上面，优先处理"。**

| 场景 | 阴影强度 | 必须/推荐 |
|------|----------|-----------|
| 悬浮层（Dropdown/Tooltip/Popover） | `shadow-lg` + `backdrop-blur` | 必须 |
| Modal / 命令面板 | `shadow-xl` + 遮罩 | 必须 |
| 独立可点击卡片 hover | `shadow-sm` | 推荐 |
| 白卡默认态 | `shadow-[0_1px_2px_rgba(0,0,0,0.04)]` | 推荐 |
| 表格行、列表行、表单控件 | 无阴影 | — |

If 元素脱离文档流（弹窗/下拉/浮层）→ Then 必须有阴影 + 背景隔离
If 元素在文档流内 → Then 默认不加阴影，靠底色差和留白建立层级

#### 1.9 边框

| 温度 | 色值 | 用途 |
|------|------|------|
| 极冷 | `zinc-200` | 分隔线、卡片边界 |
| 微温 | `zinc-300` | 可交互元素默认边框 |
| 暖橙 | `#D97757` | 激活/选中持久态 |

Focus Ring：`focus-visible:ring-2 focus-visible:ring-[#8AA8C7]/40`（键盘导航时清晰可见）。破坏性操作 Focus：`ring-[#C9604D]/30`。

### 二、交互状态与反馈

#### 2.1 交互状态语义（Selected / Focus / Active / Hover 拆清）

| 状态 | 含义 | 视觉配方 |
|------|------|----------|
| Selected | 持久选中（当前 Tab、当前菜单项、选中的列表项） | `bg-[#8AA8C7]/10` + 文字 `zinc-900 font-medium` |
| Focus | 键盘/输入焦点，短暂可访问态 | `ring-2 ring-[#8AA8C7]/40`，输入框同时 `bg-white border-zinc-300` |
| Active | 鼠标按下瞬间 | `scale-[0.98]` 或背景色加深一档 |
| Hover | 鼠标悬浮 | 列表行 `bg-zinc-100`；卡片 `translateY(-1px) shadow-sm` |
| Disabled | 禁用/只读 | `opacity-50`，图标 `zinc-300`，文字 `zinc-400` |

**规则**：同一元素在不同状态下的视觉差异必须一眼可辨。Selected 和 Hover 不能长得一样。

#### 2.2 Hover 与 Active

**Hover 分两类**：
- 静默场景（列表行、表格行、Tab、导航项）：仅底色 `bg-zinc-100`
- 微浮场景（独立可点击卡片）：`translateY(-1px)` + `shadow-sm`

**Active 回弹**：可点击元素使用 `active:scale-[0.98]`，transition 包含 `transform`，150ms。

**底线**：If 元素可点击 → Then hover 时必须有视觉变化（底色/位移/阴影三选一）

#### 2.3 动效（分层策略）

| 场景 | 动效 | 参数 |
|------|------|------|
| 日常交互（Hover/Focus） | Ease | `150ms cubic-bezier(0.4,0,0.2,1)` |
| 卡片流转（拖动、排序） | Spring | `stiffness: 300, damping: 24` |
| 页面入场 | Stagger 淡入 | `opacity 0→1, translateY 8→0, 150ms` |
| 按压反馈 | 微缩回弹 | `active:scale-[0.98]` + `150ms ease` |

允许动画属性：`background-color` / `color` / `border-color` / `box-shadow` / `transform` / `opacity`。不动 layout 属性。`prefers-reduced-motion` 下关闭非必要过渡。

#### 2.4 输入框（Soft Fill）

- 默认：`bg-zinc-100/70 border-transparent text-zinc-800`
- Focus：`bg-white border-zinc-200 shadow-sm focus:ring-1 focus:ring-zinc-950/5`

#### 2.5 加载态

- 极快响应：直出
- 较慢响应：骨架屏（zinc-100 矩形），结构与最终内容一致
- spinner：仅用于无法预估内容结构的小区域

#### 2.6 渐进式操作显影

高频但次要的行级操作：默认 `opacity-40`，Hover 时 `opacity-100`。

**前提条件**：
- 仅用于行级次要操作（编辑、删除等图标按钮）
- 主 CTA 和关键操作绝不使用此模式
- 必须保证 `:focus-visible` 可访问性
- 行内至少保留一个常显的视觉提示（如 `···`）告知用户此处有操作

## 六、美学与质感

交互是骨架，美学是皮肤和气质。两者共生——没有交互基础的美学是花瓶，没有美学的交互是工具房。本章定义：在交互到位的前提下，界面应该呈现什么气质、什么质感、什么视觉秩序。

### 6.1 美学五感

| 维度 | 含义 | 判断方法 |
|------|------|----------|
| 清晰感 | 第一眼就看懂，不靠猜不靠适应 | 新用户 3 秒内能找到主操作 |
| 秩序感 | 所有元素像被认真放置过，不是堆出来的 | 对齐网格一致、间距有规律 |
| 温度感 | 不冷硬也不热闹，有一点人文关怀 | 空状态有诗意、成功反馈有质感 |
| 质感 | 通过边框/阴影/留白/圆角建立轻微精致感 | 缩放 200% 看细节仍然干净 |
| 记忆点 | 每个核心页面有一个视觉锚，让用户记住"这是哪" | 去掉锚点后页面是否变得平庸 |

### 6.2 气质边界（是什么 / 不是什么）

**目标气质**：温润、清楚、轻量、有秩序、有一点手工打磨感。像一间光线充足的设计工作室——工具齐全、摆放有序、触手可及。

**不是什么**：
- × 企业后台的冷硬灰白（缺乏温度）
- × SaaS 官网的营销感（过度表现）
- × Notion 式的完全无表情（缺乏记忆点）
- × Apple 展示页的大面积留白（信息密度不够）
- × 金融终端的信息压迫感（缺乏呼吸）

### 6.3 色彩比例

颜色的问题不只是"用什么色"，还有"用多少"。

| 色系 | 面积占比 | 职责 |
|------|----------|------|
| 灰白层级（zinc-50~200 + white） | 70%~80% | 底色、容器、留白 |
| 深色文字（zinc-800~950） | 10%~15% | 标题、正文、核心数据 |
| 暖橙 `#D97757` | 3%~8% | 行动召唤，仅此一职 |
| 状态色（琥珀/绿/红/石青） | 5%~12% | 状态辨识，按需出现 |
| 装饰/氛围色 | ≤ 3% | 渐变光泽、墨水晕染、数据可视化溢出 |

**关键**：单屏内除灰阶外，同时出现的功能色不超过 3 种。每种色必须对应一个明确的信息含义——如果说不出"这个颜色告诉用户什么"，就不该用。

### 6.4 视觉重心

每个页面必须有且只有一个视觉重心。

**重心来源（任选一）**：核心数据大数字 / 主 CTA / AI 结论卡 / 当前任务区 / 高优状态模块。

**形成方式**：字号更大 / 色彩更明确 / 空间更宽 / 背景层级更高 / 周围元素主动退让。

**禁止**：多个大色块同时出现；多个 Display 数字同时抢占；主按钮、警告、图表、插画同时成为焦点。

### 6.5 质感组合配方

| 组件类型 | 配方 |
|----------|------|
| 普通信息卡 | `bg-white border-zinc-200 rounded-xl` 无阴影，靠边框和底色差建立存在感 |
| 重点可点击卡片 | `bg-white border-zinc-200 rounded-xl shadow-card`，hover `shadow-sm + translateY(-1px)` |
| 悬浮层 | `bg-white/95 shadow-lg backdrop-blur-md border-zinc-200/50 rounded-xl` |
| 数据焦点卡 | `bg-white rounded-2xl` + 大数字 zinc-950 + 更大内边距 + 细分区线 |
| AI 结论卡 | `bg-zinc-50 rounded-xl` + 左侧 2px 暖橙导轨 + 标题 zinc-950 semibold |
| 深色焦点卡（破格） | `bg-zinc-950 text-white rounded-2xl` + 内部文字对比度 ≥ 4.5:1，每页至多一处 |

### 6.6 精致感来源

精致来自：对齐干净、间距有节奏、色彩有比例、边框不浑浊、阴影不廉价、状态变化轻但明确、图标尺寸与文字严格匹配、同类组件全站一致。

精致不来自：大面积渐变、复杂背景纹理、随机插画、多层堆叠阴影、高饱和撞色、过度动效。

### 6.7 物理与流体感

界面是"活着"的。拥抱物理反馈：按压回弹、数据加载呼吸感、卡片流转的流体布局。动效目的：消除心智断层 + 提供"刚刚好的爽感"。

**卡片流转三件套**（看板/状态栏丝滑滑动）：
- 共享布局：`<motion.div layout layoutId="{域}-{id}">`
- 入场 spring：`stiffness: 300, damping: 24`；退场 `opacity: 0, scale: 0.96, duration: 0.15`
- 撑开式操作条：`grid-template-rows: 0fr → 1fr` 过渡

### 6.8 材质与光泽

焦点场景下允许低饱和度网格渐变、线性光泽或 Backdrop Blur。

适合：页面 hero 区、核心数据卡片局部高光、命令面板。
不适合：数据表格、列表行、表单控件、普通按钮。

### 6.9 空状态

允许有人格的飞地（≤120px）：
- 极淡灰虚线圆轨 + 十字辅助线（仪器卡尺感）
- 中心微妙几何体，允许微动效（呼吸/浮动）
- 文案有温度：标题 zinc-600、副文 zinc-400

### 6.10 多巴胺反馈

重要用户节点给予质感奖赏：局部墨水晕染（0.04~0.12 透明度，按辨识需要调节）、数据的丝滑重载、Spring Scale 弹跳。

### 6.11 状态语义动效

| 态 | 填充 | 动效 |
|------|------|------|
| Critical | `#C9604D` | 静态 |
| Active | `#D97757` | 单色呼吸（仅当前焦点） |
| Done | `#6FAA7D` | 静态 |
| Idle | `zinc-200` | 静态 |

默认不用 `animate-pulse`，不用渐变填充。

| 组件 | 默认配方 |

| 主按钮 | 暖橙实色 `rounded-lg`，文字白色，hover 加深 |

## 验收清单

设计完成后，逐条自检：

1. 主体内容是否直接铺在背景上，没有多余卡片包裹？
2. 页面缩小到 25%，能否分清标题/正文/辅助三层？
3. 单屏功能色（非灰阶）是否不超过 2 种？
4. 主 CTA 是否 3 秒内可定位？是否唯一？
5. 所有可点击元素是否都有 hover + focus 反馈？
6. Selected / Hover / Disabled 三态是否一眼可辨？
7. 是否存在 `text-zinc-400` 或更浅承载关键信息？
8. 同一语义是否全站使用同一颜色？
9. 页面是否只有一个视觉重心？
10. 悬浮层是否有阴影与底层隔离？
11. 相邻文字层级的色深差是否 ≥ 2 档？
12. 能否说出页面上每一处颜色"告诉用户什么"？
13. 状态标签是否看起来像按钮（高饱和/粗边框）？
14. 表格行是否有 hover 态和操作入口？
15. 容器内边距是否 ≥ 24px？
