# DYData 全局 UI 视觉重构方案

> 角色设定：UI 视觉架构师
> 目标受众：阿禅（精简实用、图表优先、少术语）
> 现状总结：设计语言整体清晰（Apple-style token、glassmorphism），动效考究。但存在三处结构矛盾、一批移动端适配细节以及 Token 双套系统的混用风险。

## 第一部分：全局设计规范定义

系统目前混用了 Shadcn 的 `oklch` 语义色和自定义的 `Mac 风格 RGB/HEX` 色。这在暗色模式下会导致严重的视觉断层（如 primary 色变灰白）。

**1. 统一色彩体系（The Single Source of Truth）**
*   **亮色主调 (Primary)**: `#007aff` (iOS 蓝) / `oklch(0.55 0.2 260)`
*   **暗色主调 (Primary)**: `#0a84ff` (暗黑蓝) / `oklch(0.6 0.2 260)` -> 必须修复目前暗色下的无色相断档。
*   **状态色语义必须统一**：
    *   **强/好/达标**: 蓝色 (Primary) 或 绿色 (Success/Emerald)
    *   **中/普通**: 黄色 (Warning/Amber)
    *   **弱/差/未达标**: 红色 (Danger/Rose)

**2. 圆角与毛玻璃 (Glassmorphism)**
*   保持当前的 `--radius` 基准 (0.75rem)。
*   规范 `.glass-card` 的运用：统一用于悬浮层、重点指标卡；弱化数据区使用实色卡片或降低背景透明度，避免通篇高对比度玻璃卡导致视觉疲劳。

**3. 移动端优先原则**
*   屏幕宽度 `sm` (<640px) 下，核心行动按钮必须占满宽 (`w-full`) 或使用前置悬浮。
*   横向数据（表格/雷达图/柱状图）必须增加横向滚动或最大高度截断保护。

## 第二部分：逐页优化方案与排期

按照对核心业务路径的阻断程度，将问题划分为 P0（必须修）、P1（严重影响体验）、P2（视觉/交互打磨）。

### 🔴 P0 级别（阻断核心操作，优先解决）

| 模块 | 问题描述 | 优化方案 |
| :--- | :--- | :--- |
| **全局交互** | Toast 提示在屏幕正中央弹出，会**直接遮挡并锁死**正在操作的表单和按钮。 | 改为**右上角** (`top: 1rem; right: 1rem`) 弹出，或**底部安全区**上方弹出。 |
| **全局主题** | 暗黑模式下，`primary` 变量断档，导致导航栏激活色变白灰，Logo 变白底白字，丢失品牌色。 | 在 `globals.css` 或 `design-tokens.css` 中强制对齐两套 Token，将暗色 primary 改为蓝色 (`oklch(0.6 0.2 260)` 或 `#0a84ff`)。 |
| **Dashboard** | 排行榜表格 1380px 写死宽度，在手机上**向右滚动望不到边**，且左侧固定列滑动时背景脱色，像裂开一样。 | 默认紧凑模式（只显示播放、涨粉等 4 列），通过「展开」按钮显示全部 10 列。 |
| **Growth (成长)** | 页面容器**缺少左右边距保护** (`px`)，在手机上文字直接贴到屏幕黑边。 | `growth-client.tsx` 最外层容器补充 `px-4 sm:px-6 lg:px-8`。 |
| **Growth (成长)** | 六维雷达**颜色语义完全相反**：最强的指标标红（警告），最弱的标绿（安全）。 | 统一语义：强=蓝/绿，弱=红。 |
| **Admin (管理)** | Admin 主页头部的 4 个大按钮（日期、导出等）**是摆设，点按无效**，真实功能在下方卡片。 | 直接删除这 4 个假按钮，保持页头干净。 |
| **Admin (管理)** | 成员列表的「所属小组」列，每一行都写死「即将推出」，极其占据宝贵的横向空间。 | 删除或隐藏该列。 |
| **Analytics (分析)**| 爆款分析器的顶部筛选按钮（提交人）在手机上被挤压截断，无法阅读。 | 调整 Grid 列数：移动端 2 列，按钮区改为 `flex-wrap` 或横向滑动。 |

### 🟠 P1 级别（高优体验改造）

| 模块 | 问题描述 | 优化方案 |
| :--- | :--- | :--- |
| **Auth (登录/注册)**| Logo 图片设定了无限循环的「呼吸放大」动画，极易分散用户填写表单的注意力。 | 动画改为 `repeat: 0`（仅入场播放一次），或移除放大缩放，仅保留渐显。 |
| **Dashboard** | 截图上传区在手机上过高（160px），挤占首屏空间，用户可能**看不到下方的填报区**。 | 移动端降低高度至 `min-h-28`，上传完成后支持折叠。 |
| **Dashboard** | 填报区的互动数据用了 `opacity-75` 降低透明度，看起来像**「被禁用不可填」**。 | 移除 opacity 降权，改用字号大小或标题颜色区分主次。 |
| **Growth (成长)** | 诊断建议区的「示范数据卡」长得和真实的「弱项卡」**一模一样**，极易误导初次进来的员工。 | 示范卡使用明显的灰色系，或打上大号的「Demo/示例」水印。 |
| **Growth (成长)** | 六维雷达图在手机上高度失控（随宽度等比放大），整个雷达图区块撑满手机全屏。 | 雷达图容器增加 `max-h-[300px]` 限制。 |
| **Admin (管理)** | Market 页面点「编辑」时，列表上方会弹出一个新的表单，但原来的「新增」表单还在，**出现两个表单打架**。 | 编辑操作改为 Dialog 弹窗，或在原位展开替换行。 |
| **Admin (管理)** | Advice 页面的四个数字统计卡（如\"待执行 3\"）看上去像筛选按钮，**但点击无反应**。 | 联动下方列表，点击卡片即触发状态筛选；若不做联动，则弱化卡片的按钮感。 |
| **Admin (管理)** | Admin/Analytics 与 Admin 首页的**最大宽度不同**（5xl vs 7xl），切页面会左右跳动。 | 统一设定管理后台容器最大宽度，推荐 `max-w-6xl`。 |

### 🟡 P2 级别（视觉精细度打磨）

| 模块 | 问题描述 | 优化方案 |
| :--- | :--- | :--- |
| **Dashboard** | 问候语区域（\"你好，阿禅\"）视觉单薄，且下方的待办图标在部分安卓机型可能乱码。 | h1 标题增加颜色强制限定，图标替换为 Lucide 图标，补全今天的日期展示。 |
| **Dashboard** | 手机端顶部导航，用户名被隐藏后，只剩一个孤零零的「退出」按钮。 | 用名字首字母做一个圆形的 Avatar（如「禅」），即使是小屏幕也保留身份标识。 |
| **Growth (成长)** | PK 差距条（GapBar）在一方数值为 0 时，另一方占满整条，看不出是 0 还是没数据。 | 为 0 的一方保留 2px 的极细占位条。 |
| **Growth (成长)** | 文案拆解区块的时间戳显示 `0s - --s`，格式混乱，且跟分类标签挤在一起。 | 时间戳换行显示，若无结束时间则不显示 `- --s`。 |
| **Admin (管理)** | 视频列表把随机生成的 UUID（前8位）当作副标题显示，对管理毫无意义。 | 替换为显示「账号名称」或「标签分组」。 |
| **Admin (管理)** | Guidance 页面的四个卡片全白，没有 Advice 页面好看。 | 给纳管、重点、下滑等状态配上不同颜色的底色 (slate/emerald/amber/rose)。 |
| **Content Tools** | Tab 切换后的标题描述，在头部和按钮 tooltip 里重复显示，啰嗦。 | 删除页面内容区的静态描述，只保留 Tab 上的功能说明。 |
| **Content Tools** | 选题生成器每次切日期/切账号都会自动触发 AI 重新请求，消耗成本且体验顿挫。 | 移除副作用触发，只在用户手动点击「重新生成」时请求。 |

## 最佳执行路径建议给研发（AI 助手）：

1.  **首批提交 (Fix P0 CSS & Layout)**:
    *   改掉 `globals.css` 中的 Toast 居中。
    *   修好暗黑模式 Primary Token。
    *   给 `growth-client.tsx` 加上 `px-4 sm:px-6`。
    *   删掉 Admin Header 假按钮和成员列表假数据列。
2.  **次批提交 (Dashboard & Growth 重点修复)**:
    *   排行榜横向滚动/折叠。
    *   雷达图颜色语义翻转与高度限制。
    *   上传区高度缩小。
    *   示范数据视觉隔离。
3.  **最终批次 (Admin & 细节打磨)**:
    *   修复 Admin 各个表格的堆叠、弹窗问题。
    *   完成所有 P2 级别的微调。
