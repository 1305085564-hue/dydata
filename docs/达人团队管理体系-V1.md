# 达人团队管理体系 · V1

> 版本：V1（待落地）｜适用：约 24 名达人 + 2 剪辑 + 1 拍摄指导｜定位：在创始人不全程介入下，体系仍能维持入门级以上水平。

---

## 0. 三条底线

1. **去中心化**：每个环节都要有"你不在场谁兜底"的备案。
2. **最小可运行**：先跑能跑起来的版本，跑顺了再加；避免重蹈"11-3-6"覆辙。
3. **过程管控 > 结果追责**：先把过程标准化，结果自然跟。

---

## 1. 组织架构

```
你（部门负责人）
 ├── 组长 A-E（5 人，每人带 3-4 个组员）
 ├── 第六组（你挂名，2-3 个老人，轻管理）
 └── 公共资源：剪辑 ×2（仅服务第二账号）+ 拍摄指导 ×1
```

| 角色 | 核心职责 | 汇报对象 |
|------|---------|---------|
| 你 | 定标准、培养组长、抽检、教学策划、战略 | — |
| 组长 | 全量审文案/审片、组内日盘点、带新人、执行教学 | 你 |
| 达人 | 全链路日更、网站上传数据、按反馈修改 | 组长 |
| 拍摄指导 | 现场把关、拍摄技巧教学 | 你 |
| 剪辑 ×2 | 仅剪第二账号内容、按排期交付 | 你 |

## 2. 一日 SOP · 5 个卡点

| 卡点 | 时间 | 谁做什么 | 谁检查 | 超时处理 |
|------|------|---------|--------|---------|
| ① 数据上报 | 11:00-11:15 | 达人在网站上传昨日数据 | 组长 11:15 确认全员已填 | 组长 @催，记入内容日报 |
| ② 早盘/早会 | 11:15-12:00 | 全员开会，组长用网站数据点评异常+分享案例 | 你扫一眼（15 分钟） | — |
| ③ 选题第一关 | 15:00-15:30 | 达人提交选题（股市 3 点收盘后） | 组长（方向是否对） | 15:30 未交，组长 @催 |
| ④ 写文案 + 选题第二关 | 15:30-18:00 写；18:00 二审 | 达人写文案；18:00 组长检查是否跑偏+审 6 项 | 组长全量一审 + 你抽检 3-5 人 | 18:30 仍未交，记入日报 |
| ⑤ 审片 + 发布 | 18:30-20:00 | 组长审成片，通过后达人按最佳时段发布 | 组长 | — |

**你每天的介入点只有 3 个**：
- 11:15 扫早盘（15 分钟）
- 18:00 前后 抽检文案 3-5 篇（30-45 分钟）
- 21:00 看组长日报（15 分钟）

日均管理投入 ≈ 1 小时。

## 3. 内容审核 6 项（财经复盘专项）

组长每天对组内全量过这 6 项；具体判定细则你来定，组长照做。

| 审核项 | 关注点 |
|--------|-------|
| 开头感染力 | 同一段话怎么写更有情绪张力，前 3 秒决定完播 |
| 观点质量 | 财经复盘内容的绝对核心，观点是否清晰、有依据、有态度 |
| 合规红线 | 违禁词、过度承诺、敏感表述（封号风险高） |
| 绩效钩子 | 今天给明天埋"继续看"的钩子 |
| 昨日绩效回顾 | 复盘必须前后联动，否则没意义 |
| 导粉话术（如适用） | 自然不生硬，影响转化 |

**选题双关**：15:00 看方向、18:00 看有没有写偏。

**审核层级**：组长全量一审 → 你每天每组抽 1 人共 3-5 篇 → 抽检发现问题反馈给对应组长整改。

## 4. 会议节奏

| 频率 | 形式 | 参与人 | 内容 |
|------|------|--------|------|
| 每天 11:15 | 早盘/早会 | 全员 | 组长用网站数据做异常点评+案例分享 |
| 每两周一次（周一） | 全员方向会 | 全员 | 复盘前两周大盘数据 + 定下两周方向 |
| 每周五 | 组长会 | 你 + 5 个组长 | 各组数据汇报、共性问题、组长能力校准 |

月度节奏：**约 20 次早盘 + 2 次方向会 + 4 次组长会**。你主持的固定会议每月 6 次。

## 5. 组长制度

### 5.1 KPI（按组内平均导粉量计算）

| 条件 | 月度津贴 |
|------|---------|
| 组内均导粉 ≥ 1000 | 2000 元 |
| 组内均导粉 ≥ 1500 | 3000 元 |
| 组内均导粉 ≥ 2000 | 5000 元 |

**生效门槛**：组内至少 4 人已进入导粉阶段。新人（入职<1 个月）不计入均值、也不占名额。

**激励逻辑**：组长自己导粉拉高均值有效（鼓励组长自己也出活）。

### 5.2 组长日报（仅内容层，每天 20:30 前在网站提交）

固定四个字段，鉴赏式反馈：

```
1. 选题：今日组内选题方向亮点 / 问题（举例）
2. 开头：哪条开头写得好 / 哪条开头有问题
3. 脚本文案：组内整体写作水平观察、典型问题
4. 视频：成片质量观察、突出案例 / 待改进案例
```

数据层不做主动汇报，你直接在网站看（被动反馈）。

### 5.3 组长考核

- **结果维度**：导粉 KPI（看上表）
- **过程维度**：你的抽检结果（如果你频繁抽到该组低质量内容过审 = 组长审核不到位）
- **过程维度**：日报提交率与质量

| 时间 | 动作 | 耗时 |
|------|------|-----|
| 11:15 | 扫一眼网站数据 + 听早盘 | 15 分钟 |

| 21:00 | 看 5 份组长日报，有问题私聊 | 15 分钟 |
| 周五下午 | 组长会 | 1-1.5 小时 |

**日均 ≈ 1.5 小时；周固定 ≈ 1.5 小时**。剩余精力给教学策划+授课+自我提升。

## 7. 落地节奏（2 阶段）

### 第一阶段：启动周（第 1 周）

- [ ] 全员会宣布分组+组长职责+新流程
- [ ] 组长集中对齐 1 次（讲清审核 6 项 + 日报模板 + KPI 规则）
- [ ] SOP 从下个工作日开始跑
- [ ] 你前 5 天和组长"双审"——组长先审、你复审，快速校准标准
- [ ] 网站 P0 功能上线（见第 9 节）

### 第二阶段：松手期（第 2-4 周）

- [ ] 第 2 周起逐步减少双审，转为纯抽检
- [ ] 第 2 周五开第一次组长周会，复盘头两周
- [ ] 第 3 周一开第一次双周方向会
- [ ] 根据实际运行情况微调流程
- [ ] 第 4 周组长 KPI 试算（只算不罚，让大家看到规则在跑）
- [ ] 网站 P1 功能陆续上线

## 8. 暂缓项（明确放着，不做）

| 项目 | 暂缓原因 | 重启时点 |
|------|---------|---------|
| 知识库 8 模块 | 你脱离一线太久、工程量大 | 2 个月后由组长执行、你把控 |
| 新人入职 SOP | 短期不扩张、零星入职 | 有 3 个以上新人同期入职再启动 |
| 月度全员大会 | 双周方向会已覆盖节奏 | 团队规模 50+ 再考虑 |

## 9. 网站功能建议（分级）

> 网站由 Claude Code/Codex 产出，这里一次性列全；做多少看实际情况。

### 🔴 P0 必须做（这套体系能跑起来的最低配）

| 功能 | 说明 |
|------|------|
| 组长看板 | 按组展示组员数据，组长只看自己组，你看全局 |
| 卡点状态实时追踪 | 每天 5 个卡点，每达人完成状态可见（已交/未交/已审/被打回） |
| 组长日报提交 | 固定 4 字段（选题/开头/脚本/视频），替代微信群发文字，你一页看 5 份 |
| 异常自动标红 | 连续 3 天数据下滑、当日未按时提交，自动推到组长看板顶部 |

### 🟡 P1 尽量做（显著提升效率）

| 功能 | 说明 |
|------|------|
| 审核流转模块 | 达人提交选题/文案 → 组长在网站审核（通过/打回+批注）→ 留痕 |
| 组长 KPI 自动算 | 按月自动算组内均导粉，组长能看到离下一档差多少 |
| 周/月报自动聚合 | 基于日数据自动出周月维度报表，组长开会前不用手动做 |
| 你的抽检记录 | 抽检后打标（通过/问题+备注），自动关联到对应组长，月底汇总=组长过程考核依据 |

### 🟢 P2 可以考虑（锦上添花）

| 功能 | 说明 |
|------|------|
| 选题库 | 达人提交的选题沉淀成库，标"已用/爆了/扑了"，复用参考 |
| 爆款自动标记 | 播放量超均值 X 倍自动打标，复盘会拉出来分析 |
| 达人成长曲线 | 周/月趋势可视化，转正/淘汰决策一目了然 |
| 公告与通知 | 规则更新、方向调整发到网站，不淹没在微信群 |

## 10. 落地风险与应对

| 风险 | 应对 |
|------|------|
| 组长不会审/审不准 | 第 1 周双审校准；审核 6 项你给具体细则；月度组长培训 |
| 组长不愿管/走过场 | 津贴 + 抽检倒逼 + 月度组长会公开 KPI 排名 |
| 达人抵触被管 | 启动周全员会讲清"为什么这么做"（多赚钱）；先跑核心 5 卡点，不要一次加太多 |
| 流程跑不动 | 死守"最小可运行"；卡点不合理就调时间，但不砍卡点 |
| 第六组老人不配合 | 你挂名管，只要求基础数据填报，其他不强推 |
