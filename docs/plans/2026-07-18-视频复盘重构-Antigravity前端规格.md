# 视频复盘重构 · Antigravity 前端规格

> 你只管美学与交互，从零重写。**删掉 Codex 的简陋前端，绝不参考它的前端代码**（会被污染）。只依赖「接口约定」的字段。产品背景见「产品设计总纲」，视觉遵《docs/美学规范.md》。

## 0. 总览
两层：复盘台首页（选谁复盘）→ 归因舱（复盘这一条）。核心是**归因舱首屏**——第一眼必须是"归因结论"，不是数据墙。

## 任务 A · 复盘台首页
- **顶栏异常提醒**：一条窄条，"今日异常 N 条：X 删稿·Y 腰斩 [直接去盘]"。低调、不抢主视觉，可点直达最严重那条。
- **主视觉=视频列表**：这是主角（不是待办堆，是主动挑的入口）。支持"按最差排 / 按人 / 按时间"切换。
- **列表行**：成员·账号 + 该条被标出的异常指标（如 `5s完播22%🔴 均播18%🔴`），右侧 [复盘] 按钮。让人在列表就看出该盘谁。
- 达标项绿、异常项红（数据好坏语义，非股市涨跌）。

## 任务 B · 归因舱首屏（重构最关键，慢工细活）
左右分屏：**左=数据哪不对，右=文案哪不对+怎么说**。

### B1 归因结论带（左屏顶部，第一眼焦点）
- 逐行渲染接口 `findings`：`🔴 5s完播22% ↓自己-18 → 开头弱 [看]`。
- 每行四件：指标+值 / 相对参照的差 / `points_to`（指向哪段）/ [看] 按钮。
- tone 配色：bad 红、warn 琥珀、good 绿。**绿色项也显示**（"节奏OK不用动"），让人安心别瞎改。
- 点 [看] → 右屏脚本对应段高亮 + 滚动过去（联动核心）。
- 数据缺失项显示"缺数据"，不显示 0。

### B2 数据对比大盘（左屏中部）
- 保留"本条 vs 参照"的极简条形对比（现状那个直观，可借鉴视觉但重写）。
- **参照系切换器**：`比自己 / 比团队 / 比高手 / 比某人`，切换调接口 `?ref=`。默认"比自己"。
- 留存曲线（漏斗）：标出拐点秒数，hover 拐点可反向高亮右屏对应段（进阶，可 V2）。

### B3 分段脚本（右屏上部）
- 按段渲染 `video.content`。被 `locate` 指向的段/句高亮（A 类到句、B 类到属性提示）。
- 点某句可引用至反馈（保留现状那个"点句子→进主要问题"的好设计）。

### B4 反馈下发（右屏下部，20% 权重、顺手就写）
- 证据区：把归因带的红项自动列成"诊断依据"。
- 主要问题 + 建议反馈两栏，自动存草稿（保留现状机制）。
- **[AI 参考思路] 按钮**：可选，点了才调 content-analysis，出建议给人参考。**文案必须体现"仅辅助、你来定"**，绝不写成"AI 已诊断"。
- **[确认并下发]**：两步防误触（保留现状）。下发成功 → **自动跳下一条待复盘**（提示"已下发，进入下一条"）。

## 任务 C · 可复测
- 归因舱内一个入口"看上次给他的反馈"，调 `previous` 接口，弹出上次卡片 + 那条指标，与本条并排对比。无历史则不显示该入口。

## 任务 D · 兜底态（必须做，别只做成功态）
- `snapshot_ready:false`：归因带位置显示"这条还没 24h 数据，先看已有的 X，归因待数据齐"，不空白。
- 某指标缺：该行"缺数据"。
- AI 辅助失败：toast 提示，不影响人工归因。
- 自动流转到底（无下一条）：提示"今日待复盘已清完"。

## 通用要求
- 遵《docs/美学规范.md》；红涨绿跌仅用于行情，这里红=异常/差、绿=达标/好。
- 移动端可后置，先保桌面管理端体验。
- **砍掉**：推流峰值/峰值后走势/流量承接那套人工下拉，归因舱不再出现。

## 交付顺序建议
1. 归因舱首屏 B1+B3 联动（心脏，先跑通"点红项→跳台词"）
2. B2 参照系切换 3. B4 反馈+自动流转 4. 任务 A 首页 5. C 复测 6. D 兜底打磨
