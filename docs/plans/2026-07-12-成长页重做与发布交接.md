# 成长页重做与发布交接

## 收口结论

- 正式 `/growth` 只消费 `loadGrowthPageContract` 返回的真实数据契约，不再使用虚拟报告、虚构同事或前端自算六维。
- 页面主线是「最弱项→诊断依据→下一条改法」，证据层提供雷达、核心指标、实名对标、真实脚本和趋势。
- 可信度只按真实日报数分为 0–2、3–9、≥10 三档；空数据、低样本、正常样本都有独立渲染验证。
- 团队暂无基准时，雷达区只列用户真实原始指标，不生成虚拟团队多边形。

## 废弃链路与替代

| 废弃内容 | 替代或证据 |
| --- | --- |
| `/api/growth/page-data` | 正式页在服务端直接调用 `loadGrowthPageContract`；源码无活跃请求，只剩历史网络抓包记录 |
| `growth-hydration.ts` | 页面不再做 initial/full 延迟补载，由服务端单契约一次传入 |
| 旧诊断、行动、PK、脚本、状态卡组件 | 功能已收进 `growth-client.tsx` 的契约渲染；全仓无活跃导入 |
| 旧组件与 hydration 测试 | 由新 `growth-client.test.ts` 的空数据、低样本、正常样本三类渲染测试取代 |

## 发布门槛

- 成长页契约、加载器和前端状态测试全部通过。
- 生产构建、`git diff --check` 和真实浏览器三类样本验证都是 push 前必须项。
- 本轮不修改数据库、migration、线上环境变量或无关页面。
