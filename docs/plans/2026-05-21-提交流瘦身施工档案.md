# 2026-05-21 提交流 + 审核流瘦身施工档案

## 背景
基于「话术案例库」的 brainstorming 终判，把员工提交从「员工写所有事」改为「员工只写事实，管理员写判断」，闭环重写。

## 真问题（来自现场截图诊断）
- /violations 同屏 3 个相同 CTA（顶部「提交新案例」+ hero「没找到，交一条」+ 空态「提交新案例 →」）+ 员工同时看到「管理工作台」入口
- 提交模板把员工提交、管理员审核、二次测试塞进同一表单（违规场景/风险点、为什么危险、复核建议、测试结果）
- 「违规分类」只有短视频一种 → 选项空间为 0，是死字段
- 「申诉状态」含「申诉中」中间态，逼员工分两次提交，逻辑反人性
- 平台通知文本可选 → 失去最硬证据
- 原因标签让员工选 → 员工不知道哪里违规，是管理员的判断

## 闭环重定义

| 节点 | 谁 | 干什么 | 落地字段 |
|---|---|---|---|
| 1 提交 | 员工 | 交事实+证据 | 话术原文 / 账号 / 处罚事实 / 截图 |
| 2 审核 | 管理员 | 给判断 | usage_state / risk_level / promotion_level / admin_conclusion / suggested_action / 踩雷点标签 |
| 3 反馈 | 员工 | 详情页接收结论 | 主徽 + 副信息 + admin_conclusion + 踩雷点 |
| 4 复用 | 员工 | 按 usage_state 用合规话术 | 列表/详情 |
| 5 沉淀 | 员工 | 记录效果 | result_flag + views/follows |
