# 内容批改详情侧栏 - 阿禅减法施工档案

> 2026-05-28 · CC · `src/app/(app)/admin/content/content-detail-dialog.tsx`

## 病灶诊断

阿禅在批改台的内容详情侧栏看到的是"匀速白盆地"：5×2 的 MetricBadge 网格、对比表外层 `border bg-white p-4`、规则提示 4 行散文盆地、曲线观察大盆地、AI 卡白底浅左竖线、经验标记独占一节。**信息密度足够，但视觉上没有一处主角**——破格不够，主战 KPI 和爆款 AI 卡都被普通 section 同等对待，眼睛找不到落点。

## 决策清单（11 项）

| # | 模块 | 改造手法 |
|---|---|---|
| 1 | 核心数据 | 5×2 卡片网格 → 4 主 KPI 报纸式（26px 数字 + 上方数字下方 caps label）+ 副 chip inline 一行（均播·点赞·评论·分享·收藏·涨粉，· 分隔） |
| 2 | 上一条对比 | 去除 `rounded-xl border bg-white p-4` 外层盆地；标题与作品名/时间合并到一行 Eyebrow；最大差异行用 `bg-zinc-100/40` 整行底色替代左竖线（更克制） |
| 3 | 规则指标提示 | 4 行散文 → 4 个 inline chip（指标 + 状态文字），状态色彩分级（high/low → 红，weak → 琥珀，healthy → 绿） |
| 4 | 曲线观察 | 去除外层白盆地，紧凑 `space-y-2.5`；删除"保存观察"按钮（合并到"生成 AI 辅助分析"按钮的副作用，silent 调用）；按钮孤悬右下 |
| 5 | AI 辅助分析 | 破格强化：白底 → 暖橙 `bg-gradient-to-b from-[#FFF8F3] to-white`；左导轨 `border-l-2` → `border-l-[3px]`；主结论 16px → 17px；chip `bg-zinc-100` → `bg-white/80 border` 反衬出渐变 |
| 6 | 文案原文 | 默认 inline 单行 truncate（"文案" caps label + 截断行 + 展开按钮），展开后才出 `bg-zinc-100/50` 盆地 |
| 7 | 经验标记 | 删除 feedback Tab 底部独占一节的折叠白盆地；并入 Footer DropdownMenu，提供"标记为爆款 / 标记为失败案例"两个直接菜单项；删除 form 选择器（`PillGroup`、`experienceOptions`、3 个 state） |
| 8 | 复盘依据 | 去 Eyebrow + bullet 列表；改 inline 单行 wrap：`依据 · 句一 · 句二`，盆地保留但 px-3 py-2.5 收紧 |
| 9 | 截图 | 主图 h-56 → h-48；副图 h-32 → h-28（视觉上让位给 KPI/AI） |
| 10 | 间距 | analysis Tab `space-y-6` → `space-y-5`；section 间不显式补分割线（让 Eyebrow 自身的 caps label 承担分隔） |
| 11 | 自动保存 | `handleGenerateAnalysis` 调用前先 `await handleSaveObservation(true)` 静默保存观察，避免删按钮后丢数据 |

## 核心理念

- **报纸式**：4 个主战 KPI（播放量·2s跳出·5s完播·完播率）grid-cols-4 等宽 26px 数字裸排，配 `text-[10px] uppercase tracking-[0.2em]` 副 caption——这是阿禅美学法典里的"数字优先于装饰"。
- **破格点唯一**：整个侧栏只有 AI 辅助分析卡有暖橙渐变背景。其余全部裸排或浅 zinc-100/50/60 灰盆。眼睛进入侧栏会先被 AI 卡吸引，正是它的位置——观察后的核心结论。
- **最大差异行的"框"消失**：原方案 `border-l-2 border-l-[#D97757]` 在裸表格里反而抢戏，改成 `bg-zinc-100/40` 整行底色，差值列保留语义色，足够。
- **经验标记降级**：原本独占一节带说明文字，现在变两个 dropdown action。理由：低频次级动作不应该和"确认并下发"同等占位。AI 卡内的"一键标爆款"承担分析路径上的爆款标记；Footer dropdown 承担反馈完成后的回顾标记。

## 删除项

- `MetricBadge` 组件（替换为 `KpiHero` + `KpiChip`）
- `PillGroup` 组件
- `experienceOptions` 常量
- 3 个 state：`experienceType` / `experienceNote` / `experienceFormOpen`
- "保存观察"按钮（功能合入 AI 分析按钮 silent 调用）
- 文案原文 220 字截断逻辑（改 inline truncate）
- 对比表 `min-w-[520px] overflow-x-auto`（裸表格使用容器宽度自适应）
- `handleMarkExperience` 的 `presetType?: ExperienceType` 改为必传
