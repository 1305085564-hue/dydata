## 1. 总目标

本次不是新建页面，不是重做批改台，不是替代成长分析，也不是团队经营分析。目标是在现有 `/admin/content` 批改台内增强闭环：

管理者发现异常作品 -> 内部分析 -> 编辑反馈 -> 下发给员工 / 后续沉淀为经验。

页面主人：组长、负责人、Owner。

普通组员不进入批改台，只在 `/dashboard` 查看已经下发给自己的反馈。

反馈字段已经统一简化为两个：

1. 主要问题
2. 改进反馈

后续所有分析引用、反馈编辑、员工端展示，都按这两个字段设计，不再保留“下一条动作”这个独立产品字段。

## 2. 最高禁止边界

1. 不新建一级页面。
2. 不把批改台改成成长分析。
3. 不把经营分析塞进批改台。

5. 分析 Tab 不能调用 `/api/admin/next-day-review`。

## 5. P0 技术实现要点

### 5.1 播放涨跌幅

实现文件：`src/lib/loaders/admin-content-page.ts`

新增字段：

```ts
previous_play_count?: number | null
play_count_change_pct?: number | null
play_change_signal?: "surge" | "halve" | null
```

计算规则：

1. 当前作品必须有 24h 快照播放量。
2. 上一条视频定义为同 `account_id` 下 `published_at < 当前视频.published_at` 的最近一条。
3. 上一条必须有 24h 快照播放量，且播放量 > 0。
4. `(current - previous) / previous * 100 >= 100` -> `surge`。
5. `<= -50` -> `halve`。
6. 不达阈值、无上一条、上一条播放为 0 -> 不展示信号。

性能注意：

1. Round 3 放在现有 Round 2 `Promise.all` 后独立执行。
2. 不嵌入 Round 1 / Round 2。
3. 不前端逐条请求上一条视频。
4. 不全库扫描；只围绕当前可见列表涉及账号补边界上一条。

### 7.1 新表 content_observations

建议 migration 新增，不改旧 migration。

字段按原方案：

```sql
CREATE TABLE content_observations (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  video_id uuid NOT NULL REFERENCES videos(id) ON DELETE CASCADE,
  observer_id uuid NOT NULL REFERENCES profiles(id),
  traffic_peak_level text CHECK (traffic_peak_level IN ('high', 'medium', 'low', 'unset')),
  post_peak_trend text CHECK (post_peak_trend IN ('smooth_decline', 'cliff_drop', 'multiple_peaks', 'unset')),
  traffic_retention_quality text CHECK (traffic_retention_quality IN ('good', 'average', 'poor', 'unset')),
  drop_off_stage text CHECK (drop_off_stage IN ('opening', 'middle', 'ending', 'not_obvious', 'unset')),
  suspected_problem_stage text CHECK (suspected_problem_stage IN ('opening', 'middle_content', 'topic_mismatch', 'weak_interaction', 'weak_conversion', 'unset')),
  note text,
  created_at timestamptz NOT NULL DEFAULT now(),
  updated_at timestamptz NOT NULL DEFAULT now(),
  UNIQUE(video_id, observer_id)
);
```

规则：

1. 每个管理者对每条视频只有一条观察记录。
2. 再次保存时 UPSERT 覆盖自己的记录。
3. 多个管理者可以分别写观察。
4. 普通组员不可见。

CREATE TABLE content_experience_marks (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  video_id uuid NOT NULL REFERENCES videos(id) ON DELETE CASCADE,
  feedback_card_id uuid REFERENCES content_feedback_cards(id) ON DELETE SET NULL,
