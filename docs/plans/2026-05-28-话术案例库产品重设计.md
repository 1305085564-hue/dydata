# 话术案例库产品重设计方案

## 问题诊断

当前版本（cee2770）存在以下产品逻辑错误：

1. **首页职责混乱** — 员工页面出现了审批按钮和「审批闭环」Tab，审批是管理职责
2. **Tab 无意义** — 「找可用话术」和「避坑指南」点进去页面完全一样，没有区分
3. **卡片粒度错误** — 每条话术一张独立卡片（两列网格），应该是话术列表整体作为一块大卡片，内部每条话术是一行
4. **交互方式错误** — 点击话术打开侧边抽屉，应该是中心弹窗（Dialog）
5. **管理页面退化** — UI 排版散乱无重点，且丢失了直接审批能力

## 设计原则

- 只有三个大页面：首页（员工）、管理工作台（管理员）、话术上传
- 除了这三个页面之间的切换，其他所有「查看详情」「处理审批」都用中心弹窗，不跳转
- 首页只有「找」和「上传」两个职责，零管理功能
- 管理工作台的每条待处理项可行内快速处理，也可弹窗处理

## 页面一：首页（员工视角）

**路由**：`/violations`

**职责**：找话术 + 上传入口

**结构**（从上到下）：

### 1.1 顶部 Header

- 标题「话术案例库」+ 副标题统计（团队沉淀 N 条案例）
- 右侧 CTA：「上传话术」按钮 → 跳转 `/violations/submit`
- 管理员额外显示「管理工作台」入口按钮 → 切换到 `?view=manage`

### 1.2 排行榜区域（保留）

- 两块并排：高转化话术 Top5 / 高风险话术 Top5
- 点击某条 → 中心弹窗看详情
- 风格不变，这部分目前做得可以

### 1.3 筛选栏

- 搜索框 + 导粉方式筛选 + 排序（转化率/通过率/使用次数/最新）
- 不需要 Tab 切换（「找可用话术」和「避坑指南」合并，用排序区分）
- 吸顶保留

### 1.4 话术列表（核心改动）

**整体是一块大白色卡片**（rounded-2xl border bg-white），内部是列表行：

每行结构：
```
┌─────────────────────────────────────────────────────────┐
│ [状态点] 话术文本（line-clamp-1）    [标签] [指标] [复制] │
└─────────────────────────────────────────────────────────┘
```

- 左侧：状态色点（推荐绿/可用蓝/待测试金/禁用红）
- 中间：话术文本，单行截断
- 右侧：类目标签 + 导粉方式标签 + 通过率或转化率数字 + 复制按钮
- 行间用极细分隔线
- 行高适中，信息密度高，一屏能看 10-15 条

**点击行** → 打开中心弹窗（Dialog），展示完整详情

**无审批按钮** — 员工页面不需要

### 1.5 话术详情弹窗（中心 Dialog）

弹窗内容：
- 话术全文
- 截图预览（如有）
- 状态标签（可用/推荐/禁用等）
- 统计指标（通过率/转化率/使用次数）
- 管理员结论（如有）
- 提交人 + 时间
- 底部操作：复制话术 / 关闭

弹窗宽度：max-w-lg 或 max-w-xl，居中

## 页面二：管理工作台

**路由**：`/violations?view=manage`

**职责**：审批闭环 — 处理员工提交的话术

**结构**：

### 2.1 顶部 Header

- 标题「管理工作台」
- 右侧：「员工视角」按钮 + 「替员工提交」按钮
- 待处理总数醒目显示

### 2.2 待处理区域（主体）

按优先级分组，每组是一个折叠面板：

**分组顺序**：
1. 高风险违规（红色标记）— 需立即处理
2. 缺数据案例（橙色标记）— 需补充信息
3. 待验证转化（绿色标记）— 需确认

每条待处理项是一行：

```
┌──────────────────────────────────────────────────────────────┐
│ [风险标记] 话术文本（截断）  [提交人] [时间]  [通过] [拒绝]  │
└──────────────────────────────────────────────────────────────┘
```

**行内快速操作**：
- 「通过」按钮 → 直接通过（默认 usage_state=available）
- 「拒绝」按钮 → 弹出拒绝原因选择（小弹窗/Popover）

**点击行本身**（非按钮区域）→ 打开中心弹窗，展示完整详情 + 完整审批面板：
- 话术全文 + 截图
- 审批决策区：状态（通过/拒绝）+ usage_state（可用/待测试/不推荐/禁用）+ 风险等级 + 推荐级别
- 管理员结论输入
- 确认提交按钮

### 2.3 已处理区域

- 折叠面板，默认收起
- 展开后显示最近处理的记录列表
- 每条显示：话术文本 + 处理结果 + 处理时间
- 支持撤销（5 秒反悔窗口保留）

## 页面三：话术上传

**路由**：`/violations/submit`

**职责**：提交新话术

**不改** — 现有的 4 步向导保持不变，这部分没问题。

## 交互规则总结

| 场景 | 交互方式 |
|------|----------|
| 首页 → 管理工作台 | Tab 切换（URL ?view=manage） |
| 首页 → 上传话术 | 页面跳转 |
| 首页点击话术行 | 中心弹窗（只读详情） |
| 排行榜点击某条 | 中心弹窗（只读详情） |
| 管理工作台点击待处理行 | 中心弹窗（详情 + 审批面板） |
| 管理工作台行内「通过」 | 直接操作，无弹窗 |
| 管理工作台行内「拒绝」 | Popover 选原因，非全屏弹窗 |

**统一规则**：除了三个大页面之间的导航，所有其他操作都在当前页面内完成（弹窗/行内操作）。

## 视觉要点

- 话术列表大卡片：rounded-2xl border border-zinc-200 bg-white，内部行用 border-b border-zinc-100 分隔
- 每行 hover 效果：bg-zinc-50，cursor-pointer
- 行内指标用 font-mono tabular-nums，右对齐
- 中心弹窗：Dialog 组件，max-w-xl，有遮罩层
- 管理工作台的行内按钮：小尺寸（h-7），通过用绿色调，拒绝用红色调
- 保持现有美学规范：4 语义色、rounded 层级、禁 spinner/transition-all

---

## 数据接口

不需要改后端 API，现有接口足够：
- 列表：`GET /api/violations`（已有排序/筛选/分页）
- 详情：`GET /api/violations/[id]`
- 审批：`POST /api/violations/[id]/review`
- 批量审批：`POST /api/violations/batch-review`
- 撤销：`POST /api/violations/review/restore`
- 排行榜：`GET /api/violations/dashboard-summary`
- 管理收件箱：RPC `case_library_inbox` / `case_library_inbox_counts`
