## 调研结论
1. **数据来源**：`loadDashboardPageData` 已返回 `todayReports`（含 `play_count` / `follower_gain` / `follower_convert` / `likes`/`comments`/`shares`/`favorites`），× 需要 Codex 写 RPC，前端本地聚合即可
2. **现状**：`DashboardWorkspaceHeader` 顶栏只有"今日生产流程"20px Title + 日期选择，下方 `AlertCenter` → `WorkflowDashboard`，× 主视觉锚点
3. **Display 冲突**：`dashboard/` 下 `text-[32px]` / `text-[28px]` grep 结果为 0，新建焦点卡后仅新组件使用 32px，满足硬规
4. **角色区分**：owner/admin 打开 /dashboard 时，焦点在管理/审核视角，× 需要员工维度锚点；本卡仅 `userRole === "member"` 渲染

## 焦点数字选择
- **主数**：今日累计播放量（sum of `play_count` across 本人所有账号）
  - 选择理由：比"当日转粉数"样本更稳；即使涨粉为 0 也有播放量可显
  - 万/亿自动转换单位，`tabular-nums`
- **空态**：今日未填（`todayReports.length === 0` 或总播放 = 0）→ 数字显示 "—"，Caption "今日首次填报后点亮"
- **副指标**（底部三列 13px · zinc-500）：转粉 / 涨粉 / 互动总量
- **Caption**：`{userDisplayName} · 今日节奏` + `已填报 X / Y 个账号`
