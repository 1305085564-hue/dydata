# DYData 信息架构重构计划

## 背景

GPT 诊断正确：网站功能按模块堆叠，未形成成员视频闭环流程。现在有 22+ 个页面/路由，组员、管理者、owner 进去都不知道干什么。

核心问题：**缺少产品主线**。功能是技术视角（违规库、AI助手、内容复盘），不是任务视角（我今天干什么、我怎么批改、系统健康吗）。

## 改造目标

**三个闭环 + 一个系统治理**，导航按角色任务重组：

| 闭环 | 面向角色 | 核心问题 |
|---|---|---|
| 组员执行闭环 | member | "我今天怎么把下一条视频做得更好" |
| 管理者批改闭环 | group_leader / team_admin | "我今天应该看谁、改谁、沉淀什么" |
| 经验沉淀闭环 | 全员 | 案例复用 |
| 系统治理 | owner | 系统健康、权限正确 |

## 当前状态

### 导航现状
- 顶部 NavBar：数据填报 / 成长分析 / 违规库 / AI助手(条件) / 后台管理(条件)
- 管理员侧边栏（/admin/*）：中控台 / 经营分析 / 视频资产 / 内容复盘 / 转化中心 / 权限模块 / AI配置
- 两套 admin 导航系统并存（admin-sidebar.tsx + admin-secondary-nav.tsx）

### 数据库现状
- **没有视频诊断表** — 这是最大空白
- 有 `account_reports`（日报）/ `videos`（视频）/ `next_day_reviews`（次日内容复盘）
- 有 `ai_feedback`（AI反馈）/ `violation_cases`（违规库）
- 组员每天提交的是 `account_reports` 数据

## 实施计划（分三阶段）

### 阶段一：信息架构重排（导航 + 命名）

**不做新功能，只改组织和入口**

1. **顶部导航重排**
   - 文件：`src/components/nav-bar-items.ts`
   - 新增"今日工作台"（改名自"数据填报"）
   - AI助手保持现状（已经是条件显示）
   - 后台管理入口不动

2. **/violations 页面文案升级**
   - 页面标题从"违规库"改为"话术案例库"
   - 说明文案同步更新
   - URL 和路由不动（改路由成本高，先只改文案）

3. **/dashboard 页面定位升级**
   - 从"数据填报页"升级为"今日工作台"
   - 保留现有填报功能
   - 补充：昨日反馈、下一条动作参考
   - 不改数据模型，只改页面内容组织

4. **管理员侧边栏精简**（可选，视阿禅反馈再动）
   - 合并视频资产 + 内容复盘 → "视频复盘台"（新建页面）

### 阶段二：新增视频复盘台（核心页面）

**这是整个改造的价值中枢。没有它，组员提交的数据无法转成反馈。**

1. **新建 `video_review` 表**
   - migration: `070_video_review.sql`
   - 字段：id, video_id, reviewer_id, diagnosis_result, key_issues, feedback_content, status (pending/confirmed/dismissed), created_at, updated_at

2. **新建 `/admin/video-review/page.tsx`**
   - 路由：`/admin/video-review`
   - 功能：看范围内视频列表 → 点进去 AI 初诊 → 人工复核 → 写反馈
   - 数据来源：`videos` 表 + `account_reports` 表

3. **AI 初诊能力**
   - 调用现有的 AI copywriting 接口
   - 基于视频标题/描述/数据生成诊断结果
   - 不新建 AI 通道，复用现有 `ai_copywriting`

4. **权限**
   - `review_diagnosis` 权限键（新增到 types/index.ts）
   - `group_leader` 默认开启此权限

### 阶段三：管理中控台重做

1. **重做 `/admin` 页面**
   - 从"后台功能堆积"改为"今天三个问题"：
     - 谁没交？
     - 谁需要诊断？
     - 哪些反馈待复验？
   - 不动 `admin-cockpit.tsx` 的基础结构，只改数据聚合逻辑

2. **组长权限补全**
   - 修改 `business-role.ts` 的默认值
   - `group_leader` 额外默认开启：`review_diagnosis`（新增）
   - 不破坏现有高风险权限边界

## 改造优先级

**P0（核心闭环）**：视频复盘台（新建 + DB表）
**P1（体验提升）**：顶部导航文案 / dashboard 定位 / violations 文案
**P2（管理效率）**：admin 简化 / 组长权限补全
**P3（系统治理）**：管理员侧边栏重组 / 孤岛页面清理
