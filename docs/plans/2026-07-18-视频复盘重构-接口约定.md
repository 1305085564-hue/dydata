# 视频复盘重构 · 接口约定

> Codex 后端 / Antigravity 前端并行的唯一契约。字段名、结构以本文为准，任一方改动先同步这里。产品背景见「产品设计总纲」。

## 通用规则
- 所有接口走现有 `requireScopedAdminVideo` / 管理员权限校验，沿用现状。
- 金额/比率单位沿用现状：`bounce_rate_2s` / `completion_rate_5s` / `completion_rate` 为百分数（如 22.5 表示 22.5%），`avg_play_duration` 为秒。
- 缺数据一律返回 `null`，不返回 0 冒充；前端见 `null` 显示"缺数据"。

## 一、归因结论带（新增计算，死规则，不走 AI）

### `GET /api/admin/content-attribution/:videoId?ref=self|team|top|user&refUserId=xxx`
- `ref`：对比参照系，默认 `self`（本账号近 3 条）。`team`=团队均值，`top`=今日团队流量最好者，`user`=指定人（需 `refUserId`）。
- 返回：
```jsonc
{
  "video_id": "…",
  "ref": "self",
  "ref_label": "对比自己近3条",
  "findings": [                     // 已按严重度降序排好，前端直接渲染
    {
      "metric": "completion_rate_5s",
      "metric_label": "5s完播",
      "value": 22.0,
      "ref_value": 40.0,
      "delta": -18.0,               // 相对参照系，百分点或百分比
      "tone": "bad",                // good | warn | bad | missing
      "points_to": "开头钩子弱，没给继续看的理由",   // 来自映射表
      "locate": {                   // 指向内容哪，前端据此高亮/跳转
        "kind": "segment",          // segment（到段/句） | attribute（到属性）
        "segment_hint": "opening",  // opening|middle|ending，segment 时给
        "seconds": 5                // 拐点/均播秒数，算得出才给，否则 null
      }
    }
  ],
  "missing": ["shares"],            // 缺数据的指标 key，前端标"缺数据"
  "snapshot_ready": true            // false 时前端走"归因待数据齐"兜底
}
```
- **计算规则见 Codex 规格 §2**。`points_to` / `locate.kind` 严格按总纲 §4 映射表，后端硬编码，不调 AI。

## 二、多参照系对比（改造现有接口）

### `GET /api/admin/content-comparison/:videoId?ref=self|team|top|user&refUserId=xxx`
- 现有接口只返回 `previous` + `recent3`（且 `recent3` 被前端错标"团队"）。
- 改造：按 `ref` 返回对应参照系的指标均值行，结构复用现有 `MetricRow`：
```jsonc
{ "video_id":"…", "ref":"team", "ref_label":"团队均值", "current": {…}, "reference": {…}, "ref_count": 12 }
```
- `current`=本条快照，`reference`=参照系均值（self=本账号近3条，team=团队今日均值，top=今日最高播放者，user=指定人近3条）。

## 三、可复测：上次反馈卡

### `GET /api/admin/content-feedback-cards/:videoId/previous`
- 返回同一账号"上一条已下发"的反馈卡摘要 + 那条的关键指标，供前后对比：
```jsonc
{ "has_previous": true, "previous": { "card_id":"…", "one_line":"开头留人弱", "sent_at":"…", "metrics": {…} } }
```
- 无历史返回 `{ "has_previous": false }`。

## 四、沿用不变（前端照现状调）
- `PATCH /api/admin/content-feedback-cards/:videoId`：存草稿/确认下发，动作与状态机不变。
- `POST /api/admin/content-analysis`：AI 辅助分析，降为可选按钮触发，返回结构不变。
- `content-observations` 相关：走势下拉那套**前端下线**，后端接口可留但归因舱不再调。

## 五、错误码
- `403` 越权、`404` 视频不存在、`400` 参数错（如 `ref=user` 缺 `refUserId`）。
- 归因/对比算不出（无快照）返回 200 + `snapshot_ready:false`，不报错，前端走兜底。
