# 2026-05-21 话术案例库前端重构施工档案

## 背景
- 原 `/violations` 是「双视角 tab + 一堆筛选 + 卡片列表」，员工进来不知道做什么；`/admin/conversion-hub` 是 5 tab 平铺（话术/复核/周报/分析/建议），其中 3 tab 是空壳；详情页只有单维度 status badge，无法表达「已审核但禁用」「推广候选」。
- Codex 已完成后端：迁移 071/072（usage_state + promotion_level + result_flag + case_library_inbox / counts RPC），新增 /api/case-library/inbox|counts，/api/violations/[id]/test 改为 deprecated forwarder（写到 usage-records）。
- 前端按 brainstorming 三屏判断重构（× 修补，按真实使用动作重写）。

## 数据库迁移
- 071_case_library_states.sql：新增 `violation_cases.usage_state` / `promotion_level`，回填历史；script_usage_records 新增 result_flag。
- 072_case_library_inbox_rpc.sql：`case_library_actor_scope` / `case_library_inbox` / `case_library_inbox_counts` 三个 RPC。
- 执行方式：Claude `supabase db push --linked --include-all`，对 mkkvnogkqcupvxmnoefy（孟买生产库）直连应用。
- migration list 验证：071/072 在 local + remote 双方 ✓。

## 前端改造

### A. /violations 员工首屏（重构）
- 删 perspective tabs / status 筛选 / case-filters / status-badge（员工无关）
- 搜索 hero 占主位（`StaffSearchHero`）：大输入框 + 「找一下」按钮 + 「没找到，交一条」CTA
- 条件 banner：仅当存在 promoted/testing 案例才显示
- 卡片列表：左 2px 导轨 = usage_state（推荐绿/可用蓝/待测试金/禁用红/× 推荐灰）+ 通过率 + 类目 + 管理员结论摘录
- 数据源：/api/violations?view=staff（已过滤 banned + status≠verified）
- 新增 `staff-search-hero.tsx`，rewrite `case-card.tsx`、`page.tsx`
- 删 `case-filters.tsx`、`perspective-tabs.tsx`、`status-badge.tsx`

### B. /admin/conversion-hub 单页流（重构）
- 删 SectionNav / hub-shell 旧版 / tabs/{violations,scripts,advice}-tab.tsx / weekly/view.tsx / analytics/view.tsx
- 单页结构：4 横排 KPI（待审核/高风险/缺数据/推广候选）+ 4 折叠组（5 条预览 + 「展开剩余 N 条」）+ 话术资产 KPI + TOP10
- 数据驱动：直接 RPC 调 `case_library_inbox` + `case_library_inbox_counts`（service-role server-side）
- 新增 `loadInboxData()` server loader，简化 `data.ts`：保留 ScriptsTabData/loadScriptsTab + InboxData，删 loadViolationsTab/loadWeeklyTab/loadAnalyticsTab/loadConversionHubData
- weekly/, analytics/, /admin/violations/page.tsx 三个老路由保留为 redirect stub → /admin/conversion-hub

### C. /violations/[id] 详情页四维度（重构）
- 1 主徽 = `usage_state`（UsageStateBadge，h-8 size=lg）
- 副信息行 = `status` / `risk_level` / `promotion_level` 三个 chip（PromotionLevelChip 仅在非 normal 时渲染）
- 新增 `case-state-badge.tsx`，导出 4 个组件（UsageStateBadge / ReviewStatusChip / RiskLevelChip / PromotionLevelChip）
- TestRecordForm rewrite → POST /api/conversion-hub/usage-records（result_flag pass/fail + 必填 used_at + views/follows + note）；× 再走 deprecated /test
- 新增 `review-decision-panel.tsx`：决策树式审核面板（采纳 / 驳回二选一 → usage_state 四选一 → 风险等级 → 可选推广 owner-only），管理员可见，一次性写四维度
