## 一、项目目标

把 dydata（达人数据生产操作台）**整站前端视觉语言**从原先混乱的多色系/重字重/大圆角/毛玻璃风格，统一重塑为**阿禅美学法典 V1** 定义的低调克制审美体系。

**法典核心 13 条纲领**（详见 `/docs/阿禅美学标准-V1.md`）：
1. 留白优先 — 宁疏勿密，多用 zinc-50/white 呼吸空间
2. 层级靠灰阶 — zinc-400/500/700/800 四级文字灰度
3. 字号阶梯 — `10px uppercase tracking-[0.25em]` 次要 / `13px leading-[1.7]` 正文 / `20px semibold tracking-tight` 主标
4. 色值克制 — CTA 主色 `#D97757` 暖橙；语义色 `#6FAA7D`（成功）/ `#D99E55`（警示）/ `#C9604D`（异常）/ `#8AA8C7`（信息）
5. 圆角阶梯 — 按钮 `rounded-[10px]` / 输入 `rounded-lg` / 内卡 `rounded-xl` / 主面板 `rounded-2xl`；禁 `rounded-3xl+`
6. 阴影弥散 — 只用 `shadow-sm` (≤3% 透明度)；禁 `shadow-lg/xl/2xl` 和彩色 glow
7. Ring 细光环 — 统一 `ring-1 ring-zinc-950/5`；禁 `ring-3/4` 和彩色 ring
8. 交互低调 — `hover:-translate-y-[1px] active:translate-y-0` + `transition-[具体属性] duration-150 ease-[cubic-bezier(0.4,0,0.2,1)]`；禁 `transition-all`、`hover:scale-*`
9. 字重收敛 — 最高 `font-semibold`(600)，禁 `font-black/extrabold/bold`(≥700)
10. 状态点化 — 6px 小圆点 + 文字色替代大面积彩色背景
11. 空/加载/错态统一 — `<Skeleton />` / `<EmptyState>` / `<ErrorState>`，禁 `animate-spin` + `Loader2`
12. 细节从简 — icon `stroke-[1.5]` + size 统一
13. **禁区硬规** — 以上各条对应的禁用模式（批量 grep 可自检）

## 八、后续优化可能方向（阿禅接手）

### 8.1 交互细节打磨
- 某些按钮 hover 位移 1px 是否够"手感"，可调 `transition-duration`（150 → 180ms 更顺滑但更慢）
- Focus ring `ring-1 ring-zinc-950/5` 在白底上是否够明显（可考虑 `ring-zinc-950/10`）
- 表单提交 loading → Skeleton 替换后，用户感知是否清晰（某些场景可考虑 disabled + opacity）

### 8.2 视觉节奏微调
- 当前行间距 `leading-[1.7]` 是否全站都合适（长文本区域可能需要更松）
- 状态点 6px 是否都够醒目（小卡片里可能需要 8px）
- 卡片间距 `space-y-5/6` 在宽屏上是否够松

### 8.3 数据可视化层
- recharts 当前用的 `#D97757/#8AA8C7/#6FAA7D/#D99E55/#C9604D` 5 色调色板，超过 5 条线可能不够用，可扩展次色阶
- 坐标轴字重、刻度密度、tooltip 样式是否统一
- Gauge/Radial 等特殊图表的色值落实

### 8.4 响应式 / 移动端
- nav-bar-client 在移动端是否够可用（滚动条隐藏已做）
- admin-sidebar mobileOpen 覆盖层是否 z-index 冲突
- dashboard 表单在小屏下的字段密度

### 8.5 深色模式（如需）
- 当前 token 层未准备 dark 变量组，若要加需复审整个色值系统

### 8.6 可访问性
- WCAG 对比度（法典 `#D97757` on 白底 / `#8AA8C7` on 白底需手动测过）
- 键盘焦点顺序 / `aria-label` 完整性
- 屏幕阅读器对状态点 + 文字色的理解

### 8.7 性能
- production-control-system 拆完后是否有多余 re-render（可 React DevTools Profiler 看）
- 大列表（history-list / violations）是否需虚拟滚动

## 九·末 · V1.1 后仍未到位的问题（V2 讨论输入）

### 9B.1 "过度中性化"的灵魂问题

即使 Gemini 的具体点位误判，**"骨架干净、灵魂缺席"**的整体感觉仍值得回应：

- **主 CTA 锚点存在但分散**：暖橙 `#D97757` 站内 30+ 处出现，有的用在状态点、有的 tab 激活、有的按钮;**未建立"每屏只有 1 个暖橙主 CTA"的纪律**,识别力被稀释
- **展示型页面（growth / analytics / conversion-hub）无"主动作"**：灰阶 + 数据密度是对的，但是否引入"下一步建议行动"的暖橙锚点，把被动阅读变成主动行动？—— V2 决策
- **首屏"视觉焦点层级"未定义**：法典 13 条讲"局部元素规则"，未讲"整屏阅读顺序"（eyebrow → 标题 → 关键指标 → 次级内容 → 操作区），需补"视觉节奏"章节

### 9B.2 状态点"仪表盘质感"未建立

- 状态点**无光晕 / pulse / 呼吸动效**作为"活跃中"的区分
- 完成态 / 进行中 / 待处理三态只靠颜色，**无亮度 / 饱和度梯度**
- 多状态点并列时**缺"优先级等级"**（1 个红点 vs 3 个灰点，红点是否应有视觉权重加成）
- V2 可考虑"状态点语言"系统化：`.status-dot-[critical|active|idle|done]` 带 :before 光晕 / 呼吸

### 9B.3 图表"高级感"仍有差距

- **坐标轴 / 刻度 / tooltip 样式未完全统一**（各图表仍自定义 Tooltip）
- **多条线超过 5 条时的次色阶未定义**
- **无数据态 / 加载态 / 异常态**缺乏统一样式
- **柱状图、饼图、Gauge、Radial** 等特殊图表型未在 `chart-palette.ts` 规范
- V2 需要：统一 Tooltip 样式（白底 + `shadow-sm` + `ring-1 ring-zinc-950/5`）、图例位置与样式规范、空态模板

### 9B.4 字号阶梯不够细

法典当前 3 档（10px eyebrow / 13px 正文 / 20px 主标），实际使用发现：
- **缺 11-12px 档**（表格内容 / 小卡片标题 / 次级 Tag）
- **缺 16-17px 档**（卡片主标 / 列表主文案）
- **缺 24-28px 档**（大号指标数字 / 核心看板数字）
- 导致开发时硬编码 `text-[11px]` / `text-[14px]` / `text-2xl` 等自创档位，出现不一致
- V2 需要：补齐 6-7 档字号，同步定义"用法场景"

### 9B.5 间距系统缺失

- `gap-2/3/4/5/6` 和 `space-y-2/3/4/5/6` 在站内混用
- 卡片内 `p-4/5/6`、卡片间 `space-y-4/5/6` 无纪律
- V2 需要：定义 `2/3/4/6/8` 五档间距规范（4px baseline），明确用法

### 9B.6 暗色模式 / 可访问性仍缺席

- 法典 token 未准备 `dark:` 变量组
- `#D97757` on 白底对比度 4.3:1（WCAG AA 最低 4.5:1，临界）;`#8AA8C7` on 白底 2.9:1（不过 AA）—— 小字体场景需要更深色阶
- V2 是否纳入 dark mode，需要决策

### 9B.7 微交互缺"物理感"

- 按钮 hover 只 `-translate-y-[1px]`,缺**按压反馈**（active 微缩 0.98 或 translate-y-0.5）
- 表单 Focus 动效（`bg-zinc-100/70 → bg-white`）**无先导微闪**
- 弹窗只做 fade+scale,无"从触发点展开"的**源头动画**
- V2 可补"微交互章节"
