# 批次一（合并版）· Antigravity 前端规格

> 你负责美学 + 前端实现。**从零重写，删掉项目里 Codex 的临时/简陋前端，绝不参考它的前端代码**（参考会被污染）。
> 你只依赖《Codex 后端规格》第 5 节的接口约定对接数据；数据怎么算、怎么落库不用管。
> 遵守本项目《美学规范》（`docs/美学规范.md`）。红涨绿跌（涨=红，跌=绿，A股惯例）。

## 0. 你要做出的东西（一句话）
今日工作台改成"传 + 看"一页两态：传是主，传完**自动翻到看**；看态分三类；随时能舒服地切回去补传。异常态提交时顺手补违规详情。

## 1. 页面骨架：两态

### 传态（默认进来就是它）
- 聚焦、干净，进来只干"提交今日数据"这一件事
- 外层状态开关只有两档：**正常 / 异常**
- 正常态：文案 + 全量指标 + 截图（沿用现有字段，版式重排做漂亮）
- 异常态（选"异常"后）：
  - 文案照填（异常也要传文案，文案本身就在这填）
  - 展开一块"违规详情"（来自原导粉中心提交页第2步）：**处罚类型**（下拉，原"限流/删稿"等都在这）、**平台通知**、**申诉**
  - **不要"发生时间"字段**（已砍）
  - 截图、完播留存四项：异常态改为可选/默认折叠
- 提交成功 → 见第 3 节动线

### 看态（传完自动翻过来）
三类内容（主次分明，见第 2 节）：
1. 看自己
2. 看大盘
3. 其他补充

## 2. 看态三类

### ① 看自己（**考勤为主**，第一眼给这个）
- 顶部第一眼：**今天交没交**
- **本月日历**：这个月哪些天交了、哪些没交（一眼扫完）
- 就地操作：某天可**补交**、可**给那天申请豁免**
- 往下沉（次级，想看再看）：能力体检 = 原"个人成长"那套（雷达/趋势）。合并进来做次级，不再是独立入口。

### ② 看大盘（趋势 + 排行榜收进来）
- 原"趋势查看""排行榜"不再是独立按钮，收成看态里的一块
- **先给结论**：我在公司排第几、跟优质作品差在哪，一句话说清；想深看再展开明细图表
- 别一上来堆一屏图

### ③ 其他补充（豁免 / 历史）
- 豁免、历史记录：阿禅**要员工真的用**，所以**入口放大**、显眼固定位置（不是藏进第三层下拉），但不抢主视觉

## 3. 关键交互动线（这是体验重点，务必照做）
传完提交成功后：
1. **自动翻到看态**（不用用户点）
2. 落地那一刻，冒一个**醒目的"再传一条 / 继续填"按钮** —— **几秒后自动淡出**
3. 淡出后，看态里长期常驻一个**低调的返回上传入口**（一直在，不抢戏）

一句话：刚落地给显眼的提醒式入口，几秒收起；之后靠常驻的低调入口回去补传。连续传的人爽，只传一条的人不被打扰。

## 4. 边界（别做这些）
- 别碰管理端页面（批改台/经营分析/素材库/发布履约）
- 别碰视频审核、文案助手
- 别自己发明后端字段；缺字段找接口约定，没有就记下来反馈，别硬编

## 5. 交付标准
- 传态(正常/异常两档) + 看态(三类) + 自动翻+淡出+常驻返回，全部按上文实现
- 响应式 + 可访问性达《美学规范》要求
- 你实现前端即可；**是否真正接通后端由 Codex 验收**（后端他做的，他最清楚前端有没有连通链路）
