# 批次三 · 选题库 · Antigravity 前端规格

> 分工：Antigravity 只负责前端（从零搭，不参考视频审核卡片墙）。后端接口由 Codex 提供，接口约定见 `2026-07-16-批次三-选题库-Codex后端规格.md` §3。
> 遵守《美学规范》（`docs/美学规范.md`）、红涨绿跌。
> **重要**：绝不参考 Codex 的任何临时/简陋前端代码，必须从零重写。

## 0. 总览

选题库前端只有 **两个页面**：
1. 今日工作台（首页）
2. 选题池（含子题详情）

加一个**全局录入入口**（任何页面可触发）。

下面按独立任务拆分，**每个任务可以单独完成、单独验收**，不依赖其他任务的完成顺序（除非标注了前置依赖）。

---

## 任务 A：全局录入组件

**做什么**：一个全局悬浮按钮或顶栏入口，点击弹出极简录入面板。

**交互**：
- 点击后弹出小面板（不是整页跳转）
- 面板内容：
  - 母题选择（8个按钮/下拉，高频在上低频在下，顺序见产品规格）
  - 一句话输入框（placeholder："写下你的选题想法"）
  - 提交按钮
- 提交后面板自动收起，给一个轻量成功提示（toast）
- 母题必选，一句话必填，其余不展示

**调用接口**：`POST /api/topics/sub-topics`（只传 title=一句话, hook=一句话, topic_id=所选母题）

**验收标准**：
- 任何页面都能触发
- 3 秒内完成录入（从点开到提交）
- 母题 8 项排列正确（高频上低频下）
- 提交后面板收起 + toast 提示

---

## 任务 B：今日工作台页面

**做什么**：选题库首页，展示"最近活跃"的选题。

**交互**：
- 页面分三个区块（可以是卡片组或列表）：
  - "最近被认领的选题"（N条）
  - "最近有新作品的选题"（N条）
  - "最近新录入的选题"（N条）
- 每条选题展示：子题名 + 一句话 + 母题标签 + 汇总数据（平均播放量，没有则不显示）+ 认领人数
- 每条可操作：点击进入选题详情 / 一键认领

**调用接口**：`GET /api/topics/active`

**验收标准**：
- 三个区块正确渲染，数据来自接口
- 空状态处理（某个区块没数据时显示"暂无"而非空白）
- 认领按钮调用 `POST /api/topics/sub-topics/:id/claim`，成功后刷新状态
- 候选满 5 条时认领按钮置灰 + 提示"已达上限"

---

## 任务 C：选题池页面 — 列表与筛选

**做什么**：选题池主列表页面，支持筛选和视图切换。

**交互**：
- 顶部筛选栏：
  - 视图切换：全部 / 我正在做的 / 我提交的（三个 tab 或按钮）
  - 时间筛选：近 3 天 / 本周 / 本月 / 近 3 个月（下拉或按钮组）
  - 母题筛选：8 个母题可选（可多选或单选过滤）
- 列表区：
  - 按母题折叠浏览（母题名作为分组标题，点击展开/收起该母题下的子题）
  - 每条子题卡展示第一层级信息：子题名 + 一句话 + 汇总数据 + 认领人数

**调用接口**：`GET /api/topics/pool`（传 view / time_range / topic_id 参数）

**验收标准**：
- 三个视图切换正常
- 时间筛选和母题筛选能正确过滤结果
- 母题折叠/展开交互流畅
- 空状态友好提示
- 列表支持滚动加载或分页

---

## 任务 D：子题卡 — 三级信息展示

**做什么**：子题卡的交互组件，支持三级渐进展示。

**前置依赖**：任务 C（列表里用到这个组件）

**交互**：
- **第一级（默认收起态）**：子题名 + 一句话 + 母题标签 + 汇总数据（≥1000流量作品平均播放）
- **第二级（点击展开或hover）**：最好的文案版本（播放最高的）+ 最近的文案版本 + 各自对应的视频数据（播放量/完播率）
- **第三级（点击"查看详情"进入子题详情页）**：历史所有文案版本列表 + 各自数据 + 相近选题的文案（标记"相近参考"）

**数据说明**：
- 第一级数据来自 pool 接口
- 第二级数据来自 pool 接口的 works 摘要字段
- 第三级数据调用 `GET /api/topics/sub-topics/:id/works`

**验收标准**：
- 三级渐进展示层次清晰
- 没有文案/作品数据时优雅降级（显示"暂无作品数据"）
- 数据展示遵守红涨绿跌（播放量对比时，高于平均用红色标注）

---

## 任务 E：子题详情页

**做什么**：子题完整详情页，整合所有信息和操作。

**前置依赖**：任务 D（复用子题卡组件）

**交互**：
- 顶部：子题名 + 一句话 + 母题 + 情绪标签（如有）
- 操作区：认领 / 标记脚本中 / 放回选题池
- 文案收纳区：
  - 所有历史文案版本，每版显示：文案内容摘要 + 发布号 + 发布时间 + 播放量/完播率/互动/转粉
  - 排序支持：按播放量 / 按时间
  - 相近选题文案单独分组展示（标记"相近参考"）
- 认领状态区（收起，低频）：
  - 目前有 N 人在做
  - 展开可看具体是谁、各自状态

**调用接口**：
- `GET /api/topics/sub-topics/:id`（基本信息）
- `GET /api/topics/sub-topics/:id/works`（文案+数据）
- `POST .../claim` / `POST .../start-scripting` / `POST .../return`（操作）

**验收标准**：
- 所有信息正确渲染
- 三个操作按钮根据当前用户状态显示/隐藏（未认领→显示认领；已候选→显示脚本中+放回；已脚本中→显示放回）
- 文案排序切换正常
- 相近选题文案正确标记和分组

---

## 任务 F：数据上传页"关联选题"字段

**做什么**：在批次一的数据上传表单里加一个"关联选题"字段。

**交互**：
- 位置：数据上传表单的合适位置（不阻塞主流程）
- 默认行为：调用 `GET /api/topics/sub-topics/suggest?title=xxx&content=xxx`，根据用户已填的标题和文案，系统自动推荐最可能的子题并**默认选中第一个**
- 运营可以：确认 / 换一个 / 选"暂无对应" / 一键"新建选题"
- 如果是从选题库认领路径来的（URL 带 topic_id 参数），直接锁定显示，不用再选

**调用接口**：
- `GET /api/topics/sub-topics/suggest`（兜底匹配）
- 上传时将 topic_id 传入 `POST /api/video-submit`

**验收标准**：
- 系统推荐在标题/文案输入后自动触发（防抖）
- 默认选中第一个推荐
- 从认领路径来的自动锁定
- "暂无对应"和"新建选题"选项可用
- 不影响上传表单原有功能

## 通用要求

- 遵守 `docs/美学规范.md`
- 红涨绿跌（数据对比时，高于均值/上涨=红色，低于/下降=绿色）
- 从零搭，绝不参考视频审核卡片墙或 Codex 临时前端代码
- 所有接口错误需要友好提示（toast），不能白屏
- 移动端适配（运营可能手机上快速录入）
- 路由建议：`/topics`（选题池）、`/topics/today`（今日工作台）、`/topics/:id`（子题详情）

---

## 交付顺序建议

按独立性和优先级排序：
1. **任务 A**（全局录入）— 最简单最独立，可以最先做
2. **任务 B**（今日工作台）— 独立页面
3. **任务 C**（选题池列表）— 独立页面
4. **任务 D**（子题卡组件）— 给 C 用
5. **任务 E**（子题详情页）— 依赖 D
6. **任务 F**（关联选题字段）— 改批次一的页面，最后做

每个任务完成后可以单独验收，不用等全部做完。
