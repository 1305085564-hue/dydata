# 批次三 · 选题库 · Codex 后端规格

> 分工：Codex 负责全部后端逻辑/数据/链路 + 验收前端是否对接正确。Antigravity 负责前端（从零搭，不参考视频审核卡片墙），由另一份规格单独指导。
> 本批 = 选题库新建（全站唯一 net-new）。
> 产品设计规格见：`2026-07-15-批次三-选题库-产品设计规格.md`（上）+ `...-下.md`。

## 0. 一句话目标

建出"存+选"选题库的完整后端：数据表、CRUD接口、状态流转、作品关联与文案收纳、选题池筛选、"最近活跃"工作台数据源、与批次一数据上传的对接。

## 1. 本批范围（务必只做这些）

做：
- 母题/子题/作品三层数据表
- 子题 CRUD（录入/编辑/删除）
- 认领/放回选题池的状态流转（人×子题，候选→脚本中→放回）
- 候选上限 5 条的校验
- 选题池查询接口（时间筛选 + 母题折叠 + 三个视图）
- 今日工作台"最近活跃"数据接口
- 作品关联：从批次一 `topic_id` 读入 + 兜底匹配推荐
- 子题卡汇总数据（≥1000流量作品的平均播放量）
- 文案收纳（多版本，从视频表文案字段读取聚合）
- 分组自动归类（根据子题名+一句话，NLP或关键词匹配）
- 与批次一接口对接（确保 `topic_id` 坑位可用）

不做（留后面，禁止顺手改）：
- 智能推荐算法/AI评分
- 复盘界面或复盘数据接口
- 全网热点抓取
- 文案助手联动/自动创建脚本
- 认领超时释放
- 精细权限
- 管理端改名/提入口（批次二）
- 视频审核归档/文案助手减法（批次三扫尾件）

## 2. 数据表设计

### topics（母题）
```sql
id              uuid PK
name            text NOT NULL        -- 母题名
sort_order      int                  -- 高频在上低频在下
created_at      timestamptz
```

### topic_groups（分组）
```sql
id              uuid PK
topic_id        uuid FK → topics     -- 归属母题
name            text NOT NULL        -- 分组名（如"图形战法"）
sort_order      int
```

### sub_topics（子题）
```sql
id              uuid PK
title           text NOT NULL        -- 子题名
hook            text NOT NULL        -- 一句话选题
topic_id        uuid FK → topics     -- 归属母题
group_id        uuid FK → topic_groups NULL  -- 分组（系统自动归，可空）
emotion_tag     text NULL            -- 情绪标签（恐惧/贪婪/焦虑/迷茫/认同/好奇）
source          text NULL            -- 来源（manual/system_recommend/asset_library）
audience        text NULL            -- 受众画像（beginner/anxious/mature/high_net_worth）
created_by      uuid FK → profiles   -- 创建人
created_at      timestamptz
```

### sub_topic_claims（认领记录 = 人×子题的状态）
```sql
id              uuid PK
sub_topic_id    uuid FK → sub_topics
user_id         uuid FK → profiles
status          text NOT NULL        -- 'candidate' | 'scripting' | 'returned'
claimed_at      timestamptz
returned_at     timestamptz NULL     -- 放回时间
```

约束：同一 user_id 下 status='candidate' 的记录不超过 5 条。

### 作品层：复用现有 videos 表

不新建作品表。作品 = videos 表里 `topic_id IS NOT NULL` 的记录。

批次一已定义 videos 表增加 `topic_id uuid FK → sub_topics NULL`。文案字段复用 videos 已有的 `content` 字段。数据字段复用 `video_metrics_snapshots`。

## 3. 接口清单

### 子题 CRUD
- `POST /api/topics/sub-topics` — 创建子题（必填：title, hook, topic_id）
- `GET /api/topics/sub-topics/:id` — 获取子题详情（含文案收纳数据）
- `PATCH /api/topics/sub-topics/:id` — 编辑子题
- `DELETE /api/topics/sub-topics/:id` — 删除子题（仅创建人可删，有作品关联的不可删）

### 认领/放回
- `POST /api/topics/sub-topics/:id/claim` — 认领（校验候选上限 5 条）
- `POST /api/topics/sub-topics/:id/start-scripting` — 候选→脚本中
- `POST /api/topics/sub-topics/:id/return` — 放回选题池（status→returned）

### 选题池查询
- `GET /api/topics/pool` — 选题池列表
  - 查询参数：`view`（all | my_claims | my_created）、`time_range`（3d | 1w | 1m | 3m）、`topic_id`（按母题筛选）
  - 返回：子题列表 + 每条的汇总数据（≥1000流量作品平均播放）+ 认领人数 + 最好/最近文案摘要

### 今日工作台
- `GET /api/topics/active` — 最近活跃
  - 返回三组：最近被认领的 / 最近有新作品的 / 最近录入的（各取最新 N 条）

### 子题详情·文案收纳
- `GET /api/topics/sub-topics/:id/works` — 该子题下所有作品（含文案+数据）
  - 排序：最好（按播放量）/ 最近（按发布时间）
  - 过滤：≥1000 流量的才参与汇总计算
  - 含"相近选题"的作品（同母题+同分组下其他子题的作品，标记为"相近参考"）

### 兜底匹配
- `GET /api/topics/sub-topics/suggest?title=xxx&content=xxx` — 根据标题/文案猜最可能的子题
  - 返回 top 3 候选，前端默认选中第一个

### 分组自动归类
- 子题创建/编辑时，后端自动根据 title + hook 匹配 topic_groups（关键词匹配优先，后续可升级 NLP）
- 匹配不上则 group_id 留空

## 5. 与批次一的对接

- 确认批次一 `POST /api/video-submit` 已预留 `topic_id` 入参（可空）。
- 选题库上线后，前端在数据上传表单加"关联选题"字段，调用 `/api/topics/sub-topics/suggest` 获取推荐，默认选中第一个，运营确认/改/选"暂无对应"。
- 从选题库认领路径发起的上传，`topic_id` 由前端自动带入，运营无感。
