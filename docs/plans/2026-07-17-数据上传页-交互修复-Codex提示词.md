## P1（6 项，主流程出错或误导）

### P1-1 "重新识别"按钮点了没反应
- 位置：`src/components/submission/截图槽位区.tsx:212-222`（对所有 failed 状态都显示重试按钮）+ `video-submit-form.tsx:1236-1242`（`handleSlotRetry` 只在 `slot.error === NETWORK_RETRY_MESSAGE` 时才真的重试，OCR 失败直接 return）。
- 修法：改 `handleSlotRetry`——只要 `slot.status === "failed"` 且 `slot.file` 存在就重新走 `handleSlotUpload(role, slot.file)`（OCR 失败和服务端返回 failed 的分支里 file 都保留着，可直接重试）；file 不在时按钮本来就不该起作用，保持静默或 toast 说明均可。
- 验证点：OCR 识别失败的槽位点🔄能重新识别；网络错误重试行为不变。

### P1-2 点推荐选题 chip 后，继续打字被系统抢走选择权
- 位置：`video-submit-form.tsx:1989-2000`，建议 chip 的 onClick 只 `updateMeta("topicId", s.id)`，没设 `isManuallySet`（对比：搜索弹窗选中在 2278 行设了、"暂无对应"在 1935 行设了）。
- 修法：chip onClick 里补 `setIsManuallySet(true)`。

### P1-3 "认领锁定"可以绕过
- 位置：URL 锁定逻辑 513-523；三个操作按钮（换一个/暂无对应/新建选题）1922-1950 无条件渲染；`handleNewTopic` 552-576 不判断锁定。
- 修法：
  1. `urlLocked` 为 true 时，隐藏这三个按钮（锁定态只保留选题卡片 + Lock 徽章）。
  2. `handleNewTopic` 开头加判断：锁定中或已有手动选择的选题时直接 return，不再自动绑定。注意闭包旧值问题，用 ref 读最新的 `urlLocked`/`isManuallySet`/`meta.topicId`。

### P1-4 恢复草稿冲掉认领锁定的选题
- 位置：`handleRestoreDraft` 747-761，`setMeta(draft.meta)` 整体替换。
- 修法：恢复时若处于 URL 锁定态，保留当前锁定的 topicId：
  ```ts
  setMeta((current) => ({
    ...draft.meta,
    topicId: urlLocked ? current.topicId : draft.meta.topicId,
  }));
  ```
  锁定态下也跳过 isManuallySet 的恢复（见 P2-15）。

### P1-5 选题名加载失败永久"获取选题信息中..."+ 张冠李戴
- 位置：`fetchTopicName` 526-549；展示处 1961-1963。
- 现状两个病：(a) 查询报错只 console、查不到数据（题被删/URL id 无效）时不做任何事 → 永远显示"获取选题信息中..."；(b) topicId 从 A 切到 B 时旧名字不清，B 查询失败就一直显示 A 的名字。
- 修法：
  1. effect 开头先清空 `selectedTopicName`/`selectedTopicCategory`，再发请求。
  2. 新增错误状态（如 `topicNameError: "not_found" | "load_failed" | null`）：`data` 为 null → `not_found`；catch → `load_failed`。
  3. UI：选题卡片区域按状态显示——`not_found` 显示"未找到该选题（可能已被删除）"+ "清除关联"小按钮（点击后 topicId=null、urlLocked=false、isManuallySet=true）；`load_failed` 显示"选题信息加载失败" + "重试"小按钮（用一个 retrySeq state 加进 effect 依赖重新触发）。样式沿用现有小字按钮风格，× 新设计。

### P1-6 Esc 键全局冲突
- 位置：1495-1524 window 级 keydown，Esc 无条件切换截图列表/指标核对面板——弹窗（搜索选题/删除确认/全局新建选题）开着时按 Esc 会同时关弹窗+翻面板；输入框里按 Esc 也触发。
- 修法：Esc 分支前加两道闸：
  1. 目标元素是 INPUT/TEXTAREA/contentEditable → 不处理（现有代码对 cmdEnter 已有 target 判断，Esc 同样适用）。
  2. 页面存在打开的弹窗 → 不处理。判断方式：`document.querySelector('[role="dialog"]')` 非空即返回（shadcn/radix Dialog 渲染 role="dialog"，可覆盖本页三个弹窗 + 全局新建选题弹窗）。

## P2（9 项，体验受损）

### P2-7 防抖推荐无乱序防护
- 位置：579-609 suggest effect。450ms 防抖但无请求序号/取消，慢的旧响应可能后到，覆盖新推荐和已选选题。
- 修法：加请求序号 ref（如 `suggestSeqRef`），发请求前自增，响应回来时比对，过期响应直接丢弃（setSuggestions 和自动选中都跳过）。

### P2-8 自动选中只在空时填
- 位置：597-599，现在每次推荐回来都 `updateMeta("topicId", data[0].id)` 覆盖。
- 修法：只在当前没有选中题时自动填。注意闭包里 `meta.topicId` 可能过期，用函数式更新判断：
  ```ts
  setMeta((current) => current.topicId ? current : { ...current, topicId: data[0].id });
  ```
  （`isManuallySet` 的既有判断保留。）

### P2-9 "暂无对应"没有已选状态
- 位置：暂无对应按钮 1931-1940；空态展示区 1977-2007。
- 修法：`isManuallySet === true && !meta.topicId && !urlLocked` 时，关联区显示一行状态："已标记：本条作品暂无对应选题"，右侧给"重新自动匹配"小按钮（点击 `setIsManuallySet(false)`，让防抖推荐重新跑）。初始未操作状态（isManuallySet=false）保持现有提示文案不变。

### P2-10 搜索弹窗闪错误空态 + 残留旧结果
- 位置：搜索 effect 611-645；弹窗 2246-2299。
- 现状：打开弹窗后 300ms 防抖才开始，`searching` 初始 false、`searchResults` 为 [] → 先闪现"未搜索到匹配的选题"；且关闭弹窗不清 searchResults，下次打开先看到上次的旧结果。
- 修法：`searchDialogOpen` 变 true 时同步 `setSearching(true)` 再进防抖；弹窗关闭时（onOpenChange）清空 `searchQuery` 和 `searchResults`。

### P2-11 搜索只搜 title + 直连库错误静默
- 位置：617-636。
- 修法：
  1. 匹配 hook 字段：`.or(\`title.ilike.%kw%,hook.ilike.%kw%\`)`。kw 先清理，去掉/转义 `%,()"` 等会破坏 PostgREST 语法的字符。
  2. `const { data } = await query` 现在丢了 error——改为 `{ data, error }`，error 时弹窗内显示"搜索失败，请重试"（新增 searchError state），catch 同样落到这个文案，× 只 console。

### P2-12 新建选题自动绑定顶掉手动选择
- 位置：`handleNewTopic` 552-576（与 P1-3 同处，一起改）。
- 修法：未锁定但已有手动选择的选题（isManuallySet && meta.topicId）时也不自动绑定，直接 return。只有表单还没选定选题时才自动关联新题。

### P2-13 成功页整卡点击取消倒计时
- 位置：1548-1550，成功卡片 div 的 `onClick={() => setCountdown(null)}`。
- 修法：删掉这个 onClick（各按钮已有自己的 setCountdown(null)，自动跳转的取消改由按钮承担）。卡片上的 select-none 等其他类名保留。

### P2-14 suggest 接口失败完全静默
- 位置：593-605，非 ok 响应直接落空，用户永远看到"输入标题或文案，系统将自动匹配选题库。"
- 修法：非 ok / catch 时设 `suggestFailed` state，关联区显示内联小字"选题推荐加载失败" + "重试"（retrySeq 重新触发，可与 P1-5 的 retry 共用模式）。× toast（防抖会刷屏）。

### P2-15 手动选择状态不随草稿保存
- 位置：DraftData 713-721、draftData useMemo 723-736、handleRestoreDraft 747-761。
- 修法：
  1. DraftData 增加 `isManuallySet: boolean`，draftData 里带上，useMemo 依赖补上。
  2. 恢复时：`setIsManuallySet(draft.isManuallySet ?? Boolean(draft.meta.topicId))`（旧版本草稿没这字段，有 topicId 就当手动选择保护起来）。URL 锁定态下跳过此项（P1-4）。
  3. `src/lib/video-submit-draft.ts` 的 `isVideoSubmitDraftEmpty` 不看这两个字段，无需改；若你改了它的类型定义，确认现有用例不受影响。

## 验收标准（逐条自查）

1. OCR 失败 → 点🔄 → 重新识别；网络失败重试不变
2. 点建议 chip → 继续改标题 → 选中不被覆盖
3. URL 带 topicId 进入 → 三按钮消失；新建选题/恢复草稿都动不了锁定的题
4. 锁定态恢复草稿 → 表单内容恢复，但关联仍是锁定的题
5. 无效 topicId → 显示"未找到该选题"而非永远加载中；A→B 切换不残留旧名
6. 弹窗打开时按 Esc → 只关弹窗，面板不翻页；输入框里 Esc 不翻页
7. 快速连续改标题 → 最终展示的是最后一次输入对应的推荐
8. 系统自动选中后继续打字 → 选中不被新推荐顶掉
9. 点"暂无对应" → 出现"已标记"状态行，可一键回自动模式
10. 打开搜索弹窗 → 先 loading 不闪"未搜索到"；重开不残留上次结果
11. 搜索关键词能命中 hook 内容；模拟接口/权限失败 → 弹窗内有错误文案
12. 手动选好题后新建选题 → 已选题不被顶
13. 提交成功页点卡片 → 倒计时不被取消（按钮取消仍有效）
14. 断网改标题 → 关联区出现"推荐加载失败"可重试
15. 选好题 → 存草稿刷新 → 恢复草稿后再打字 → 选题不被自动推荐覆盖
16-18. 快捷键提示可见；方向键可切正常/异常；对调时有 toast 说明
