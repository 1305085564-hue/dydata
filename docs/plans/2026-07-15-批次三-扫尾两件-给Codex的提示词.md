# 批次三 · 扫尾两件 · 给 Codex 的提示词

> 用法：阿禅把本文整段发给 Codex，让它**先做方案 → 跟阿禅沟通确认 → 再执行**。
> 背景：网站产品重构分三批。批次一(工作台传+看)、批次二(命名+入口治理)已另有规格。批次三 = 选题库新建(**另议，主要人用，单独设计，本提示词不含**) + 本文这两件扫尾。

## 通则（两件都适用，务必遵守）
1. **先查后动**：先调研依赖、产出方案、跟阿禅逐项确认，**禁止盲删/一刀切**。
2. **不碰选题库**：选题库是另一条产品线（主要人用、多运营协作、重新搭思路），本批不做、别顺手动。
3. **范围纪律**：只做下面两件，不越界改别的模块。
4. 遵守项目架构治理：动手前读全局文档，动手后更新全站模块地图/相关文档。
5. 交付时给**一键回滚指令** + 标明**哪些立即生效 / 哪些需上线部署**。

## 任务 A：视频审核 `/video-review` 归档

**现状**：短期过渡项目，批次二已降级（移出主导航、标"过渡"）。`manage` 路由已 redirect 到 `/admin/content`。

**锚点**：
- 页面：`src/app/(app)/video-review/*`（含 submit/exemption/archive/manage 子路由、`components/case-card.tsx` 卡片墙）
- loader：`src/lib/loaders/video-review-page.ts`
- API：`/api/work-submissions`、`/api/publish-drafts/*`、`/api/exemptions/*`、`/api/work-screenshots/upload`
- 表：`work_submissions`、`publish_drafts`、`exemptions`

**要做**：
1. 查清还有谁依赖 video-review（路由、API、表、组件是否被别处引用），列出来。
2. **考勤精华处理**：对比 video-review 的考勤逻辑（`daily_target` / `submitted_count` / `gap` / `exemption_status` / `alert_level`）与**发布履约 `/admin/fulfillment` 现有能力**（它已是考勤日历工作台，有月度矩阵/履约率/飞书催交/申诉）。**缺的才补进发布履约，重复的直接弃**——不要为搬而搬。
3. **卡片墙不再保留**：选题库已确定不用卡片墙那套（阿禅决定重新搭思路），所以 `case-card` 随归档一起处理，不迁移。
4. 出**归档方案**（三选一并说明理由）：软隐藏 / 保留只读入口 / 彻底移除；说清存量数据（`work_submissions` 等）怎么办，**不丢数据**。
5. 跟阿禅确认方案后再执行。

## 任务 B：文案助手减法（`/content-tools/rewrite`）

**现状（已膨胀失控）**：横跨三入口 + 拖两套 API——
- 员工端 `/content-tools/rewrite`（V3 工作站，类 IDE 写作站）
- 员工端 `/content-tools`（选题建议 / 模板库 / 发布时间推荐）
- 管理端 `/admin/ai-config?tab=rewrite`（改写路由配置）
- 两套 API：`/api/rewrite/*` 与 `/api/content-tools/rewrite/*`（迭代残留）

**锚点**：`src/components/content-tools/rewrite-v3/*`（10 文件）、`src/lib/rewrite/`（`shared.ts` 84KB、`generation.ts` 30KB，4 种 runType：full_rewrite/paragraph_patch/variant_generate/chat_reply）

**阿禅要保回的核心**：传文案 → **固定提示词 + 模型组合**（Claude 管逻辑与文笔、Gemini 管情绪与用词）→ 改写润色。

**减法原则（阿禅原话）**：先解析后减法；砍**不稳定 + 使用价值低**的一小部分；**实在不好删宁可不删**；**不一刀切到底**。

**要做**：
1. **先解析**：把现有全部功能/子模块/入口/两套 API 列成清单，每项标注：功能 → 稳定性 → 使用价值 → 是否属核心主线。（膨胀点参考：技能 CRUD/SkillCabin、版本时光机 TimelineDiff、导 Word、修订模式、定稿阅览室、参数轮数剪枝、撤销重做、废弃的那套 API）
2. **出减法方案**：建议砍哪些、留哪些、各自理由；明确"传文案 + 固定提示词 + 模型组合"这条主线如何保住并跑通。
3. **一件事要先问、别擅自处理**：`/content-tools` 那组（选题/模板/发布时间）是否归入本次减法、还是另算——先问阿禅。
4. 跟阿禅**逐项确认砍/留**后再执行。
