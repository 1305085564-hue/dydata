# 批次二 · 命名与入口治理（主方案）

> 三批总览：批次一(工作台传+看，员工端大改，已出规格) → **批次二(本文，命名+入口治理，小改高性价比)** → 批次三(选题库新建 + 视频审核卡片墙/考勤精华搬走 + 文案助手减法)。
>
> 分工：**Codex = 工程 + 验收 + 统筹**（本批主力，偏路由/标签/信息架构，是它的地盘）；**Antigravity = 前端执行辅助**（只在需要视觉/交互重排的局部出手，由 Codex 调度）；前期规划 = 本文。

## 0. 定位（一句话）
阿禅诊断：管理端没人用不是产品/代码问题，是**命名劝退 + 入口太深/太散**。本批**只动标签、导航、入口可见性、信息架构归位，绝不动产品逻辑与业务代码**。这是"改名字、理入口、收散页"的一批，风险低、见效快。

## 1. 前置依赖（务必先确认）
本批**必须在批次一完成并验收后再做**，因为：
- 批次一已把"上传数据"和"查看成长/大盘"物理分离：`/dashboard` 专注上传与补交，`/growth` 承接成长分析、趋势与排行榜。
- 批次一已把异常数据提交并入工作台表单，但 `/violations` 仍承载话术库、案例详情、避坑查看等独立内容，不能直接当废页重定向。

若批次一未完成，本批的导航移除/重定向会指向空页面，禁止提前动。

## 2. 需阿禅拍板的 3 个点（已给推荐默认，可直接采纳或改）
- **D1 批改台改名**：推荐 → **「视频复盘」**（备选：视频诊断 / 复盘台）。这是全批最该改的名字，"批改台"让人以为要批文档。
- **D2 导粉中心处理**：已定 → **保留 `/violations` 独立页，员工主导航移除**。本批不做重定向、不做归并；入口放在工作台和成长大盘的次级入口，避免误删话术库和案例详情。
- **D3 视频审核降级程度**：已定 → **短期过渡入口**。从员工端主导航移除或弱化，保留可直接访问，并标注"过渡/短期使用"；预计下周后不再作为主流程迭代，真归档 + 搬卡片墙/考勤精华留到批次三。
- **D4 个人成长处理**：已定 → **保留 `/growth` 独立页**。因为趋势、排行榜和成长分析已从 `/dashboard` 迁到 `/growth`，本批只允许把主导航标签从「个人成长」改得更准确，例如「成长大盘」或「内容成长」，禁止把 `/growth` 重定向回工作台。

> 阿禅不特别指定就按推荐默认执行。

## 3. 本批范围
做：

不做（留批次三或以后，禁止顺手改）：
