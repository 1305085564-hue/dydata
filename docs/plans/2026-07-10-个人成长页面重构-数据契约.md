# 个人成长页面重构：数据契约

> 面向下一棒 Antigravity。正式前端只消费本文契约，不再从原始日报自行计算诊断、六维、可信度或对标。

## 一、加载入口

- 函数：`loadGrowthPageContract`
- 代码：`src/lib/loaders/growth-page.ts`
- 参数：

```ts
{
  supabase: SupabaseClient;
  userId: string;
  userEmail: string | null | undefined;
  now?: Date;
}
```

- 返回：`Promise<GrowthPageContract>`
- 类型与唯一规则来源：`src/lib/growth-page.ts`
  - 契约生成器：`buildGrowthDataContract`
  - 六维唯一配置：`GROWTH_DIMENSION_RULES`

正式链路入口：`/growth`。`growth-data-container.tsx` 只负责加载契约并传给 `growth-client.tsx`；前端不再自行计算诊断、六维、可信度或对标。

## 二、完整返回结构

```ts
type GrowthPageContract = {
  identity: {
    profileName: string;
    accountCount: number;
    reportCount: number;
  };

  credibility: {
    level: "low" | "mid" | "high";
    label: string;
    sampleCount: number;
  };

  verdict: null | {
    weakestDimension:
      | "开头留人"
      | "中段跳出"
      | "整体完播"
      | "增长转化"
      | "互动吸引"
      | "话题爆点";
    diagnosis: string;
    prescription: string;
    source: "rule" | "ai";
    metric: {
      self: number;
      teamAvg: number;
      unit: string;
    };
  };

  radar: Array<{
    dimension: "开头留人" | "中段跳出" | "整体完播" | "增长转化" | "互动吸引" | "话题爆点";
    self: number;
    teamAvg: number;
    rating: "strong" | "mid" | "weak";
  }>;

  metricsOverview: Array<{
    label: string;
    value: number;
    trend: number | null;
    unit: string;
  }>;

  benchmark: {
    state: "ok" | "fallback_team_avg" | "none";
    peer?: {
      name: string;
      dimensionValue: number;
      scriptSnippet: string;
    };
    teamAvg?: number;
  };

  scriptBreakdown:
    | {
        state: "ok";
        segments: Array<{
          type: "hook" | "background" | "core_point" | "action_cta" | "closing";
          order: number;
          content: string;
        }>;
      }
    | { state: "none" };

  trend: Array<{
    date: string;
    playCount: number;
    followerGain: number;
    completionRate5s: number;
    completionRate: number;
  }>;

  emptyState: {
    isEmpty: boolean;
    reason?: string;
  };
};
```

## 三、字段含义与空值规则

### 1. `identity`

| 字段 | 含义 | 空时 |
| --- | --- | --- |
| `profileName` | 当前登录用户姓名；无姓名时使用登录邮箱 | `""` |
| `accountCount` | 当前用户名下真实账号数 | `0` |
| `reportCount` | 本次 30 天体检窗口内的真实日报条数 | `0` |

### 2. `credibility`

只表示当前结论有多少真实样本支撑，不阻挡用户查看数据。

| 真实样本数 | `level` | `label` |
| ---: | --- | --- |
| 0–2 | `low` | `样本不足，仅供参考` |
| 3–9 | `mid` | `样本累积中` |
| ≥10 | `high` | `样本充足` |

原任务文字对“10 条”同时写了 `3-10` 和 `>=10`，两者重叠。代码按更明确的 `>=10` 执行，因此 10 条进入 `high`。

### 3. `verdict`

- 有真实日报：始终返回规则版体检结论。
- 没有真实日报：返回 `null`。
- `diagnosis` 已填入当前用户和团队真实数字，前端直接展示，不再拼句子。
- `prescription` 是后端按最弱能力维度给出的单一可执行动作。
- V1 的 `source` 恒为 `rule`；V2 才允许返回 `ai`。

### 4. `radar`

- 固定按六维唯一顺序返回 6 项。
- `self`、`teamAvg` 都是原始指标值，不是虚构分数。
- 没有真实日报时返回 `[]`。
- 团队均值只统计其他同事，不把当前用户混入团队基准。

### 5. `metricsOverview`

- 来源是当前真实日报的发布数、总播放、总涨粉、平均点赞率、平均完播率。
- `trend` 是与上一观察窗口的真实变化；没有可比较值时为 `null`。
- 没有真实日报时返回 `[]`。

### 6. `benchmark`

| `state` | 返回内容 | 前端做法 |
| --- | --- | --- |
| `ok` | `peer` 存在 | 展示实名同事、该维度数值和文案片段 |
| `fallback_team_avg` | `teamAvg` 存在，`peer` 不存在 | 展示团队均值兜底，不展示虚构同事 |
| `none` | 不带 `peer`、不带 `teamAvg` | 隐藏实名对标，提示继续积累团队样本即可 |

对标按同事聚合真实日报。除“中段跳出”外取该维度最高者；“中段跳出”是越低越好，因此取真实流失率最低者。文案片段优先取结构化脚本片段，没有时才取该同事日报中的真实文案；两者都没有时返回空字符串。

### 7. `scriptBreakdown`

- 找到与最新真实日报对应的结构化脚本分段：`state = "ok"`，按顺序返回 `segments`。
- 脚本表没数据、关联不上、字段或表结构缺失：统一返回 `{ state: "none" }`。
- 前端遇到 `none` 时整块隐藏，不展示“暂无脚本”空壳。

### 8. `trend`

- 只返回当前用户真实日报历史，按日期从旧到新排序。
- 每项只包含趋势绘图需要的日期、播放、涨粉、5 秒完播、整体完播。
- 没有真实日报时返回 `[]`。

### 9. `emptyState`

- 0 条真实日报：`{ isEmpty: true, reason: "还没有真实日报数据" }`。
- 有真实日报：`{ isEmpty: false }`。

## 四、六维唯一口径

唯一代码位置：`src/lib/growth-page.ts` 的 `GROWTH_DIMENSION_RULES` 和 `getDimensionValues`。阈值、维度方向、诊断尾句和规则改法不得在前端复制。

| 维度 | 指标 | 计算公式 | 越高越好 |
| --- | --- | --- | --- |
| 开头留人 | 5 秒完播率 | `completion_rate_5s` 的样本平均值 | 是 |
| 中段跳出 | 中段流失率 | `max(5秒完播率 - 整体完播率, 0)` | 否，越低越好 |
| 整体完播 | 完播率 | `completion_rate` 的样本平均值 | 是 |
| 增长转化 | 涨粉率 | 每条 `follower_gain / play_count × 100` 后取平均 | 是 |
| 互动吸引 | 综合互动率 | 每条 `(评论×0.35 + 分享×0.25 + 点赞×0.25 + 收藏×0.15) / 播放 × 100` 后取平均 | 是 |
| 话题爆点 | 平均播放量 | `play_count` 的样本平均值 | 是 |

### 对比比例与评级

- 越高越好的维度：`ratio = self / teamAvg`。
- 越低越好的“中段跳出”：`ratio = teamAvg / self`。
- `ratio >= 1.1` → `strong`。
- `ratio <= 0.9` → `weak`。
- `0.9 < ratio < 1.1` → `mid`。
- 最弱项取六维中 `ratio` 最低的一项。
- 团队无样本时不伪造团队数字；诊断会明确写“团队暂无可比样本”。

## 五、七种状态

| 状态 | 必须返回 |
| --- | --- |
| 0 数据 | `emptyState.isEmpty = true`；`verdict = null`；`radar/metricsOverview/trend = []`；`benchmark.state = "none"`；`scriptBreakdown.state = "none"` |
| 1–2 条 | 真实数据照返；`credibility.level = "low"`；结论照常生成，不补造报告 |
| 3–9 条 | `credibility.level = "mid"`；其余按真实数据正常返回 |
| ≥10 条 | `credibility.level = "high"`；其余按真实数据正常返回 |
| 无脚本 | `scriptBreakdown = { state: "none" }`，其余块不受影响 |
| 找不到实名对标 | 有团队均值则 `fallback_team_avg`；连团队样本也没有则 `none` |
| AI 状态 | V1 不调用 AI，`verdict.source` 恒为 `rule`；仅保留 `"ai"` 枚举给 V2 |

## 六、隐私边界

新契约不返回同事的用户 ID、账号 ID、邮箱、完整日报、账号标签或其它指标。实名对标只允许以下三个同事字段：

1. `name`：真实姓名；
2. `dimensionValue`：当前最弱维度上的真实对标值；
3. `scriptSnippet`：该同事用于互学的真实文案片段。

下一棒前端不得继续依赖旧页面的 `teamReports` 自算对标。正式重写完成后，应只消费 `GrowthPageContract`，以免把服务端内部查询结果扩大暴露到浏览器。
