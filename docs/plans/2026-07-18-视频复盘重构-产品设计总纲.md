# 视频复盘重构 · 产品设计总纲

> 2026-07-18 定稿。这是本次重构的地基与续工锚点。逻辑分工见「Codex 后端规格」，美学分工见「Antigravity 前端规格」，双方并行的对接契约见「接口约定」。

## 0. 一句话定义

视频复盘 = 拿一条视频（或某人今天的表现），**用多维数据对比发现问题 → 顺着不对的数字挖到具体那句台词 → 最后才反馈**。重心在对比与归因，下发是尾巴。

## 1. 权重（拿来挡一切偏离的第一尺子）

| 权重 | 目标 | 说明 |
|---|---|---|
| **80%** | **归因**：这条数据好/不好，问题在哪、体现在数据的哪里、文案的哪句 | 找不到、或体现不清晰 = 这页失败 |
| 辅助 | 顶栏异常标识 | 是归因的提效版，帮更快找到"该盘谁"；有最好，没有也不致命 |
| **20%** | 怎么让他做对（反馈下发 + 可复测） | 最后一步，不是重心 |

**验收只认一条：管理者的痛点解没解，不是功能齐不齐。** 任何设计答不上"它解哪个痛"，砍。

## 2. AI 边界（铁律，谁都不许越）

- **AI 只做三件**：算（标出哪项异常、偏离多少）、摆（把证据/对应段落/参照对比摆到面前）、提示（"这里可能有问题，建议看看"）。
- **AI 绝不做**：下"这句就是不好、这条就是失败"的结论；替人填结论；自动决策。
- **判断权 100% 在人手里。** 归因结论带是**死规则计算**的产物，不是 AI 生成；AI 分析只是可选的辅助按钮。

## 3. 成功判据

| 成功（赢） | 失败（输） |
|---|---|
| 进页面能自己挑人/挑视频，顶栏异常提醒瞄一眼就懂、不抢视觉 | 只能等系统报警才有得复盘，或异常条糊住主视觉 |
| 默认比自己近几条，一键切"比团队/比高手/比某人"，5 秒看出哪项数字不对 | 只能比自己（现状），想比别人比团队没得比 |
| 每个标红项写清"这数字→指向内容哪段"，点它跳到对应台词 | 六个数字平铺，不知哪个是问题、更不知对应哪句话 |
| 处理完自动进下一条 | 每次退回列表肉眼找下一个 |
| 复盘新视频能一键调出上次给他的卡片，前后对比 | 不记得上次提了啥，没法判断他改没改 |

## 4. 归因映射表（产品的心脏）

10 个指标库里 24h 快照全存。按"能落到内容哪、多细"分三层：

**A 类 · 留存质量（能落到时间轴上的段/句）**

| 指标（定位词） | 差了→内容哪出问题 | 落到哪 |
|---|---|---|
| 2s 跳出（开篇钩子失败率） | 封面/标题/前两句没抓住人 | 开头（三四十字）**可到句/词** |
| 5s 完播（前半段留存力） | 开头钩子弱，没给"继续看的理由" | 开头前 5 秒那一两句 **可到句** |
| 均播（内容吸引力） | 多数人看到某处就划走=那段崩了 | 均播/总长反推位置→**那段几句** |
| 完播率（整体合格线） | 中段拖沓 / 结尾没收住 | 中段冗余段 + 结尾 **到段** |

**B 类 · 互动转化（落到"内容属性/某个点"，不硬凑单句）**

| 指标（定位词） | 差了→内容哪出问题 | 落到哪 |
|---|---|---|
| 点赞（内容认可度） | 观点平淡，没戳中情绪 | 核心观点句 / 情绪点 |
| 评论（互动情绪值） | 没争议/共鸣/记忆点 | 讨论钩子（常在结尾） |
| 分享（传播裂变力） | 没社交货币，不值得转 | 整片价值属性（多半到不了单句） |
| 收藏（实用干货度） | 没干货、看完即弃 | 可复用的干货点 |
| 涨粉（账号留存价值） | 人设/价值不清晰 | 人设呈现 + 结尾关注引导 |

**C 类 · 流量结果（前提，不直接归因，往下拆）**

| 指标（定位词） | 怎么归因 |
|---|---|
| 播放量（基础流量池） | 结果不是原因。播放低→先看封面标题，再往下拆 A 类（2s/5s 没撑住→平台停推流） |

**颗粒度规则**：能细则细（A 类顺曲线拐点到句），细不了就诚实标"只能到这段/这个属性"，绝不假装精确。B 类落属性，不硬凑句子。

## 5. 页面结构（两层）

### 第一层 · 复盘台首页（选谁复盘）
- 顶栏：一条细异常提醒（"今日异常 N 条：X 删稿·Y 腰斩 [直接去盘]"），瞄一眼、不抢视觉。
- 主视觉：全部视频列表，可"按最差排 / 按人 / 按时间"，每行露出该视频的异常指标（红/绿标）。
- 决策：主动挑当主角，异常雷达降为顶栏配角。

### 第二层 · 归因舱（复盘这一条，重构最关键的一屏）
- **第一眼 = 归因结论带**（死规则算，一次答完"哪项不对+指向哪段"）。绿色项也列（"这不用改"）。
- **左屏答"数据哪不对"**：归因结论带 + 数据对比大盘（参照系一键切）+ 留存曲线（标拐点）。
- **右屏答"文案哪不对+怎么说"**：分段脚本（问题段高亮）+ 反馈下发。
- **点红项 → 右屏脚本对应段高亮跳过去**（拐点↔台词联动）。
- 砍掉：推流峰值/峰值后走势/流量承接那套人工下拉（错抓手）。
- AI 降为"要不要听听 AI 怎么看"的辅助按钮。

## 6. 决策与流转
- 预排序：列表/异常队列按"该先盘谁"排（异常越重越靠前）。
- 默认不空白：进舱归因结论已算好摆着。
- 一键下发：保留两步防误触。
- 自动流转：下发完自动跳下一条。

## 7. 兜底
- 没满 24h/缺快照：归因带不空白，明说"先看已有的 X，归因待数据齐"。
- 某项指标缺：那项标"缺数据"，其余照常归因。
- AI 辅助挂了：不影响，人自己顺数据归因。
- 拐点算不出：退到"只能定位到这段"，标明原因。

## 8. 现状与改造边界
- 复用不动：闭环状态机（`content-review-readiness.ts` 的诚实兜底状态现成）、反馈卡 PATCH 自动存草稿、经验沉淀、异常雷达打分、下发锁定。
- 改标签：现对比接口的"团队"其实是**本账号近 3 条**（口径错），标签改对。
- 新建重点：横向对比（比团队均值/比高手/比某人）——一行没做；归因结论带的死规则计算；拐点↔台词联动；自动流转。
