## 3. 设计原则（本项目锚定 3 条）

| # | 原则 | 含义 | 冲突时怎么办 |
|---|---|---|---|
| 1 | **结论先行** | 打开先给"你最弱哪项、下一条怎么改"，证据（雷达/指标/对标）和细节往后放、可折叠 | 任何设计让员工"先看一堆数据再自己推结论" → 退回 |
| 2 | **真实透明** | 只展示真实数据；可信度永远标清楚；不确定就明说不确定，绝不编 | 任何"为了页面好看而造数/补数" → 退回 |
| 3 | **AI 是升级不是地基** | 规则底座永远在，AI 有就更深、没有照样能用 | 任何"AI 挂了页面就塌/空壳" → 退回 |

## 4. 业务主线与角色

**主线六问**：
1. 定位：员工的**内容体检台**——看自己内容表现，拿到"下一条怎么改"。
2. 核心闭环：看指标 → 定位最弱项 → 拿到"该学谁 + 怎么改" → 去拍下一条 → 回来看有没有进步（复测）。
3. 用户最高频动作：看"我最弱哪项、下一条怎么改"。
4. 管理者最高价值动作：**不在本页**（管理动作在 /admin，本页是员工私人视角）。
5. 最重要结果指标：员工打开后能明确说出"我下一条要改的是什么"。
6. 最该降级到后台：全员排名、跨人对比大盘 → 属于 admin/analytics，不进本页。

**角色（只一类）**：

| 角色 | 核心动作 | 可操作 | 禁止 |
|---|---|---|---|
| 用户（员工看自己） | 看诊断、拿改法、看自己趋势 | 查看、切换对比对象、展开细节 | 看他人私有数据、任何审核/管理操作 |

> 结论：本页无"管理者/运营者"角色，无审核/处罚/权限操作。第 6 章的"待办队列"机制大部分不适配（本页不是逐条处理任务），只借其"降低理解/判断/执行成本"的思想。

## 5. 核心任务路径

员工唯一核心任务：**搞清楚下一条视频该怎么改。**

```
入口(顶部导航"成长")
 → 首屏体检结论：一眼看到"最弱项 + 一句话诊断 + 下一条怎么改"
 → 想看证据？展开六维能力画像（雷达）看差距来自哪
 → 想更深？看 AI 诊断（有则深度版，无则规则版）
 → 想学人？看"该学谁"对标 + 对方的文案片段
 → 想抠文案？看自己最近一条的文案拆解
 → 记住改法 → 去拍下一条
 → 几天后回来 → 看趋势有没有改善（复测闭环）
```

路径规则自检：
- 高频结论（最弱项+改法）在首屏，不藏二级 ✅
- 首屏 5 秒能说出"我该改什么" ✅
- 不需要跳出页面就能拿到完整改法 ✅
- 有明确"下一步"（去拍 → 回来复测）✅

## 6. 决策与流转（本页按"降三成本"思想，非待办队列）

本页不是逐条处理任务的工作台，不套预排序/一键确认/自动流转。但"降三成本"照样用：

| 成本 | 本页怎么降 |
|---|---|
| 理解成本（看懂在干嘛）| 首屏唯一重点 = 最弱项那张大卡，其余降级 |
| 判断成本（知道做什么）| 直接给"该学谁 + 具体改法"，不让员工自己从雷达图推 |
| 执行成本（落地几下）| 改法必须是"一句可照做的动作"（例：开头3秒先抛冲突，别讲背景），不是空泛建议 |

**闭环最后一公里（复测）**：员工按改法拍了新的一条、提交日报后，页面趋势区要能体现"改法生效了没"——把上次诊断的那一项，在新数据里高亮对比。

## 7. 页面主次与首屏（进来第一眼看什么）

**首屏唯一焦点 = 体检结论卡**（一张大卡，占首屏主区）：
- 大标题：`你现在最该补的是「开头留人」`
- 一句话诊断：`你的 5 秒完播 28%，团队均值 41%，前 3 秒没留住人`
- 一句话改法：`下一条开头 3 秒先抛冲突或结果，别先讲背景`
- 可信度标签：`样本 6 条 · 结论仅供参考` / `样本充足`
- 若有 AI：这张卡内容由 AI 生成，标 `AI 深度诊断`；无 AI：规则版，标 `规则分析`

**信息分层**：

| 层级 | 内容 | 位置 |
|---|---|---|
| P0 | 最弱项 + 诊断 + 改法（体检结论卡）| 首屏主区，最大 |
| P1 | 六维能力画像雷达 + 核心指标概览（证据）| 主区次级 / 结论卡下方 |
| P2 | 该学谁对标详情、文案拆解、成长趋势 | 折叠 / 页面下半 / 展开看 |
| P3 | 全员排名、跨人大盘 | 不在本页（去 admin/analytics）|

**工作台自检**：员工进来第一眼看到的是"今天/最近我该改什么"，不是一堆图表菜单 → 达标。

## 8. 兜底设计（三件事：认知安全感 + 反向场景 + 状态反馈）

### 8a. 认知与安全感
- 每个结论旁边永远有可信度标签，员工知道"这话能信几分"。
- 改法给"为什么这么改"的一句依据（对标了谁/差距多少），不是命令式空建议。
- 术语旁给白话（如"5秒完播率 = 100 个人点进来，5 秒后还剩几个没划走"）。

### 8b. 反向场景（本页必须覆盖这几种）

| 反向场景 | 应对设计 |
|---|---|
| **完全没数据**（新员工0条）| 不空白、不造假。显示引导页："还没有你的数据，去提交今天的日报，就能解锁你的成长体检" + 直达日报入口 |
| **数据很少**（1-2条）| 照常展示真实那几条 + 团队真实数据兜底做参照，全程标"样本不足，仅供参考"，不挡人看 |
| **AI 全挂/未触发** | 无缝显示规则版结论卡，标"规则分析"，页面完整可用，不报错不空壳 |
| **没录脚本** | 文案拆解模块整块优雅隐藏（不显示"暂无文案"空壳），其余不受影响 |
| **团队数据不足**（对标找不到人）| "该学谁"降级为"参考团队均值以上样本" / 或"暂无同题材可对标的人，先复盘自己历史最好的3条" |

规则：每个核心区块都要能独立降级，一块没数据不拖垮整页。

### 8c. 状态反馈（每个状态告诉员工"发生了什么 + 能干什么"）

| 状态 | 告诉员工 | 能做 |
|---|---|---|
| 空（0数据）| 为什么空、去哪开始 | 去提交日报 |
| 加载中 | 骨架屏，正在算你的体检 | 等待 |
| 当天刚提交、AI 跑着 | "正在生成今日 AI 深度诊断"，**先显示规则版垫着** | 先看规则版，AI 好了自动替换 |
| AI 失败 | 不打扰，静默用规则版（可选：小字"AI 暂不可用，当前为规则分析"）| 照常看规则版 |
| 样本不足 | "样本 X 条，结论仅供参考" | 照常看，继续攒数据 |
| 正常（样本足）| 完整体检 + 可信度"充足" | 看诊断、拿改法 |
| 复测（提交了新数据）| 高亮"上次让你改的那项，这次变了多少" | 看进步对比 |

规则：不只有成功态；AI 失败绝不卡死；状态文案解释"发生了什么"。

## 9. AI 与人工边界（本页是 AI 产品，本章适用）

**AI 在本页的职责**：把规则版的"最弱项 + 改法"升级成更具体、结合脚本和对标的深度诊断。**只出建议，不出评价/处罚/绩效结论。**

| 项 | 规则底座（无 AI 也有）| AI 升级层（有 AI 更好）|
|---|---|---|
| 最弱项识别 | 真实指标 vs 团队均值算出来 | 同上（AI 不改这个，用同一套数）|
| 诊断根因 | 模板句："X 低于均值 Y，先补这项" | 结合你的脚本+对标，说清"为什么弱、弱在哪一句" |
| 改法建议 | 按维度预设的一句改法 | 针对你这条内容的具体改法 |
| 该学谁 | 团队里该项最高的人 | 同上 + AI 提炼对方"可复制的动作" |

**边界铁律**：
- AI 只解读，不评级、不打分、不影响任何考核。
- 员工能看到 AI 结论的依据（用了哪些数据、对标了谁）。
- AI 挂了只降体验（回落规则版），业务不停。
- **AI 输入**：员工真实指标（近 7/30 天）、最近一条脚本原文、最弱项、对标对象数据。
- **AI 输出**：诊断根因（弱在哪）、具体改法（怎么改）、对标提炼（学谁的什么动作）。
- **触发**：员工当天提交了数据 → 当天首开自动跑一次并缓存；当天没提交 → 读上次缓存，不跑。一天最多一次。

## 10. 版本切割

三问判定，切成 V1/V1简化/V2/不做：

| 类型 | 内容 | 三问结论 |
|---|---|---|
| **V1 必做** | 真实指标概览（砍假数据+可信度标签）| 闭环基础，缺了整页失真 |
| **V1 必做** | 统一六维能力画像雷达（合并两套打架维度）| 体检主图，缺了没证据 |
| **V1 必做** | 体检结论卡：最弱项+规则版诊断+改法 | 首屏焦点，页面核心，规则版即可跑通 |
| **V1 必做** | 空/少数据/降级全套状态（8b/8c）| 缺了就会重蹈"造假/空壳"覆辙 |
| **V1 必做** | 清理死代码组件 | 重构前提 |
| **V1 简化做** | 同事对标（实名）| 先用规则算"该学谁"，AI 提炼动作放 V2 |
| **V1 简化做** | 文案拆解 | 有脚本就按结构切块展示；AI 深度拆解放 V2 |
| **V2 再做** | AI 深度诊断（替换结论卡内容 + 对标动作提炼 + 文案深拆）| 闭环靠规则版已通，AI 是升级 |
| **V2 再做** | 复测高亮对比（上次改法这次变好没）| 体验增强，非最小闭环 |
| **不做** | 全员排名/跨人大盘 | 稀释首屏焦点，且属 admin/analytics |
| **不做** | OCR 流量/留存曲线 | 依赖 content_asset 补列，被 TODO 挡着 |

**最小闭环（V1 交付即可用）**：进页 → 看真实体检结论（规则版）→ 拿改法 → 去拍 → 回来看真实指标变化。全程不依赖 AI。

## 11. 交付验收（三方开工清单）

### 后端（Codex）
- **数据实体**：daily_reports（真实指标）、accounts、profiles、script_document/segment（脚本）、ai_insight_result（AI 缓存）。
- **必改**：删 `generateVirtualReports` 及全部虚拟数据逻辑；统一六维口径（合并 growth-page.ts 与 diagnosis-card.tsx 两套维度为一套，口径写死一处）。
- **可信度分级**：按真实样本条数返回 `<3 / 3-10 / >=10` 三档标签，不设硬门槛。
- **降级**：脚本表缺数据 → 返回空且标记"无脚本"，前端隐藏该块；对标找不到人 → 返回团队均值兜底。
- **AI 触发/缓存**：当天有提交 → 首次请求跑 AI 并写缓存；当天无提交 → 只读缓存。收口到 `/api/growth/ai-insight`。
- **权限**：对标块走服务端取数，只返回展示所需字段（姓名/该维度指标/文案片段），不泄露同事其他隐私字段。

### 前端（Antigravity）
- **首屏**：体检结论大卡为唯一焦点（最弱项+诊断+改法+可信度标签）。
- **信息分层**：P0 结论卡 → P1 雷达+指标 → P2 对标/文案拆解/趋势（折叠）。
- **状态分支**：空（引导去日报）/ 加载（骨架）/ AI跑着（先垫规则版）/ AI失败（静默规则版）/ 样本不足（标仅供参考）/ 正常 / 复测高亮，全部要做。
- **降级**：每块能独立空/降级，不互相拖垮。
- **清理**：删 benchmark-card / sample-library / capability-grid / advice-panel / ai-advice 等未引用组件。
- **响应式**：电脑为主兼顾移动，做真实浏览器验收。
- **读**：动手前读 `docs/美学规范.md`。

### AI
- **输入**：员工近 7/30 天真实指标、最近一条脚本原文、最弱项、对标对象数据。
- **输出**：诊断根因、具体改法、对标动作提炼（JSON 结构化）。
- **人工确认点**：无（本页 AI 只解读不决策，无审核动作）。
- **降级**：AI 失败/超时 → 回落规则版，页面不停。

### 验收标准（对应规划书成功判据）
| 成功 | 失败 |
|---|---|
| 5 秒看懂"最弱项+下一条怎么改" | 还是一堆指标堆砌 |
| 全页只有真实数据+可信度标注 | 还有任何凭空造的数字 |
| AI 挂了照常可用、有 AI 明显更深 | AI 挂了白屏/空壳 |
| 雷达和诊断同一套维度 | 两套维度继续打架 |
