## 必修 1：修 loader 的 Supabase 类型错误

**文件**：`src/lib/loaders/growth-page.ts`，约 23 处报错。

**报错形态**：大量
- `TS2739: Type 'PostgrestFilterBuilder<...>' is missing the following properties from type 'Promise<unknown>': catch, finally, [Symbol.toStringTag]`
- `TS18046: 'xxxResult' is of type 'unknown'`

**根因方向**（你自己核实）：这类错通常是 Supabase 查询构造器（`supabase.from(...).select(...)`）没有被正确当作 Promise 处理，或 `Promise.all([...])` 的元素类型没接上，导致后续 `.data` 取值全变成 `unknown`。查询本身是对的（运行时测试都过了），是**类型标注/写法**让 TS 认不出来。

**怎么修**：
- 按报错逐行定位，让每个查询结果拿到正确的类型（该 await 的 await、该给泛型的给泛型、该断言的按真实返回结构断言）。
- 不要改查询逻辑和字段（那些线上验证过、测试也过了），只修类型让 `tsc` 认得出。
- 修完 `npx tsc --noEmit` 里 `src/lib/loaders/growth-page.ts` 和 `src/lib/growth-page.ts` 零报错。
