# 批次三 · 选题库 · 产品设计规格

> 全站产品重构第三批的心脏，也是全站唯一 net-new（其余都是改/删/搬）。
> 主要人用、多运营协作、重新搭思路（不参考视频审核那套卡片墙+分类浏览）。
> 服务对象：短视频运营。
> 本文是产品设计规格（先讲清长什么样、怎么用），Codex 后端 / Antigravity 前端的分工规格待阿禅确认本设计后再拆。

## 0. 一句话定位

选题库的核心是**存 + 选**：帮运营**积累选题、挑对选题**。不为复盘服务——复盘在数据上传那边看。

运营每天打开选题库，要能快速完成两件事：**丢进去一个新想法**、**从池里挑出今天值得做的选题**。做不到这两件事=又变摆设。

## 1. 三条铁律（地基，不可违背）

1. **录入要极轻**——想到一个选题，3 秒内能丢进去，只填"母题 + 一句话选题"，其余字段全部后置、可空。字段一多，运营就"先记脑子里晚点录"，然后永远没录。
2. **系统推荐必须带"为什么推它"**——从素材库浮上来的候选，每条必须说清依据（基于哪条爆款、什么信号）。不带理由的推荐=垃圾信息，运营直接无视。
3. **绝不让运营额外维护**——选题库要嵌进他现有的干活流程（挑选题→写脚本→上传数据），让他顺手就把数据留下，而不是干完活还要回来"维护选题库"。自动归类、自动关数据，人只做确认。

## 2. 数据结构：母题 / 子题 / 作品 三层

- **母题**＝长期存在的大方向（例：龙头战法、财报避坑、宏观看盘）。数量少、变化慢，是分类的骨架。
- **子题**＝一个具体、能直接拿去拍的选题（例：龙头一进二战法怎么打）。这是运营每天真正在挑、在做的单位。
- **作品**＝某个运营基于某个子题、实际拍出来发布的那条视频（带文案、发布号、发布后的数据）。

关系：一个母题下挂多个子题；一个子题可以被不同人/不同号做成多条作品。这样才能沉淀出"**哪类选题、在哪个号最灵**"。

### 母题骨架（预置 8 大类，来自团队现有内容体系）

系统预置 8 个母题（高频放上面、低频放下面），运营不用从零分类：

1. 暴力战法类
2. 热点/新闻解读类
3. 情绪周期类
4. 案例拆解/复盘类
5. 避坑防雷类
6. 降维认知类
7. 顶级心法类
8. 工具/神技类

每个母题下有**分组**（如"暴力战法类"→ 图形战法/分时盘口/模式战法/龙头选股/打板连板/止盈止损），分组是轻量标签、不是独立层级，系统按子题名自动归，运营不用手动选。

### 字段清单

**母题**
- 母题名 *（预置 8 个，后续可加）
- 下属分组列表

**子题（选题卡）**

录入必填（3 秒完成）：
- 子题名 *
- 一句话选题（讲什么、什么钩子）*
- 归属母题 *（8 大类选一个，高频在上低频在下）

选填（默认不展开，想填就填）：
- 情绪标签（恐惧/贪婪/焦虑/迷茫/认同/好奇，单选）
- 来源（人工录入 / 系统推荐 / 素材库浮出）

收在"更多"里（低频）：
- 受众画像（小白入门/焦虑进阶/成熟验证/高净值，四档）

系统自动挂（运营不用管）：
- 分组（系统按子题名自动归到母题下的分组）
- 创建人、录入时间

已明确砍掉：~~切入角度~~、~~时效~~、~~建议配号/IP~~、~~合规标签~~、~~内容价值标签~~

**注意：子题本身不带"状态"。** 状态跟着"人 × 子题"走——见下篇 §4。

**作品**（= 某人基于某子题拍出来发布的视频）
- 关联子题 *
- 认领人/发布号/IP
- 文案内容（多版本收纳，见 §子题卡信息层级）
- 发布时间
- 数据：播放量、完播率、互动、转粉（发布后回填）

### 子题卡信息层级（文案收纳）

运营在选题池里挑选时，子题卡按以下优先级展示信息：

1. **第一眼（判断选题好不好）**：子题名 + 一句话 + 母题 + 汇总数据（≥1000流量的作品平均播放量）
2. **第二眼（参考执行情况）**：最好/最近的文案版本 + 对应视频数据
3. **展开详情（深入参考）**：历史所有文案版本 + 各自数据 + 相近选题的文案也归纳其中

> 文案收纳是核心功能——运营挑选题时不只看"一句话描述"，还能看到"这个选题被做成了什么文案、跑出了什么数据"，帮他判断值不值得做、可以参考什么角度。支持多版本。

## 3. 选题从哪来（三条路，人工为主）

1. **运营主动录入（主力）**——遵守铁律1，极轻：母题 + 一句话，丢进候选池。想到就丢，不打断手上的活。
2. **系统从素材库浮候选（辅助，带理由）**——素材库里的优质内容（高播放/高完播/高转粉），系统提炼成候选选题浮上来，每条带"为什么推"（基于哪条爆款、什么信号、可换什么新角度、适合哪个号）。**系统只负责发现，运营点一下决定收不收**，不自动灌进库、不造垃圾选题。
3. **素材库轻挂钩**——素材库作为来源之一接进来（第2条就是它的用法），但**不深度耦合**：素材库照旧管它的视频资产，选题库只是"从里面浮候选"这一条单向通道。
