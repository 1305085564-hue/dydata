# 数据口径

## 1. 6个核心指标

| 指标 | 定义 | 公式/口径 | 来源 |
|------|------|-----------|------|
| 24h播放量 | 视频发布满 24 小时后的播放量 | 原始值，无二次计算 | `play_count`，作为各比率分母与爆款系数分子 |
| 互动率 | 点赞、评论、收藏、分享总和占播放的比例 | `(点赞 + 评论 + 收藏 + 分享) / 播放` | `interactionRate()` |
| 粉转率 | 涨粉占播放的比例 | `涨粉 / 播放` | `followerConversionRate()` |
| 导粉率 | 导粉占播放的比例 | `导粉 / 播放` | `fanConversionRate()` |
| 主页访问率 | 主页访问占播放的比例 | `主页访问 / 播放` | `homepageVisitRate()` |
| 爆款系数 | 当前视频 24h 播放相对账号基线的倍数 | `本视频24h播放 / 基线中位数` | `breakoutCoefficient()` |

说明：
- 以上 6 项里，5 个计算公式来自 [src/lib/video-metrics.ts](../src/lib/video-metrics.ts)。
- `24h播放量` 是原始观测值，不在该文件里做函数计算，但它是所有比率指标的分母，也是爆款系数的分子。

## 3. 爆款判定标准

爆款判定口径：
- `24h播放 ≥ 账号近30条中位数 × 2`

等价表达：
- `爆款系数 ≥ 2`

说明：
- 账号基线优先取近 30 条视频的播放样本中位数。
- 若样本不足，按下文“爆款系数冷启动规则”处理。

## 4. 下滑判定标准

下滑判定口径：
- `近7条中位数较前7条下降30%`

公式：
- `(近7条中位数 - 前7条中位数) / 前7条中位数 ≤ -30%`

判定前提：
- 两个窗口都必须有足够样本。
- 若前7条中位数为 0 或为空，不判定下滑。

## 5. 异常状态枚举

| 状态 | 含义 |
|------|------|
| normal / 正常 | 数据处于正常波动区间，无明显异常信号 |
| abnormal / 异常 | 新提交统一异常标记；异常详情下沉到 `punish_type`、`platform_notice`、`appeal` |
| 限流 | 旧数据兼容状态，读模型映射为异常，处罚类型为 `limited` |
| 删稿 | 旧数据兼容状态，读模型映射为异常，处罚类型为 `deleted` |
| 投流 | 旧数据兼容状态，读模型映射为异常，处罚类型为 `paid_boost` |
| 活动干预 | 旧数据兼容状态，读模型映射为异常，处罚类型为 `campaign_intervention` |
| 未满24h | 视频尚未到 24h 观察点，暂不进入 24h 口径比较，也不自动进入避坑案例库 |

说明：
- 新写入只使用 `normal / abnormal` 两档。
- 视频提交链路的异常状态仍以 `videos.anomaly_status` 为准，用于视频复盘与历史异常详情兼容；避坑案例大盘列表统一从 `violation_cases` / `knowledge_cases` 读取，不再把 `videos` 当列表主数据源。
- 高转化话术新提交写入 `knowledge_cases(status=submitted)`，违规话术新提交写入 `violation_cases`。

## 6. 观察窗口说明

采用 T+0 / T+1 双阶段观察：

| 时间 | 要求 |
|------|------|
| T+0 | 提交基础信息：视频标题、发布时间、基础截图、账号归属等 |
| T+1 | 补 24h 数据：播放、点赞、评论、分享、收藏、涨粉，以及基于 24h 数据计算的核心比率 |

说明：
- T+0 主要解决“有无发布、发了什么、何时发”的记录问题。
- T+1 才进入统一比较口径，避免未满 24h 的视频直接横向比较。

## 7. 爆款系数冷启动规则

账号基线由 `getAccountBaseline()` 计算，规则如下：

| 样本量 | 口径 | strategy |
|--------|------|----------|
| ≥10条 | 直接取账号自身播放中位数 | `self` |
| 3-9条 | 账号自身中位数 × 50% + 团队中位数 × 50% | `mixed` |
| <3条 | 不给基线，不计算爆款系数 | `insufficient` |

补充：
- 中位数函数见 [src/lib/video-metrics.ts:33-39](../src/lib/video-metrics.ts#L33-L39)。
- 基线策略见 [src/lib/video-metrics.ts:41-56](../src/lib/video-metrics.ts#L41-L56)。

## 8. 内容吸引力指标优先级

内容吸引力优先看“先吸引点击，再承接停留，再放大互动与转化”。建议优先级如下：

| 优先级 | 指标 | 作用 |
|--------|------|------|
| P1 | 24h播放量 | 最终结果指标，先看是否拿到分发 |
| P2 | 爆款系数 | 判断是否显著跑赢账号常态基线 |
| P3 | 互动率 | 判断内容是否激发反馈与参与 |
| P4 | 主页访问率 | 判断内容是否推动用户进一步了解账号 |
| P5 | 粉转率 | 判断内容是否带来直接涨粉 |
| P6 | 导粉率 | 判断内容是否完成更深层转化 |

使用建议：
- 看单条是否“爆”时，先看 `24h播放量 + 爆款系数`。
- 看内容本身是否“有吸引力”时，先看 `互动率`，再看 `主页访问率 / 粉转率 / 导粉率`。
- 若播放高但转化弱，优先排查内容承接问题；若播放低但转化高，优先排查分发与选题问题。

## 9. 个人成长页六维能力口径

成长页只允许使用以下一套能力六维，唯一代码来源是 `src/lib/growth-page.ts` 的 `GROWTH_DIMENSION_RULES` 与 `getDimensionValues`：

| 维度 | 计算 |
| --- | --- |
| 开头留人 | `completion_rate_5s` 平均值 |
| 中段跳出 | `max(5秒完播率 - 整体完播率, 0)`，越低越好 |
| 整体完播 | `completion_rate` 平均值 |
| 增长转化 | 每条 `follower_gain / play_count × 100` 后取平均 |
| 互动吸引 | 每条加权互动数除以播放后取平均；权重为评论 0.35、分享 0.25、点赞 0.25、收藏 0.15 |
| 话题爆点 | `play_count` 平均值 |

评级统一比较当前用户与其他同事团队均值：表现比例 `≥1.1` 为强、`≤0.9` 为弱，中间为中；中段跳出因越低越好，比例方向反转。完整空值、可信度、对标和前端字段约定见 `docs/plans/2026-07-10-个人成长页面重构-数据契约.md`。
