## 任务清单（按顺序执行）

### 任务 1：VideoSubmitForm 增加草稿自动保存

**目标**：用户填写过程中每 30 秒自动保存到 localStorage，刷新后可恢复。

**文件**：`src/app/(app)/dashboard/video-submit-form.tsx`

**要求**：
1. **抽离为独立 hook**：新建 `src/hooks/use-form-draft.ts`，不要在原文件中直接堆逻辑
2. **hook 接口**：
   ```typescript
   function useFormDraft<T>(key: string, formData: T, deps: unknown[]): {
     hasDraft: boolean;
     restoreDraft: () => T | null;
     clearDraft: () => void;
     lastSavedAt: Date | null;
   };
   ```
3. **自动保存逻辑**：
   - 使用 `useEffect` + `setInterval(30000)` 或防抖（推荐 `useDebounce`）
   - 只在表单有内容且未提交时保存
   - 保存前检查数据是否有变化（深比较），无变化不写入
   - 保存时带时间戳
4. **草稿 key**：`dydata.draft.videoSubmit.{userId}`，按用户隔离
5. **数据范围**：保存所有表单字段（截图槽位状态、元数据、话题标签、文案等）
6. **提交成功后**：自动清除草稿

**验收标准**：
- 填写表单，等待 30 秒，刷新页面，数据恢复
- 提交成功后，草稿被清除，再次刷新无恢复提示
- 多用户登录时，草稿不串号

### 任务 2：草稿恢复提示

**目标**：页面加载时发现草稿，给用户选择权。

**文件**：`src/app/(app)/dashboard/video-submit-form.tsx`

**要求**：
1. 组件挂载时检查 `hasDraft`
2. 有草稿时，在表单顶部显示横幅：
   - 文案：「检测到未提交的草稿（最后保存于 14:32），是否恢复？」
   - 按钮：「恢复草稿」/「丢弃，重新填写」
   - 样式：使用 warning 色 `#D99E55` 边框，背景浅色
3. 点击「恢复草稿」：调用 `restoreDraft()` 填充表单
4. 点击「丢弃」：调用 `clearDraft()`，横幅消失
5. 横幅可手动关闭（X 按钮）

**验收标准**：
- 有草稿时显示横幅，无草稿时不显示
- 恢复后表单数据正确
- 丢弃后横幅消失，表单为空

### 任务 3：截图槽位增加具体错误原因

**目标**：用户知道截图为什么识别失败，知道怎么解决。

**文件**：截图相关组件（在 `video-submit-form.tsx` 或拆分的子组件中）

**要求**：
1. 定义错误类型和对应文案：
   ```typescript
   const OCR_ERROR_MESSAGES: Record<string, string> = {
     "BLURRY": "图片模糊，请重新截图确保文字清晰",
     "NO_TEXT": "未识别到文字内容，请检查截图是否包含数据",
     "LOW_CONFIDENCE": "识别置信度低，请手动核对或重新截图",
     "TIMEOUT": "识别超时，请检查网络后重试",
     "UNKNOWN": "识别失败，请手动填写或稍后重试",
   };
   ```
2. OCR 返回错误时，解析错误码，显示对应文案
3. 在截图槽位下方显示错误原因（红色小字）
4. 提供「重新识别」和「手动填写」两个操作按钮

**验收标准**：
- 模糊截图显示「图片模糊...」
- 无文字截图显示「未识别到文字...」
- 用户知道下一步该做什么

### 任务 4：入队审批增加批量操作

**目标**：管理员可以一次性处理多条入队申请。

**文件**：`src/app/(app)/admin/join-request-review-list.tsx`

**要求**：
1. 每行增加复选框，表头有「全选」复选框
2. 选中至少 1 行时，顶部出现浮动操作栏：
   - 「批量同意（X）」按钮，主色 `#D97757`
   - 「批量拒绝（X）」按钮，红色 `#C9604D`
   - 「取消选择」按钮
3. 点击批量拒绝时，弹出一个模态框输入统一拒绝原因（textarea）
4. 批量操作使用乐观更新：选中的行立即消失，toast 提示「已批量同意 X 条申请」
5. 如果部分失败：显示「成功 X 条，失败 Y 条」，失败的行恢复显示

**验收标准**：
- 可选中多行，顶部出现操作栏
- 批量同意后行消失，toast 提示
- 批量拒绝时弹出原因输入框
- 部分失败时正确恢复失败的行
