# 数据填报美学施工方案 V1（全程强制使用 skill）

## 0 施工总原则（执行前必读）

1. 本文件是"原思路"经法典 V1.2 校对后的**施工版**，原思路的电影卷轴动画、奇偶斑马行、`animate-pulse` 已废止（见 §5 禁止项）
2. 任何与法典冲突的本地方案一律按法典为准
3. 改造前后都要跑附件 grep 清单（法典附件 V1.2）
4. 两个场景分开施工，不互相串味：
   - **员工端**：`src/app/(app)/dashboard/`（填报+本人数据，30 人日常用）
   - **管理端**：`src/app/(app)/admin/analytics/`（复盘+爆款分析+长列表，负责人用）
5. 员工端追求"填得顺"，管理端追求"扫得清"，两套密度/节奏不同

## 1 员工端（/dashboard）施工清单

### 1.1 场景画像

- 用户：30 人团队成员
- 任务：每日 11:15 前填完当日数据 + 扫一眼本人趋势
- 痛点：输入框干瘪、反馈冷漠、没有"填完一条"的成就感
- 目标：填得快、反馈轻、本人主数据一眼看到

### 1.2 改造项

#### 1.2.1 输入框 Soft Fill（D.1）

**现状**：`video-submit-form.tsx` / `日报提交面板.tsx` 输入框底色偏生硬

**改为**：

```tsx
// 默认态
className="bg-zinc-100/70 border-0 rounded-[8px] px-4 py-3 text-[14px] text-zinc-800"

// Focus 态（替换原有 focus:ring-blue-*）
className="focus:bg-white focus:ring-1 focus:ring-zinc-950/5 focus:outline-none transition-[background,box-shadow] duration-150"
```

验收：grep 确认没有 `focus:ring-blue` / `focus:ring-indigo` / `border-blue` 残留

#### 1.2.2 数字 tabular-nums（C.3 / E.1）

**涉及文件**：`dashboard-content.tsx` / `history-list.tsx` / 任何展示播放量/点击率的位置

**规则**：

- 数字本身：`font-mono tabular-nums text-[18px] text-zinc-800 font-semibold`（Title 档）
- 单位（"万" / "%" / "条"）：`text-[12px] text-zinc-400 ml-1 font-normal`
- 对比值（+12% / -3%）：只用四语义色文字，不用彩色背景
  - 正向 `text-[#6FAA7D]`
  - 负向 `text-[#C9604D]`
  - 持平 `text-zinc-400`

#### 1.2.3 填报成功反馈（克制惊喜）

**位置**：日报提交成功后

**现状**：可能有 toast 或蓝色提示

**改为**：

- 提交按钮 → 短暂变 `#6FAA7D` 文字「已提交 {时间}」，2 秒后静默收起
- **禁止**：spinner、弹窗、动画图标、`animate-pulse`
- **允许**：按钮本身一次性的 `active:translate-y-0` 回弹（法典 D.4）

#### 1.2.4 本人趋势卡（A.3 主次分明）

**位置**：`dashboard-content.tsx` 顶部

**现状**：数据并排堆砌，主次不分

**改为**：

- 第一眼锚点 = **本人今日核心数字**（例如累计播放量 or 转粉数），用 `text-[32px] font-semibold tabular-nums text-zinc-800`
- 其余指标降级为 Caption 档 (13px) + zinc-500，横向平铺在下方
- 锚点数字的卡片周围留白 ≥ 48px（A.2 舒适档）

**Display 档唯一性硬规（法典 C.3）**：

- 每屏最多 1 处 32px Display，此锚点用掉了员工端首屏唯一名额
- 施工前 grep `text-\[32px\]` 及 `text-\[28px\]`（视为 Display 近档），确认首屏其他位置（alert-center / video-submit-panel / history-list）无并列大数字；有则降到 Title 18px 或 Section 24px
- 其他场景如需大数字，必须在此锚点之外的"第二屏"（滚动后才进入视口）或弹窗内

#### 1.2.5 异常提示（四语义色替换蓝色）

**涉及**：alert-center.tsx / 任何 `text-red-500` / `bg-yellow-100` 残留

**替换表**：

| 原 | 改 |
|---|---|
| `text-red-*` / `bg-red-*` | `text-[#C9604D]` + 2px 左导轨 `border-l-[#C9604D]` |
| `text-amber-*` / `bg-yellow-*` | `text-[#D99E55]` |
| `text-green-*` / `bg-emerald-*` | `text-[#6FAA7D]` |
| `text-blue-*` / `bg-blue-*` | 按语义判断，多半是"信息"→ 降级到 zinc 灰阶 + 2px 导轨 |

## 2 管理端（/admin/analytics）施工清单

### 2.1 场景画像

- 用户：你（owner）+ admin
- 任务：复盘 50–450 条视频数据、找爆款、看团队排名、导出月报
- 痛点：一口气 450 条堆在一起视觉焦虑、爆款和异常混在一起看不出重点
- 目标：第一眼锁定爆款 → 2px 导轨翻页 → 点击单条看详情

### 2.2 改造项

#### 2.2.1 页面第一眼锚点：焦点卡（A.3）

**位置**：analytics-workbench.tsx / analytics-content.tsx 顶部

**设计**：

- 单行焦点卡：当周/当日爆款第一名
- 卡片尺寸明显大于列表行（padding 32px，数字 32px）
- 左 2px 导轨 `border-l-[#D97757]`（Primary 激活）
- 不要任何阴影，仅靠 L2 白 + 1px zinc-200 细边 + 留白包围

**拒绝**：多卡片并排（违反 A.3 并列=失败）

#### 2.2.2 视窗化滚动 + 2px 时间导轨（E.6 升级用法）

**原思路保留**：右侧 2px 滚动导轨 + 滚动时出现 `#D97757` 激活点 + 悬浮显示"51–100 / 450"

**施工细节**：

- 列表容器固定高度：`max-h-[min(70vh,calc(100vh-280px))]` + `min-h-[360px]`
  - `calc(100vh-280px)` 适配常规尺寸；MacBook Air 13" 1.5× 缩放后视口约 700px，剩余 420px 仍可装下 6 行 `py-3` 列表
  - `min-h-[360px]` 兜底，避免极小屏被压到 < 5 行；容器内部滚动
- 容器内部滚动，浏览器整页不滚
- 右侧导轨：`w-[2px] bg-zinc-200`，绝对定位在容器右内侧 4px
- 激活点：`h-2 w-2 rounded-full bg-[#D97757] ring-1 ring-white`，按滚动比例垂直位移
- 悬浮提示：`text-[12px] text-zinc-500 tabular-nums`，距离激活点左侧 12px
- **动画**：仅位移，使用 `transition-transform duration-150 ease-out`，禁止弹簧

#### 2.2.3 分批切换（静默替换，禁用电影卷轴）

**原思路里的"电影卷轴滑动转场"废止**（违反 A.5 第一/第二条）

**改为**：

- 滚动到底部出现占位符按钮「查看更早的批次 (Batch 2 · 101–150)」
- 点击后：
  - 列表内容**静默替换**，使用骨架屏（2–3 行 zinc-100 占位条）过渡 ≤ 200ms
  - 右侧导轨激活点同步跳到新批次位置
  - 滚动容器 `scrollTop = 0`（回到顶部）
- **禁止**：任何 translate 全列表位移、opacity 整块淡入淡出

#### 2.2.4 列表行设计（E.3 / E.6）

**结构**：

```
[ 2px 语义导轨 ] [ 标题 · 作者 ] ............ [ 核心数字 ] [ 状态点 ]
```

**规则**：

- 行背景：默认 `bg-[#FAFAFB]` (L1)，Hover `bg-white shadow-sm`（E.3 去色化）
- **禁止斑马行**（原思路的 L0/L1 奇偶交替作废，违反 A.2）
- 行间分隔：仅靠 12px 留白，**不加 border-bottom**（F 禁区：删卡片底边）
- 左 2px 导轨按语义：
  - 爆款 → `border-l-[#6FAA7D]`
  - 异常/数据跌幅 → `border-l-[#C9604D]`
  - 当前选中 → `border-l-[#D97757]`
  - 普通 → `border-l-transparent`（保留 2px 槽位，避免跳动）
- 行高：`py-3`（舒适档，禁用 py-2 紧凑档）

#### 2.2.5 数字排版（C.3 / E.1）

- 所有数字列：`font-mono tabular-nums`
- 主数字 Title 18px / 次数字 Label 13px
- 单位降到 Caption 12px + zinc-400
- 百分比的正负号用四语义色文字，不用背景色块

#### 2.2.6 状态点（E.5，去循环动画）

**位置**：每行右侧 / 焦点卡右上

**规则**：

- 尺寸 `h-2 w-2`（8px，法典上限）
- 外圈 `ring-1 ring-white`（必选）
- 颜色语义：
  - Critical `#C9604D`
  - Active `#D97757`（不循环）
  - Done `#6FAA7D`
  - Pending `#D99E55`
- **禁止 animate-pulse / animate-ping**（原思路废止项）
- 如需强化异常，改用"整行文字变 `#C9604D`"，不要用动画

#### 2.2.7 Tab / Segmented Control（E.4）

**位置**：列表顶部「今日 / 本周 / 高光」筛选

**规则**：

- 容器 `bg-zinc-100 rounded-xl p-1`
- 激活 `bg-white shadow-sm text-zinc-800 rounded-lg`（法典 E.4 / E.7 定死）
- 非激活 `text-zinc-500`
- 切换动画：如用 framer-motion 请用 `layoutId` 做滑块，禁弹簧过冲

#### 2.2.8 分页器（原思路保留）

- 当前页：`bg-white` + 1px `#D97757/40` 描边 + `text-[#D97757]`
- 非当前：`text-zinc-500` 纯数字
- 左右切换：lucide 图标 `stroke-[1.5]`，hover `-translate-y-[1px]`

#### 2.2.9 日历/日期选择器（F 禁区强制）

**现状**：`dashboard-form.tsx` 及任何日历组件可能残留默认蓝色选中态

**改为**：

- 选中日 `bg-[#D97757] text-white rounded-[8px]`
- 今天标记 `ring-1 ring-[#D97757]/40`
- Hover `bg-zinc-100`
- 圆角按 C.1 规范：日格用 8px，容器用满圆/12px

#### 2.2.10 黑墨主按钮（E.7）

- 每屏至多 1 个 `bg-zinc-900`，位置建议：顶部「导出月报」或底部「归档复盘」
- 其他所有按钮降级为 outline（`border-zinc-200 text-zinc-800`）或 text
- 禁止黑按钮加 shadow

每批 PR 合并前跑一次附件 grep 清单 + 截图比对

### 4.1 并行执行方案（Worktree 多 Opus）

**为什么不能粗暴并行**：

- 公共组件（status-badge / status-theme / `<Input>`）是全站基座，P0 必须先串行完成
- 同一文件被两批改动会 merge 冲突（如 video-submit-form 同时涉及 Soft Fill 和填报反馈）

**切法**：P0 串行 → P1 + P2 并行 → P3 串行收尾

```
T0  主会话 P0（公共组件 + 员工端）→ push v1.2-upgrade
T1  开两个 worktree + 两个 Opus 会话：
    ├─ ../dydata-p1 / 分支 p1-admin-focus  → P1 管理端焦点卡 + 视窗化 + 时间导轨
    └─ ../dydata-p2 / 分支 p2-admin-list   → P2 列表行 + 状态点 + 分页器 + 日历
T2  两边 PR 合回 v1.2-upgrade → 主会话 P3 grep 验收 + 截图
```

**前置条件**：P0 完成后必须派 Explore 预扫一次，确认 P1 / P2 文件集不重叠（重点核对：列表行组件是不是 analytics-workbench 的子组件；如在同文件，P1/P2 不可并行）

**搭建命令（P0 完成后执行）**：

```bash
cd /Users/mac/Projects/dydata
git worktree add ../dydata-p1 -b p1-admin-focus v1.2-upgrade
git worktree add ../dydata-p2 -b p2-admin-list v1.2-upgrade

# 各 worktree 独立起 dev，端口错开
cd ../dydata-p1 && pnpm dev -p 3001    # Opus-A
cd ../dydata-p2 && pnpm dev -p 3002    # Opus-B
```

**会话交接档案**：每批做完写 `docs/plans/2026-05-11-数据填报美学施工进度.md`，4 行即可：已完成 / 当前状态 / 未完成线索 / 下次读哪些文件。新 Opus 启动先读此档 + 本施工方案对应章节。

**预期工时**：3 天 → 约 1.5 天（P0 半天 + P1/P2 并行 1 天 + P3 半天）

**冲突兜底**：若 P1/P2 合回时出现冲突，主会话手动解决（优先保留 P1 的容器改动 + P2 的行内改动），不要让 worktree 内部 rebase。

## 5 明确禁止项（原思路作废的部分）

1. **电影卷轴转场**（全列表 translate 位移） → 静默替换 + 骨架屏
2. **L0/L1 奇偶斑马行** → 只靠留白分隔
3. **`animate-pulse` / `animate-ping` 状态点** → 静态实色 + ring-white
4. **状态徽章彩色背景**（`bg-red-100` 等） → 2px 左导轨 + 文字着色
5. **弹簧过冲动画** → 统一 ease-out 150ms
6. **多个黑按钮并列** → 每屏 1 个终局黑按钮
7. **spinner** → 骨架屏 + 静默加载
8. **卡片底部 border-top** → `mt-6 pt-6` 留白

## 6 grep 验收清单（施工后必跑）

```bash
cd /Users/mac/Projects/dydata/src

# 禁区 1：蓝色/默认彩色

# 禁区 4：font-bold / font-black
grep -rnE "font-(bold|black)" app/

# 禁区 5：spinner
grep -rnE "<Loader2|<Spinner" app/

# 禁区 6：蓝色 focus
grep -rnE "ring-blue|border-blue" app/
```
