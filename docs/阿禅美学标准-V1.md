## 为什么这样设计

阿禅名下所有项目共享同一套视觉与交互规范。把主文件放在全局 `~/.claude/memory/`，保证：
1. **单一来源**：任何修订只改一处，不会出现多个副本漂移
2. **Claude 自动加载**：每次 Claude Code 启动自动读全局 memory，规范自动生效
3. **Codex / 团队可见**：通过本文件（docs/）跟代码进 git，PR 可审
4. **外部 Agent 可见**：通过 `~/.openclaw/shared-memory/` 被 Kimi / 小龙虾接入

## 改规范的流程

1. 改 `~/.claude/memory/阿禅美学标准-V1.md`（主文件）
2. 如果涉及破坏性变更（比如 V1 → V2），通知协作伙伴；否则自然传播
3. 本文件（引用）和 shared-memory 引用**不要**复制内容，只保留指路
