# 提示词：Sprint 6 — 催交与豁免闭环

## 你的身份
你是 DYData（抖音数据日报平台）的资深全栈开发工程师，精通 Next.js App Router + Supabase + TypeScript。

## 任务背景
催交系统和豁免系统是两个独立模块，没有联动：已申请豁免的成员仍会收到催交通知，管理员也不知道成员是否已申请豁免。

## 任务清单（按顺序执行）

### 任务 1：催交逻辑过滤已豁免成员

**目标**：已申请豁免的成员不收到催交。

**文件**：`src/app/api/remind/route.ts`（或等效催交 API）

**要求**：
1. 在查询未填报成员时，联查 `exemptions` 表
2. 过滤条件：
   - 该成员在目标日期有状态为 `pending`（待审核）或 `approved`（已通过）的豁免申请
   - 或者豁免申请覆盖的日期范围包含目标日期
3. 被过滤的成员不计入催交名单，也不发送飞书 webhook
4. 在催交记录中标记「已豁免跳过」，便于后续统计

**验收标准**：
- 已申请豁免的成员不在催交列表中
- 未申请豁免的正常成员仍收到催交
- 豁免被拒绝的成员继续收到催交

---

### 任务 2：豁免申请页面显示催交记录

**目标**：申请豁免时，成员知道该日期已被催交多少次。

**文件**：豁免申请相关页面（如 `src/app/(app)/exemption/**` 或等效文件）

**要求**：
1. 在豁免申请表单中，增加「催交记录」区域
2. 显示该成员在目标日期前 7 天内的催交次数
3. 文案：「该日期前后您已被催交 X 次」
4. 如果 X > 2，用 warning 色 `#D99E55` 高亮提示
5. 数据来自催交记录表（如不存在，可先 mock 或从现有数据推算）

**验收标准**：
- 申请页面显示催交次数
- 次数正确
- 高频催交有视觉提示

---

### 任务 3：催交记录可查

**目标**：管理员能看到历史催交记录。

**文件**：
- 新建页面或弹窗组件
- 修改 `src/app/(app)/admin/components/admin-cockpit.tsx` 增加入口

**要求**：
1. 在驾驶舱「待催交成员」卡片上，增加「查看历史」链接
2. 点击后打开弹窗或侧边栏，显示催交记录列表
3. 列表字段：
   - 催交日期
   - 被催交成员
   - 催交方式（飞书 webhook）
   - 发送状态（成功/失败）
   - 是否已豁免（是/否）
4. 支持按日期筛选、按成员搜索
5. 数据量大时分页，每页 20 条
6. 如果现有数据库无催交记录表，可：
   - 方案 A：新建表（需 migration，先问复核者）
   - 方案 B：从现有日志或 API 调用记录中推算

**验收标准**：
- 管理员可查看历史催交记录
- 筛选和搜索正常
- 分页正常
