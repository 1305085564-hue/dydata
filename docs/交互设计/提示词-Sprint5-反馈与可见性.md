# 提示词：Sprint 5 — 反馈与可见性

## 你的身份
你是 DYData（抖音数据日报平台）的资深前端开发工程师，精通 React + TypeScript + Next.js + Tailwind CSS。

## 任务背景
用户经常不知道系统在干什么：网络断了没提示、错误信息一闪而过、演示模式和真实数据分不清、SOP 逾期需要刷新才知道。

## 任务清单（按顺序执行）

### 任务 1：增加全局网络状态指示器

**目标**：用户知道网络是否可用。

**文件**：
- 新建 `src/components/network-status-bar.tsx`
- 修改根布局文件（如 `src/app/(app)/layout.tsx`）

**要求**：
1. 监听 `navigator.onLine` 和 `online`/`offline` 事件
2. 离线时，在页面顶部显示固定横幅：
   - 文案：「网络已断开，部分功能可能不可用」
   - 颜色：error 色 `#C9604D` 背景，白色文字
   - 高度：32px，固定在顶部，`z-index: 50`
3. 网络恢复时，横幅变为绿色（success 色 `#6FAA7D`），显示「网络已恢复」，2 秒后自动消失
4. 不要遮挡页面内容（给 body 加 padding-top 或占位）
5. 首次加载时如果已离线，立即显示

**验收标准**：
- 断开 WiFi，横幅立即出现
- 恢复 WiFi，横幅变绿后消失
- 页面内容不被遮挡

### 任务 2：toast 增加「查看详情」展开

**目标**：错误信息可以展开看完整内容。

**文件**：`src/components/ui/feedback-toast.tsx`

**要求**：
1. 扩展 `feedbackToast.error`，支持传入 `details?: string` 参数
2. 有 details 时，toast 内容区域显示「查看详情」链接
3. 点击后展开显示 details 内容（使用折叠动画）
4. 开发环境（`process.env.NODE_ENV === 'development'`）下，details 默认显示错误堆栈
5. 生产环境下，details 只显示用户友好的说明，不暴露技术细节
6. 保持现有 toast 样式不变，只增加展开能力

**类型扩展**：
```typescript
interface ToastOptions {
  // ... 现有选项
  details?: string;
}
```

**验收标准**：
- 普通 toast 不受影响
- 带 details 的 toast 可展开/收起
- 开发环境自动显示堆栈
- 生产环境不暴露堆栈

### 任务 3：演示模式增加全局标识

**目标**：用户不会混淆演示模式和真实数据。

**文件**：
- `src/app/(app)/demo/layout.tsx` 或 `src/app/(app)/demo/page.tsx`
- 如有全局布局，修改 `src/app/(app)/layout.tsx`

**要求**：
1. 在演示模式页面顶部，增加固定横幅：
   - 文案：「演示模式 — 所有数据为模拟数据，不可提交」
   - 颜色：warning 色 `#D99E55` 背景，深色文字
   - 高度：36px，固定在顶部
2. 横幅右侧增加「退出演示」按钮，点击跳转到 `/login`
3. 演示模式下所有「提交」「保存」「删除」等操作按钮：
   - 视觉上禁用（opacity-50）
   - 点击时 toast 提示「演示模式不可操作」
   - 不要真正调用 API
4. 如果演示模式有独立 layout，在 layout 中统一处理；如果没有，在每个操作点判断 `DEMO_VIEWER` 角色

**验收标准**：
- 演示模式顶部有明显标识
- 提交按钮点击后提示不可操作，无 API 调用
- 退出演示按钮正常工作

### 任务 4：SOP 逾期增加主动查询提示

**目标**：管理员不用刷新也知道有逾期。

**文件**：`src/app/(app)/admin/components/admin-cockpit.tsx`

**要求**：
1. 在现有轮询（`useSafeFetch` 60 秒）基础上，增加 SOP 检查点状态的独立轮询
2. 或者复用现有数据，解析出逾期状态
3. 当存在逾期检查点时，对应状态卡片显示红色角标（右上角小圆点，带数字）
4. 角标样式：红色圆 `#C9604D`，白色数字，直径 18px
5. 点击角标或卡片，跳转到 SOP 详情页（如有）或展开详情
6. 如果 SOP 数据不在驾驶舱现有 API 中，可先 mock 或预留接口

**验收标准**：
- 有逾期时卡片显示红色角标
- 无逾期时不显示
- 角标数字正确
