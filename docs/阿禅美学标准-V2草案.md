# 阿禅美学标准 V2 · 纸本仪表 Paper Instrument

> 灵魂主题：**界面是铺在桌上的工作纸本，上面嵌着精密的指示灯与刻度**
> 诞生日期：2026-05-10 · 基于 V1 外部验收（Kimi + Gemini × 2 轮）收敛形成
> V1.1 commit：`8555188` · V2 未开工，本文件为**标准定稿前草案**

## 零、V2 解决的核心问题

V1 落地后阿禅感到"彩色像补丁贴上去"。根因不是 CTA 纪律，是：

- **底色色温冲突**：`zinc-100 (#F4F4F5)` 冷灰 vs `#D97757` 暖橙，两者色温不同族 → 彩色无归属感
- **彩色缺物理约束**：二维屏幕里，彩色直接平贴灰底 → 漂浮

**V2 解法 = 宏观色温一统 + 微观镶嵌约束 + 分工纪律**

---

## 一、灵魂三原则（V2 新增，凌驾于 V1 13 条之上）

### 1. 介质色温一统（宏观）

- **Zinc → Stone** 全局迁移（冷灰 → 暖灰），文字灰阶同步迁移
- 底色三层改为**暖米色系**：
  - L0 `#FBFBF8` Bone White（主底）
  - L1 `#F5F5EF` Parchment（次底 / 侧栏 / 导航）
  - L2 `#FFFFFF` Pure / 或 `#FAF9F7` Cream（卡片 / 焦点区）
- `#D97757` 暖橙在暖米底上天然同族 → 彩色从介质中"长出来"，不再"贴"上去

### 2. 彩色镶嵌，不裸贴（微观）

- **主 CTA 暖橙按钮**：背景 `#D97757` + 1px 同色系深边 `#C96442`（倒角收边，压住浮躁）
- **状态点**：6-8px 实心圆 + **1px 底色外圈描边 `#FBFBF8`**（模拟珐琅镶嵌在纸面凹槽）
- **左 2px 彩色导轨**：保留并升级为"嵌入纸面的刻度条"视觉
- **禁用**：内阴影拟物凹槽（违反 V1「弥散阴影 ≤3%」哲学，会走向 iOS6 拟物复辟）

### 3. 灰黑做阅读，彩色做几何体（分工纪律）

- 文字永远 `stone-800` / `stone-500`，**禁止文字大面积语义色**
- 彩色只作为**独立几何体**出现在文字旁边：点 / 条 / 块 / 图表线
- 违禁示例：`text-red-500` 大段错误文案 → 改为 `text-stone-800` + 左 2px `#C9604D` 导轨 + 文字前 4px 实心小方块

## 二、字号阶梯 · 7 档（V1 3 档 → V2 7 档）

基于 4px 纵向节奏，字重遵守 V1「最高 semibold」硬规。

| 级别 | 字号 | 行高 | 字重 | tracking | 应用场景 |
|---|---|---|---|---|---|
| Display | 32px | 40px | semibold | tight | 核心数据大数 / 看板首页主标 |
| Section | 24px | 32px | semibold | tight | 模块化标题 / 大卡片标题 |
| Title | 18px | 24px | medium | normal | 弹窗标题 / 列表分组标题 |
| Body | 13px | 22px | regular | normal | **标准正文**（继承 V1 13px/1.7，拒绝改 14px 避免全站返工） |
| Label | 13px | 16px | medium | normal | 表格表头 / 侧栏导航（紧凑行高区别于 Body） |
| Caption | 12px | 16px | regular | normal | 图表脚注 / 输入辅助文字 |
| Eyebrow | 10px | 16px | medium | **0.25em** | 全大写小眉题（保留 V1 Claude 标志性字距，拒绝 Gemini 的 5%） |

**驳回 Gemini**：
- Body 从 13px 改 14px：全站返工成本极高，且 13px/1.7 经 V1 验证可读性已足
- Eyebrow tracking 5%：丢失 V1「Claude 暖色 × 宽字距」的标志性识别度
- 字重术语"Bold 600"：违反 V1 禁用词「bold」，统一口径为 `semibold` / `medium` / `regular`

## 三、间距系统 · 6 档（V1 未定义 → V2 新建）

基于 4px baseline，Gemini 5 档基础上**补 32px**。

| 级别 | 值 | 范畴 | Tailwind | 典型用法 |
|---|---|---|---|---|
| XS | 4px | 组件内 | `gap-1` | 图标↔文字 / 输入框内边距 |
| S | 8px | 组件间 | `gap-2` | 按钮组 / 标签组 |
| M | 16px | 卡片内 | `p-4` | **标准卡片 padding** / 列表项垂直 |
| L | 24px | 卡片间 | `gap-6` | Dashboard 卡片横纵间隙 |
| XL | 32px | 区块间 | `gap-8` | **卡片内重区块分隔** / 密度面板（V2 新增） |
| 2XL | 40px | 模块间 | `gap-10` | 顶部导航↔内容 / 大功能块 |

## 四、状态点 V2 · 四态镶嵌规则（V1 单规则 → V2 四态）

所有状态点统一 **1px `#FBFBF8` 外圈描边**（珐琅镶嵌），**禁用内阴影**（违反弥散阴影哲学）。

| 状态 | 尺寸 | 填色 | 外圈 | 动效 | 权重 |
|---|---|---|---|---|---|
| Critical | 8px | `#C9604D` | 1px `#FBFBF8` | 2s pulse 呼吸 | 最高 |
| Active | 8px | `#D97757` | 1px `#FBFBF8` | 静态 | 高 |
| Done | 8px | `#6FAA7D` | 1px `#FBFBF8` | 静态 | 中 |
| Idle | 6px | `#E7E5E4` stone-200 | 无 | 静态 | 低 |

**并列规则**：
- Critical + Active 共存时，Critical 保持 8px 脉冲，Active **降权 6px 静态**
- 纯 Idle 集群不得加动效，避免仪表盘"处处在闪"

## 五、主 CTA 纪律 · 展示页替代方案（V1 缺失 → V2 新建）

### 5.1 唯一性判定

- **主 CTA 定义**：页面内唯一能触发"数据状态变更 / 漏斗下钻"的动作
- **一屏一暖橙**：若存在两个暖橙按钮，次要者降级为 Outline（`border-stone-200 bg-transparent text-stone-800`）或 Stone-100 填充

### 5.2 展示型页面（growth / analytics / conversion-hub）

- **不强制**加暖橙"下一步行动"锚点 → 避免每屏都喊着"快点我"
- **替代方案**：引导性几何体
  - 目标模块左侧 2px 暖橙垂直线段（与左 2px 状态导轨同款但更短，控制在模块局部）
  - 关键数字下划线：1px thickness / 2px offset / `stone-300`（低调锚点，聚焦不打扰）

## 九、回滚预案

V2 动 token 层，风险最高。开工规则：

1. 开工前 `git tag v2-start`
2. 每批次单独 feature 分支，main 合入前单独 tag
3. V2 一键回滚：`git reset --hard v2-start`
4. V1 基座回滚：`git reset --hard 0bcf223`
5. 全部回滚至重塑前：`git reset --hard 76e56f6`（慎用）
